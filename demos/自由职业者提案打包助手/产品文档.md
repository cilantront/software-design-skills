## 1. 产品概述

### 1.1 产品定位

自由职业者提案打包助手是一款面向独立设计师、程序员、摄影师、小型外包服务商等自由职业者的AI辅助提案生成效率工具。通过整合客户管理、沟通记录、AI提案生成、智能报价、模板品牌化、PDF导出等能力，将提案准备时间从1-2小时缩短至10-15分钟。

### 1.2 核心价值主张

| 用户痛点 | 产品解决方案 |
|---------|------------|
| 客户信息散落在微信、邮件中，查找困难 | 集中化客户档案+沟通记录管理 |
| 每次写提案从零开始，耗时1-2小时 | AI基于历史案例+客户需求自动生成提案大纲 |
| 报价凭感觉，缺乏数据支撑 | AI智能报价建议（行业基准+历史数据） |
| 提案格式不统一，缺乏品牌感 | 模板库+品牌化元素一键应用 |
| PDF排版耗时，反复调整 | 一键导出专业PDF |

### 1.3 产品差异化

- 聚焦"个人/小团队"，不做 PandaDoc/HelloSign 等高价企业工具
- AI驱动，从沟通记录→需求提取→提案生成全链路AI辅助
- 低价入门：免费版3份/月，Pro版仅¥29/月
- 5天MVP快速上线

### 1.4 目标用户

| 用户角色 | 描述 | 典型场景 |
|---------|------|---------|
| 自由职业者（核心用户） | 独立设计师、程序员、摄影师、独立经纪人 | 每周需要向2-3个潜在客户发送定制化提案 |
| 小型团队负责人 | 2-10人团队的外包服务商/工作室负责人 | 需要统一管理团队成员提案，展示团队品牌 |

### 1.5 商业模式

| 版本 | 价格 | 核心权益 |
|-----|------|---------|
| 免费版 | ¥0 | 3份提案/月，基础模板，个人使用 |
| Pro版 | ¥29/月 | 不限提案，高级模板，团队协作，客户品牌化 |

## 3. 功能规格说明

### 3.1 客户管理模块

#### 3.1.1 客户档案列表页

**功能描述**：以卡片/列表形式展示所有客户，支持搜索和状态筛选。

**页面元素**：
- 顶部搜索栏：支持按客户名称、公司名称搜索
- 状态筛选标签：全部 / 活跃 / 休眠 / 已成交 / 潜在
- 客户卡片（列表项）：显示客户名称、公司、状态标签、最近沟通时间、关联提案数
- "新建客户"按钮：固定在列表顶部

**交互逻辑**：
- 默认按最近沟通时间倒序排列
- 搜索实时过滤，无需回车
- 点击卡片进入客户详情页
- 空状态显示引导文案："还没有客户？立即创建第一个客户"

#### 3.1.2 客户详情页

**功能描述**：管理单个客户的完整信息、沟通记录、需求摘要、历史提案。

**页面结构**：
- 顶部区域：客户基本信息（名称、公司、联系方式、状态标记）
- Tab切换：沟通记录 | 需求摘要 | 历史提案
- 操作按钮：编辑客户、新建沟通记录、创建提案

**客户信息字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户名称 | 文本 | 是 | 个人姓名或公司简称 |
| 公司名称 | 文本 | 否 | 所在公司 |
| 联系电话 | 电话 | 否 | |
| 邮箱 | 邮箱 | 否 | |
| 行业 | 下拉选择 | 否 | 互联网/金融/教育/医疗/零售/其他 |
| 来源渠道 | 下拉选择 | 否 | 朋友介绍/平台接单/社交媒体/其他 |
| 客户状态 | 单选 | 是 | 潜在/活跃/已成交/休眠/已流失 |
| 备注 | 长文本 | 否 | |

### 3.2 沟通记录模块

#### 3.2.1 沟通记录添加

**功能描述**：记录每次与客户的沟通内容。

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 沟通时间 | 日期时间 | 是 | 默认当前时间 |
| 沟通方式 | 单选 | 是 | 微信/电话/邮件/面谈/其他 |
| 内容摘要 | 长文本 | 是 | 本次沟通的关键内容 |
| 涉及需求 | 文本标签 | 否 | 本次沟通涉及的需求关键词 |

#### 3.2.2 沟通时间线

**功能描述**：按时间线形式展示与某客户的全部沟通历史。

**展示形式**：
- 垂直时间线，最新记录在顶部
- 每条记录显示：时间、方式图标、内容摘要、需求标签
- 支持按时间范围筛选

### 3.3 AI需求提取模块

#### 3.3.1 AI需求提取

**功能描述**：AI自动分析沟通记录，提取并归纳客户的核心需求。

**输入**：该客户的全部沟通记录文本
**输出**：结构化需求摘要，包含：
- 核心需求（列表）
- 偏好描述
- 预算范围（如有提及）
- 时间节点要求
- 特殊要求

**交互逻辑**：
- 用户点击"AI提取需求"按钮触发
- 显示加载状态（预计5-10秒）
- 生成结果展示在需求摘要区域，支持用户编辑/修正

#### 3.3.2 需求摘要编辑

**功能描述**：用户可编辑AI提取的需求摘要，确保准确性。

**交互逻辑**：
- 需求摘要以可编辑卡片形式展示
- 每个字段（核心需求、偏好、预算、时间节点）均可单独编辑
- 修改后标记"已手动编辑"

### 3.4 提案管理模块

#### 3.4.1 提案列表

**功能描述**：展示所有已创建的提案，支持搜索和状态筛选。

**页面元素**：
- 搜索栏：按客户名称、项目名称搜索
- 状态筛选：全部 / 草稿 / 已发送 / 已成交 / 已失效
- 提案卡片：显示项目名称、客户名称、创建时间、状态、报价总额
- "新建提案"按钮

#### 3.4.2 创建提案流程

**步骤一：选择客户**
- 从客户列表中搜索选择
- 选择后自动加载客户基本信息和需求摘要

**步骤二：填写项目简述**
- 项目名称（必填）
- 项目目标描述（必填）
- 预期交付时间
- 项目类型（设计/开发/摄影/咨询/其他）

**步骤三：AI生成提案大纲**
- 系统基于客户画像+需求摘要+项目简述，自动调用AI生成提案大纲
- 生成完成后进入提案编辑页

### 3.5 AI提案生成模块

#### 3.5.1 AI大纲生成

**功能描述**：AI自动生成定制化提案大纲。

**生成逻辑**：
- 输入：客户信息、需求摘要、项目简述、项目类型
- 输出：结构化提案大纲（JSON格式）

**默认大纲结构**：
1. 项目背景与理解
2. 解决方案
3. 工作范围与交付物
4. 时间计划与里程碑
5. 报价明细
6. 关于我们/个人介绍
7. 合作流程与下一步

**交互逻辑**：
- 生成过程显示进度（预计10-15秒）
- 生成完成后可逐章节编辑
- 支持拖拽调整章节顺序
- 支持增删章节

#### 3.5.2 提案内容编辑器

**功能描述**：富文本编辑器，支持自由编辑提案内容。

**编辑器能力**：
- 标题（H1-H4）
- 段落文本
- 列表（有序/无序）
- 表格
- 图片插入
- 分隔线

**交互逻辑**：
- 左侧为章节导航（可拖拽排序）
- 右侧为编辑区域
- 实时自动保存
- 工具栏提供格式操作

### 3.6 AI智能报价模块

#### 3.6.1 AI智能报价

**功能描述**：AI基于项目信息生成报价建议。

**输入**：项目简述、需求复杂度、项目类型
**输出**：
- 分项报价表（每个工作项的名称、预估工时、单价、小计）
- 总价区间（最低-最高）
- 报价依据说明

**交互逻辑**：
- 在提案大纲中包含"报价"章节时自动触发
- 也可在报价专区手动触发
- 生成结果以可编辑表格展示

#### 3.6.2 报价明细编辑

**功能描述**：用户可修改AI建议的报价明细。

**操作**：
- 修改单项价格
- 增删报价条目
- 调整工时/单价
- 总价自动重算

### 3.7 模板与品牌化模块

#### 3.7.1 内置模板库

**模板分类**：
- 设计类（设计师提案）
- 开发类（程序员技术方案）
- 摄影类（摄影服务提案）
- 通用咨询类

**模板内容**：预定义的章节结构、排版样式、配色方案

#### 3.7.2 模板自定义

**功能描述**：基于内置模板修改布局、章节结构、配色，保存为自定义模板。

**可自定义项**：
- 章节结构与顺序
- 字体选择
- 配色方案
- 页眉页脚内容

#### 3.7.3 品牌元素设置

**品牌元素**：

| 元素 | 说明 |
|------|------|
| Logo | 上传个人/团队Logo，显示在提案页眉 |
| 品牌色 | 主色+辅助色，影响提案配色方案 |
| 标准字体 | 标题字体+正文字体 |
| 联系页脚 | 自动添加在提案末页 |

### 3.8 提案导出模块

#### 3.8.1 全屏预览

**功能描述**：在生成PDF前，全屏预览提案最终效果。
- 显示品牌化元素
- 显示完整排版
- 支持翻页浏览

#### 3.8.2 一键导出PDF

**功能描述**：将编辑完成的提案导出为专业级PDF文件。

**导出逻辑**：
- 渲染提案内容+模板样式+品牌元素为PDF
- 保留完整排版
- 导出完成后自动下载

### 3.9 账号与订阅模块

#### 3.9.1 注册/登录

- 邮箱+密码注册，邮箱验证码激活
- 邮箱+密码登录
- 登录状态管理（记住登录、忘记密码）

#### 3.9.2 订阅管理

- 查看当前套餐、剩余提案数、到期时间
- 升级/续费操作

#### 3.9.3 个人资料

- 头像、昵称、个人简介、联系方式、个人网站

### 4.3 核心用户流程

#### 流程一：新建提案完整流程

```
工作台 → 点击"新建提案"
  → 选择客户（从客户列表）
  → 填写项目简述
  → AI自动生成提案大纲（10-15秒）
  → 进入提案编辑器
    → 编辑各章节内容
    → 调整报价明细
    → 选择模板
    → 应用品牌元素
  → 预览
  → 导出PDF
```

#### 流程二：AI需求提取流程

```
客户详情页 → 沟通记录Tab
  → 添加多条沟通记录
  → 点击"AI提取需求"
  → 系统分析沟通记录（5-10秒）
  → 展示结构化需求摘要
  → 用户编辑/确认
```

## 5. 数据模型设计

### 5.1 核心实体

#### User（用户）

```
User {
  id: UUID
  email: string (unique)
  password_hash: string
  nickname: string
  avatar_url: string
  bio: string
  phone: string
  website: string
  subscription_plan: enum (free, pro)
  subscription_expires_at: datetime
  monthly_proposal_count: int
  monthly_proposal_reset_at: datetime
  created_at: datetime
  updated_at: datetime
}
```

#### Client（客户）

```
Client {
  id: UUID
  user_id: UUID (FK -> User)
  name: string
  company: string
  phone: string
  email: string
  industry: string
  source_channel: string
  status: enum (potential, active, closed_won, dormant, lost)
  notes: text
  created_at: datetime
  updated_at: datetime
}
```

#### CommunicationRecord（沟通记录）

```
CommunicationRecord {
  id: UUID
  client_id: UUID (FK -> Client)
  user_id: UUID (FK -> User)
  communication_time: datetime
  channel: enum (wechat, phone, email, meeting, other)
  summary: text
  requirement_tags: string[]
  created_at: datetime
  updated_at: datetime
}
```

#### RequirementSummary（需求摘要）

```
RequirementSummary {
  id: UUID
  client_id: UUID (FK -> Client)
  user_id: UUID (FK -> User)
  core_requirements: text[]
  preferences: text
  budget_range: string
  timeline: string
  special_requirements: text
  is_ai_generated: boolean
  is_manually_edited: boolean
  extracted_at: datetime
  updated_at: datetime
}
```

#### Proposal（提案）

```
Proposal {
  id: UUID
  user_id: UUID (FK -> User)
  client_id: UUID (FK -> Client)
  project_name: string
  project_description: text
  project_type: enum (design, development, photography, consulting, other)
  expected_delivery: date
  status: enum (draft, editing, sent, closed_won, closed_lost)
  total_amount_min: decimal
  total_amount_max: decimal
  template_id: UUID (FK -> Template)
  created_at: datetime
  updated_at: datetime
  sent_at: datetime
}
```

#### ProposalSection（提案章节）

```
ProposalSection {
  id: UUID
  proposal_id: UUID (FK -> Proposal)
  title: string
  content: text (rich text / HTML)
  sort_order: int
  section_type: enum (background, solution, scope, timeline, pricing, about, next_steps, custom)
  is_ai_generated: boolean
  created_at: datetime
  updated_at: datetime
}
```

#### QuoteItem（报价条目）

```
QuoteItem {
  id: UUID
  proposal_id: UUID (FK -> Proposal)
  item_name: string
  description: text
  estimated_hours: decimal
  unit_price: decimal
  subtotal: decimal
  sort_order: int
  created_at: datetime
  updated_at: datetime
}
```

#### Template（模板）

```
Template {
  id: UUID
  user_id: UUID (FK -> User, nullable for system templates)
  name: string
  category: enum (design, development, photography, consulting, general)
  is_system: boolean
  section_structure: JSON (预定义章节结构)
  style_config: JSON (字体、配色、布局)
  created_at: datetime
  updated_at: datetime
}
```

#### BrandConfig（品牌配置）

```
BrandConfig {
  id: UUID
  user_id: UUID (FK -> User)
  logo_url: string
  primary_color: string
  secondary_color: string
  title_font: string
  body_font: string
  footer_text: text
  created_at: datetime
  updated_at: datetime
}
```

- 不同用户数据严格隔离

- Chrome 90+, Firefox 90+, Safari 15+, Edge 90+
- 优先适配桌面端（1280px+），兼顾平板

## 9. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| AI生成质量不稳定 | 用户体验差 | 提供手动编辑兜底，标识AI内容需人工审核 |

| 免费版转化率低 | 营收压力 | 在关键节点（如第3份提案）引导升级 |
