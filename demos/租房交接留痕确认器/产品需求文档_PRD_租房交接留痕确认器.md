# 1. 产品概述

## 1.1 产品定位

**一句话定位**：交接那一刻的第三方见证人。

租房交接留痕确认器是一款面向合租房租客、个人房东和小型公寓管家的轻量级微信小程序+WEB管理工具。产品聚焦于租客入住/退租交接这一高频且易产生纠纷的场景，通过结构化检查清单、多媒体留痕（照片/视频/读数/备注）、双方电子确认和自动PDF报告生成，为交接双方提供"说得清、有据查"的交接凭证。

## 1.2 目标用户

| 用户类型 | 画像 | 核心痛点 | 使用频率 | 付费意愿 |
| --- | --- | --- | --- | --- |
| 合租房租客 | 20-35岁，一二线城市，手机重度用户 | 退租时被房东扣押金说不清 | 年1-2次 | 单次¥9可接受 |
| 个人房东 | 30-55岁，有1-3套房源出租 | 租客损坏物品无据可查 | 年2-6次 | 单次¥9或月付¥49 |
| 小型公寓管家 | 25-40岁，管理5-30套房源 | 多房源交接效率低、归档混乱 | 月均10+次 | 月付¥49必要 |

## 1.3 核心价值主张

1. **结构化留痕**：按房间逐项检查，每一项都有照片/视频/读数/备注，不留死角
2. **不可篡改**：自动时间戳+锁定机制，留痕内容提交后不可修改
3. **一键出报告**：双方确认后自动生成A4排版的PDF交接报告，可直接用于协商或维权
4. **零门槛参与**：租客微信扫码即用，无需下载APP、无需注册

## 1.4 产品形态

| 端 | 形态 | 使用者 | 技术栈 |
| --- | --- | --- | --- |
| 租客端 | 微信小程序 | 租客（入住/退租） | 微信小程序原生 / Taro |
| 房东端 | 微信小程序 | 个人房东 | 微信小程序原生 / Taro |
| 管理后台 | WEB端 | 房东/管家 | Vue3 + Element Plus |

# 3. 功能详述

## 3.1 租客端核心功能

### 3.1.1 加入交接

**功能描述**：租客通过扫描二维码或点击微信分享链接进入交接任务。

**页面说明**：

| 元素 | 说明 |
| --- | --- |
| 交接信息卡片 | 显示房源地址、交接类型（入住/退租）、房东昵称 |
| 操作按钮 | "开始交接" — 进入身份验证或直接进入检查清单 |
| 提示文案 | "无需注册，微信扫码即可参与" |

**交互规则**：
- 首次加入：需通过手机号验证码验证身份，验证后自动进入检查清单
- 已有账户：微信授权登录后直接进入检查清单
- 邀请过期：显示"该交接邀请已过期，请联系房东重新发起"
- 链接有效性：24小时内有效，过期后需房东重发

### 3.1.2 检查清单与逐项留痕

**功能描述**：按房间分组展示检查项，租客逐项完成拍照/录像/读数/备注留痕。

**页面说明**：

| 元素 | 说明 |
| --- | --- |
| 顶部进度条 | 显示"已完成 X/Y 项"，实时更新 |
| 房间Tab | 横向滑动Tab，切换房间（主卧、次卧、客厅等），当前房间高亮 |
| 检查项卡片 | 每个检查项一张卡片，包含：检查项名称、状态标记（待检查/已完成）、参考照片（如有） |
| 状态标记按钮 | 四选一：完好(绿)、有损坏(红)、缺失(橙)、需维修(黄) |
| 照片上传区 | 点击调用相机或相册，支持多张，缩略图预览 |
| 视频上传区 | 点击录制或从相册选择，时长≤60秒 |
| 读数输入 | 数字键盘输入水电燃气表读数，自动计算与上次读数的用量差 |
| 备注输入框 | 文字输入，支持语音转文字按钮 |
| 底部导航 | "上一项"/"下一项"按钮，自动跳转到未完成项 |

**交互规则**：
- 标记为"损坏"或"缺失"时，照片上传变为必填项，未上传无法进入下一项
- 每项操作自动记录精确时间戳（yyyy-MM-dd HH:mm:ss），显示在卡片右下角
- 照片上传前自动压缩，单张不超过2MB
- 视频上传支持断点续传，上传中显示进度条
- 网络中断时数据暂存本地，网络恢复后自动上传（显示"已暂存，待上传"标记）
- 支持返回已完成的项修改（未提交确认前）

### 3.1.3 费用确认（退租场景）

**功能描述**：退租交接时，租客查看房东填写的扣费明细，逐项确认或标记异议。

**页面说明**：

| 元素 | 说明 |
| --- | --- |
| 扣费明细列表 | 每项显示：扣费项目、扣费金额、扣费依据、关联的损坏/缺失照片 |
| 逐项操作 | 每项右侧有"确认"和"异议"两个按钮 |
| 异议弹窗 | 点击"异议"弹出原因输入框，可补充照片 |
| 费用汇总 | 底部显示扣费总额、已确认项数、异议项数 |
| 确认按钮 | 全部处理后方可点击"确认费用" |

**交互规则**：
- 标记异议时必须填写异议原因（≥10字），可选补充照片
- 异议内容将原样体现在PDF报告中
- 全部项处理完毕后，底部"确认费用"按钮变为可点击状态

### 3.1.4 签名确认

**功能描述**：双方在确认页手写电子签名，确认交接内容属实。

**页面说明**：

| 元素 | 说明 |
| --- | --- |
| 留痕预览区 | 可上下滑动浏览全部留痕内容的缩略摘要（照片缩略图+读数+备注） |
| 签名板 | 横向签名区域，提示"请在此处手写签名" |
| 清除/重签按钮 | 清除当前签名重新书写 |
| 确认提交按钮 | 签名后方可点击 |
| 确认提示 | "提交后留痕内容将被锁定，不可修改，请确认无误后提交" |

**交互规则**：
- 签名笔迹以图片形式保存，写入PDF报告
- 提交后弹出确认对话框，确认后记录锁定
- 锁定后所有留痕内容变为只读状态

### 3.1.5 PDF报告

**功能描述**：双方确认完成后自动生成PDF交接报告。

**页面说明**：

| 元素 | 说明 |
| --- | --- |
| 报告预览 | A4排版的PDF预览，可翻页 |
| 下载按钮 | 保存到手机本地 |
| 分享按钮 | 通过微信发送给房东或其他联系人 |
| 复制链接 | 生成报告链接，可分享给非微信联系人 |

**PDF报告内容结构**：

```
第1页：封面
  - 报告标题："租房交接报告"
  - 房源地址
  - 交接类型（入住/退租）
  - 交接日期
  - 租客姓名、房东姓名

第2页：检查清单与留痕
  - 按房间分组
  - 每项包含：检查项名称、状态标记、留痕照片（缩略图）、读数、备注、时间戳

第3-N页：留痕照片详情
  - 每页4张排列
  - 照片下方标注所属房间和检查项

末页：双方确认信息
  - 水电燃气表读数汇总与用量计算
  - 扣费明细（退租场景）
  - 异议记录（如有）
  - 租客电子签名 + 签名时间
  - 房东电子签名 + 签名时间
  - 报告生成时间戳
```

## 3.2 房东端核心功能

### 3.2.1 创建交接

**功能描述**：房东通过分步引导完成交接任务创建。

**页面说明**：

| 步骤 | 页面内容 | 操作 |
| --- | --- | --- |
| Step 1 | 选择房源（列表或新增） | 点选房源卡片 |
| Step 2 | 选择交接类型 | 入住交接 / 退租交接 |
| Step 3 | 选择检查模板 | 从模板列表选择，或"空白创建" |
| Step 4 | 编辑检查清单 | 增删房间和检查项，调整排序 |
| Step 5 | 设置表计初始读数 | 填写当前水电燃气表读数 |
| Step 6 | 生成邀请 | 显示二维码和分享链接 |

**交互规则**：
- 退租交接时，自动拉取该房源最近一次入住交接记录作为对照基准
- 检查模板支持"使用上次交接清单"选项
- 二维码支持保存到相册，链接支持微信好友/朋友圈分享
- 邀请生成后，房东端显示"等待租客加入"状态

### 3.2.2 审核留痕与确认

**功能描述**：房东查看租客留痕内容，确认通过或退回修改。

**页面说明**：

| 元素 | 说明 |
| --- | --- |
| 留痕总览 | 按房间分组的留痕内容列表，显示各项状态 |
| 留痕详情 | 点击查看单张照片大图、播放视频、查看读数和备注 |
| 差异对比（退租） | 左右对比入住时照片与退租时照片，高亮差异 |
| 操作按钮 | "确认通过" / "退回修改"（需填写退回原因） |
| 扣费填写区（退租） | 针对损坏/缺失项逐项填写扣费金额和依据 |

**交互规则**：
- 退回修改时需填写退回原因（≥10字），租客端收到通知
- 退租场景下，房东确认通过后方可填写扣费明细
- 扣费明细填写完毕后，需先预览汇总再提交给租客确认
- 双方均完成签名后，记录自动锁定

# 4. 数据模型

## 4.1 核心实体关系

```
用户（User）
  ├── 1:N → 房源（Property）      [房东创建]
  ├── 1:N → 交接记录（Handover）   [房东发起/租客参与]
  └── 1:1 → 订阅（Subscription）   [房东版]

房源（Property）
  ├── 1:N → 房间（Room）
  ├── 1:N → 交接记录（Handover）
  └── N:1 → 检查模板（Template）   [可选关联]

房间（Room）
  └── 1:N → 检查项（CheckItem）

检查模板（Template）
  ├── 1:N → 模板房间（TemplateRoom）
  └── 1:N → 模板检查项（TemplateCheckItem）

交接记录（Handover）
  ├── 1:N → 交接房间（HandoverRoom）
  ├── 1:N → 留痕记录（Evidence）
  ├── 1:N → 读数记录（MeterReading）
  ├── 1:N → 扣费明细（FeeItem）
  ├── 1:1 → 租客签名（TenantSignature）
  ├── 1:1 → 房东签名（LandlordSignature）
  └── 1:1 → PDF报告（Report）

留痕记录（Evidence）
  ├── 1:N → 照片（Photo）
  └── 1:N → 视频（Video）
```

## 4.2 核心字段说明

### 交接记录（Handover）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | UUID | 主键 |
| property_id | UUID | 关联房源 |
| type | enum | 入住交接/退租交接 |
| status | enum | 待参与/进行中/待审核/待确认费用/待签名/已完成/已归档/已过期/已取消 |
| related_handover_id | UUID | 退租时关联的入住交接记录ID |
| invite_code | string | 邀请码（用于生成二维码） |
| invite_expire_at | datetime | 邀请过期时间 |
| created_by | UUID | 创建人（房东） |
| participant_id | UUID | 参与人（租客） |
| locked_at | datetime | 记录锁定时间 |
| created_at | datetime | 创建时间 |
| completed_at | datetime | 完成时间 |

### 留痕记录（Evidence）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | UUID | 主键 |
| handover_id | UUID | 关联交接记录 |

# 6. 技术约束

| PDF生成 | Puppeteer / wkhtmltopdf | HTML模板渲染为PDF |
| 消息队列 | Redis (Bull) | 异步任务（PDF生成、通知推送） |

## 6.2 关键约束

| 约束 | 说明 |
| --- | --- |
| 小程序包体积 | 主包≤2MB，分包总计≤16MB（微信限制） |

| PDF生成 | 50张照片的报告≤15秒生成，A4排版 |
| 离线暂存 | 小程序本地存储≤200MB暂存空间，网络恢复后自动同步 |

| 签名 | MVP阶段电子签名不具备司法鉴定资质 |

## 6.3 安全要求

| 项目 | 要求 |
| --- | --- |
| 数据传输 | 全链路HTTPS加密 |
| 文件访问 | 对象存储使用签名URL，有效期24小时 |
| 用户数据 | 手机号脱敏存储，敏感操作需二次验证 |
| 接口鉴权 | JWT Token + 请求签名 |
| 数据隔离 | 用户只能访问自己的交接数据 |

# 10. 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| 照片/视频上传慢 | 用户体验差，可能中断交接 | 图片压缩+分片上传+离线暂存+进度提示 |
| PDF生成失败 | 用户无法获取交接凭证 | 异步生成+重试机制+生成失败通知 |
| 电子签名法律效力不足 | 用户信任度低 | MVP阶段明确告知"辅助凭证"定位；后续接入第三方CA认证 |
| 小程序审核不通过 | 上线延期 | 提前研究微信小程序审核规范，避免敏感类目 |
| 并发交接数据冲突 | 留痕数据丢失 | WebSocket实时同步+乐观锁+冲突检测 |
