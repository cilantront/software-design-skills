> **产品阶段：** MVP（10-14天）

## 1. 产品概述

### 1.1 产品定位

**排课助手**是一款面向独立瑜伽/健身教练的**轻量级业务管理小程序**，聚焦"排课—学生—课时"三大高频场景，帮助教练从微信群+Excel的低效管理模式中解放出来。

**一句话定位：** 独立教练的掌上工作室——3步完成排课，学生扫码即约。

### 1.2 产品形态

| 端 | 形态 | 说明 |
|---|---|---|
| 教练端 | 微信小程序 | 教练日常操作主入口，排课/学生/课时/数据 |
| 学生端 | H5预约页（微信内打开） | 学生通过教练分享链接访问，无需下载App |
| 工作室管理端 | 教练端小程序内Pro模块 | 2-3人合伙工作室的多教练管理 |

### 1.3 目标用户画像

#### 画像A：李教练（独立瑜伽教练）
- **年龄：** 28岁，全职瑜伽教练
- **现状：** 带15个学生，用微信群通知上课时间，Excel记录课时
- **痛点：** 每次排课要在群里反复沟通时间；课时记录混乱，月底对账经常和学生有争议
- **期望：** 学生能自己看时间预约，课时自动扣减，自己专注教学

#### 画像B：王教练+张教练（合伙工作室）
- **规模：** 2人合伙，共30个学生
- **现状：** 各管各的学生，排课偶尔冲突，收入各算各的
- **痛点：** 缺少统一的工作室视角，无法了解整体运营情况
- **期望：** 一个工具管理两位教练的排课，能看到工作室整体收入

### 1.4 商业模式

| 版本 | 价格 | 功能范围 | 目标用户 |
|---|---|---|---|
| 免费版 | ¥0 | 1教练 + 10学生 + 基础排课/学生/课时管理 | 刚起步的独立教练 |
| Pro版 | ¥29/月 | 无限学生 + 工作室管理 + 数据导出 + 微信支付 + 独立学生小程序入口 | 成熟独立教练 / 小型工作室 |

**转化路径：** 免费版 → 学生数超10人触发升级提示 → 体验Pro功能7天 → 付费转化  
**目标转化率：** 5%-10%

### 1.5 竞品差异化

| 维度 | 健身房SaaS（勤鸟/青橙） | 通用日历工具 | **排课助手** |
|---|---|---|---|
| 目标用户 | 中大型健身房 | 个人日程管理 | **独立教练 / 小型工作室** |
| 功能复杂度 | 重（门禁/器械/会员） | 轻（无行业特性） | **轻量聚焦（排课+学生+课时）** |
| 价格 | ¥200-500/月 | 免费/低价 | **免费版 + Pro ¥29/月** |
| 学生端 | 需下载App | 无 | **H5链接，无需下载** |
| 上手时间 | 1-2周 | 即时 | **5分钟完成初始设置** |

## 3. 核心功能详细设计

### 3.1 模块一：排课管理

#### 3.1.1 周视图日历

**功能描述：** 以周一到周日的7天网格展示课程安排，是教练日常最高频使用的界面。

**页面元素：**
- 顶部：周导航栏（< 本周 > + 日期范围显示）
- 左侧：时间轴（8:00-22:00，每30分钟一格）
- 中间：7天列 × 时间格的网格
- 课程色块：显示课程类型名称 + 已约/总人数 + 地点
- 底部：+ 快速排课 浮动按钮

**交互说明：**
1. **左右滑动** 切换上一周/下一周
2. **点击课程色块** → 弹出课程详情底部弹层（显示预约学生列表、编辑/取消按钮）
3. **点击空白时间格** → 弹出快速创建课程表单（预填日期和时间）
4. **长按课程色块** → 进入拖拽模式，可调整时间（MVP后迭代）
5. **颜色规则：** 每种课程类型有独立颜色标识（教练自定义），色块高度按时长成比例

**数据规则：**
- 默认显示当周，最多可前后查看8周
- 灰色区域表示教练设定的非工作时间/休息日
- 已满课程色块右上角显示"满"角标
- 已取消课程不显示（在列表页可查看已取消记录）

**边界情况：**
- 多个课程时间重叠 → 同一天列内水平并排显示（最多3列，超出显示+N）
- 跨天课程（如23:00-01:00）→ MVP不支持，提示教练拆分为两天

#### 3.1.2 快速创建课程

**功能描述：** 教练在日历空白处点击后，快速创建一节课程。

**表单字段：**
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 课程类型 | 下拉选择 | 是 | 从已创建的课程类型中选择 |
| 日期 | 日期选择 | 是 | 默认为点击的日期 |
| 开始时间 | 时间选择 | 是 | 默认为点击的时间格，步长30分钟 |
| 时长 | 数字选择 | 是 | 默认60分钟，可选30/45/60/90/120分钟 |
| 最大人数 | 数字输入 | 是 | 默认根据课程类型（1对1=1，小班=4，团课=10） |
| 地点 | 文本输入 | 否 | 如"XX工作室"、"线上" |
| 备注 | 文本输入 | 否 | 如"请自带瑜伽垫" |

**交互说明：**
1. 提交后课程立即出现在日历上
2. 如需立即开放预约 → 勾选"同时发布到预约页"（默认勾选）
3. 创建成功 → 底部toast提示"课程已创建"，不跳转页面

**校验规则：**
- 时间不能与已有课程冲突（同教练同时段）
- 时间必须在教练设定的工作时间范围内
- 日期不能是过去的日期

#### 3.1.3 模板排课

**功能描述：** 教练设置每周固定的排课模板，一键批量生成未来4周的课程，极大减少重复排课操作。

**操作流程：**
1. 进入"模板排课"页面
2. 选择一周中每天的可用时段（如周一 10:00 1对1私教、周一 19:00 团课）
3. 预览生成的课程列表（共4周 × 每周N节）
4. 确认生成 → 批量创建课程

**冲突处理：**
- 如果模板生成的课程与已有手动排课冲突 → 跳过冲突时段，生成时提示"已跳过3月5日19:00（与已有课程冲突）"
- 已预约学生的课程不会被模板覆盖

#### 3.1.4 课程类型管理

**功能描述：** 教练创建和管理课程分类，每种类型有独立的参数设置。

**课程类型参数：**
| 参数 | 说明 | 默认值 |
|---|---|---|
| 类型名称 | 如"1对1私教"、"小班课"、"团课" | - |
| 课程时长 | 默认时长（分钟） | 60 |
| 最大人数 | 1对1=1，小班=2-4，团课=5-15 | 1 |
| 颜色标识 | 从8种预设颜色中选择 | 绿色 |
| 单价 | 单次课程价格（展示用） | - |
| 课程说明 | 展示在学生预约页的介绍文字 | - |
| 状态 | 启用/停用 | 启用 |

**预设类型（新用户引导创建）：**
1. 1对1私教课（绿色）
2. 小班课（蓝色）
3. 团课（橙色）

### 3.2 模块二：学生管理

#### 3.2.1 学生列表

**功能描述：** 展示教练所有学生的列表，支持搜索和排序。

**列表项显示：**
- 学生头像（默认首字母头像）
- 姓名
- 剩余课时（大号数字 + "次剩余"）
- 最近上课时间（如"3天前"）
- 标签（VIP / 体验课 / 需关注）

**排序方式：**
- 最近上课时间（默认，降序）
- 添加时间（降序）
- 剩余课时（升序，方便关注即将耗尽的学生）

**筛选方式：**
- 全部学生
- 按标签筛选
- 课时不足（≤2次）

**空状态：** "还没有学生，分享预约链接开始招生吧 →"（附分享按钮）

#### 3.2.2 学生档案详情

**功能描述：** 单个学生的完整信息页，是教练了解学生的核心入口。

**页面结构（从上到下）：**

**① 基本信息卡片**
- 头像 + 姓名 + 手机号（脱敏：138****1234）
- 来源渠道（微信预约/手动添加/转介绍）
- 编辑按钮

**② 身体信息卡片**
- 身高/体重/BMI
- 伤病史（红色高亮显示）
- 过敏信息
- 运动基础
- 编辑按钮

**③ 课时包状态卡片**
- 当前有效课时包名称 + 剩余次数/总次数
- 进度条可视化
- 到期时间
- 操作：手动调整课时 / 绑定新课时包

**④ 上课记录列表**
- 按时间倒序
- 每条记录：日期 + 课程类型 + 签到状态（已到/未到/已取消）
- 点击可展开查看详情

**⑤ 备注区域**
- 教练可添加多条文字备注
- 按时间倒序排列
- 如："2026-06-25 今天腰不舒服，避免前屈动作"

#### 3.2.3 学生身份识别规则

| 场景 | 处理方式 |
|---|---|
| 新手机号首次预约 | 自动创建学生档案，姓名取自预约填写，其他字段待教练补充 |
| 已有手机号再次预约 | 自动匹配已有学生档案，预约记录归入该学生 |
| 教练手动添加 | 教练填写完整信息，学生后续预约自动匹配 |
| 同一学生换手机号 | 教练在学生档案中手动修改手机号 |

### 3.3 模块三：课时管理

#### 3.3.1 课时包产品

**功能描述：** 教练创建可售卖的课时包产品模板。

**课时包类型：**

| 类型 | 参数 | 扣减规则 | 示例 |
|---|---|---|---|
| 次卡 | 总次数、有效期（天）、价格 | 每次到课扣1次 | 10次卡：30天有效，¥800 |
| 月卡 | 有效期（天）、价格 | 不限次数（可设每日上限） | 月卡：30天有效，¥1200，每日最多1次 |
| 不限次卡 | 有效期（天）、价格 | 不限次数 | 季卡：90天有效，¥3000 |

**课时包产品字段：**
| 字段 | 说明 |
|---|---|
| 产品名称 | 如"10次私教卡" |
| 类型 | 次卡/月卡/不限次卡 |
| 次数/天数 | 次卡填次数，月卡/不限次卡填有效天数 |
| 价格 | 销售价格 |
| 有效期 | 从激活日起算的有效天数 |
| 适用课程类型 | 可选限制适用哪些课程（默认全部） |
| 状态 | 启用/停用 |

#### 3.3.2 学生课时绑定

**操作流程：**
1. 教练在学生档案中点击"绑定课时包"
2. 选择课时包产品（从已启用的列表中选）
3. 填写实际收款金额和支付方式
4. 确认 → 课时包立即生效，开始倒计时

**规则：**
- 一个学生可同时持有多个课时包，优先扣减先到期的
- 激活日期 = 绑定日期（可手动修改为其他日期）
- 课时包一旦激活不可撤销（可通过手动调整课时纠正）

#### 3.3.3 课时扣减规则

| 签到状态 | 课时扣减 | 说明 |
|---|---|---|
| 已到课 | 自动扣减1次 | 签到时立即扣减 |
| 未到课（教练标记缺席） | 根据教练设置：扣减或不扣 | 默认扣减，教练可在全局设置中修改 |
| 未到课（学生提前取消） | 不扣减 | 开课前2小时以上取消 |
| 未到课（学生迟到取消） | 根据教练设置：扣减或不扣 | 开课前2小时内取消 |

#### 3.3.4 续费提醒机制

**触发条件：**
- 学生某课时包剩余 ≤ 2次 → 系统推送提醒给教练
- 学生课时包到期前3天 → 系统推送提醒给教练
- 学生课时包耗尽 → 学生端预约功能冻结，教练收到"需跟进"提醒

**提醒渠道：**
- 教练端小程序内通知（默认开启）
- 微信服务号推送（需教练授权）
- 短信（备用通道，Pro版）

#### 3.3.5 收款记录

**每条收款记录字段：**
| 字段 | 说明 |
|---|---|
| 学生姓名 | 付款学生 |
| 课时包名称 | 购买的课时包 |
| 金额 | 实际收款金额 |
| 支付方式 | 微信/支付宝/现金/其他 |
| 收款日期 | 默认当天，可修改 |
| 备注 | 选填 |

**统计维度：**
- 本月收入总额
- 本周收入总额
- 各支付方式占比
- 新购课时包数量

### 3.4 模块四：预约管理

#### 3.4.1 今日课程列表

**功能描述：** 教练打开小程序首页，默认展示今日所有课程。

**课程卡片内容：**
- 时间（如 09:00-10:00）
- 课程类型名称 + 颜色标识
- 已预约/最大人数（如 2/4）
- 学生头像列表（最多显示3个头像 + " +1"）
- 地点
- 操作按钮：[签到] [详情]

**卡片状态：**
- 待上课（未到开始时间）→ 正常显示
- 进行中（到达开始时间）→ 顶部高亮"进行中"
- 已完成（已签到）→ 灰色，显示"已完成"
- 已取消 → 不显示在今日列表中

#### 3.4.2 签到管理

**功能描述：** 教练在课程开始时确认到场学生，触发课时扣减。

**签到方式：**
1. **逐个签到：** 点击学生名字旁的签到按钮
2. **批量签到：** 点击"全部签到"按钮，一键签到所有已预约学生
3. **标记缺席：** 点击学生名字旁的"缺席"按钮

**签到后自动操作：**
- 扣减到场学生课时（根据扣减规则）
- 更新学生上课记录
- 检查课时余额，触发续费提醒（如需要）

#### 3.4.3 预约变更通知

| 变更类型 | 通知内容 | 通知对象 |
|---|---|---|
| 新建课程并发布 | "教练发布了新课程：X月X日 XX课程" | 所有学生 |
| 学生预约成功 | "新预约：XXX预约了X月X日 XX课程" | 教练 |
| 学生取消预约 | "XXX取消了X月X日 XX课程的预约" | 教练 |
| 教练修改课程 | "课程变更：XX课程时间/地点已调整" | 已预约学生 |
| 教练取消课程 | "课程取消：X月X日 XX课程已取消，课时已返还" | 已预约学生 |

- 课程类型列表：每种课程显示名称 + 时长 + 价格 + "预约"按钮
- 底部：浮动"立即预约"按钮

#### 3.5.2 预约日历页

**页面结构：**
- 顶部：日期横向滑动条（显示未来14天，周几+日期）
- 中间：选中日期的可用时段列表
  - 每个时段卡片：时间 + 课程类型 + 剩余名额 + "预约"按钮
  - 已满时段：灰显 + "已满"标签
  - 非工作时段：不显示

**交互说明：**
1. 左右滑动日期条，时段列表联动更新
2. 点击"预约" → 弹出预约表单底部弹层

#### 3.5.3 预约成功页

#### 3.5.4 取消预约

## 7. 非功能需求（产品视角）

- 预约提交 → 结果反馈 ≤ 1秒
- 日历周切换 ≤ 500ms
- 支持500教练同时在线

- 手机号脱敏显示（138****1234）

- HTTPS全站加密
- 符合《个人信息保护法》

### 7.3 兼容性要求
- 教练端：iOS 12+ / Android 7.0+ 微信小程序
- 学生端：微信内置浏览器（iOS Safari / Android WebView）
- 屏幕适配：320px - 428px宽度

### 7.4 可用性要求
- 服务可用性 ≥ 99.5%
- 数据每日自动备份
- 推送通知到达 ≤ 30秒

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 教练不愿从微信迁移 | 产品无人使用 | 提供"微信一键导入排课"功能；5分钟引导流程降低迁移成本 |
| 学生不愿通过链接预约 | 教练觉得没用 | 预约页设计极致简洁，3步完成；支持教练代预约作为过渡 |
| 课时记录争议 | 用户信任问题 | 所有课时变动有完整日志；学生端实时可查 |
| 免费版用户不转化 | 无法盈利 | 学生数>10触发升级提示；Pro版提供7天免费试用 |
| 微信小程序审核不通过 | 上线延迟 | 提前了解审核规则；预留3天审核缓冲期 |
