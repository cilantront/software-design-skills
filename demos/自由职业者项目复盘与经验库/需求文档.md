# 自由职业者项目复盘与经验库 - 用户需求文档

> 文档版本：v1.0
> 创建日期：2026-06-29
> 文档状态：初稿
> 适用产品：自由职业者项目复盘与经验库（以下简称"复盘经验库"）

---

## 1. 文档概述

### 1.1 产品愿景

为自由职业者提供一款聚焦"项目结束后复盘与经验沉淀"的轻量级效率工具，帮助用户系统性回顾每个已完成项目的工时、成本、客户满意度和经验教训，自动生成盈利能力分析报告，并构建可检索的经验库，为未来报价和接单提供数据化决策依据。

### 1.2 文档目的

本文档定义"复盘经验库"产品的用户需求，作为后续产品设计、UI 原型、开发与测试的核心依据。文档聚焦于 **需求层面**，不涉及具体技术实现方案。

### 1.3 适用范围

本文档适用于产品、设计、开发、测试团队，以及参与 MVP 评审的相关方。

### 1.4 术语表

| 术语 | 定义 |
|------|------|
| 项目复盘 | 项目结束后，对项目的工时、成本、质量、经验教训进行系统性回顾和记录的过程 |
| 预估工时 | 项目启动前，用户基于经验和客户需求预估的项目所需工作时间 |
| 实际工时 | 项目执行过程中，用户实际投入的工作时间 |
| 利润率 | （项目收入 - 项目成本）/ 项目收入 × 100% |
| 时薪 | 项目净收入 / 实际工时，反映用户的时间价值 |
| 经验条目 | 从项目复盘中提炼的可复用知识，包括问题描述、解决方案、适用场景等 |
| 经验库 | 按客户类型、项目类型、行业等维度组织的可检索经验条目集合 |
| 客户价值 | 综合考量项目收入、利润率、客户满意度、复购率等维度的客户质量评估 |

## 2. 目标用户与使用场景

### 2.1 目标用户画像

#### 画像 A：独立设计师
- **典型特征**：28-35 岁，承接品牌设计、UI 设计、插画等外包项目
- **工作模式**：同时服务 3-5 个客户，项目周期从 1 周到 2 个月不等
- **痛点**：报价时凭感觉定价，做完才发现某些项目根本不赚钱；同样的设计问题反复踩坑
- **核心诉求**：知道哪些项目真正赚钱、哪些客户值得长期维护，下次报价时有数据参考

#### 画像 B：自由程序员
- **典型特征**：30-40 岁，承接网站开发、小程序开发、系统集成等技术项目
- **工作模式**：项目制工作，单个项目周期 1-6 个月，报价按项目整体或按人天
- **痛点**：总是低估开发难度导致工时超标；项目结束后缺乏总结，相同技术坑反复踩
- **核心诉求**：准确评估项目工时和成本，积累技术经验避免重复踩坑

#### 画像 C：独立咨询师
- **典型特征**：35-50 岁，提供管理咨询、IT 咨询、财务咨询等专业服务
- **工作模式**：按项目或按小时收费，客户以中小企业为主
- **痛点**：无法量化每个咨询项目的价值；难以向新客户证明自己的专业能力
- **核心诉求**：建立客户价值档案，用历史数据支撑报价和客户选择

#### 画像 D：自由撰稿人/内容创作者
- **典型特征**：25-40 岁，为媒体、品牌、公众号提供内容创作服务
- **工作模式**：按篇/按字数/按月结算，同时服务多个客户
- **痛点**：不清楚自己的真实时薪；难以判断哪类内容创作性价比高
- **核心诉求**：了解自己的时间价值，优化接单策略

#### 画像 E：小型外包团队负责人
- **典型特征**：3-5 人团队，承接设计、开发、运营等综合性外包项目
- **工作模式**：团队协作，项目周期较长，涉及多人分工
- **痛点**：团队成员缺乏复盘习惯，经验无法沉淀和共享；报价缺乏团队层面的数据支撑
- **核心诉求**：建立团队复盘机制，共享项目经验，提升整体报价能力

### 2.2 典型使用场景

#### 场景 1：项目结束后的快速复盘
> 设计师小李刚完成一个品牌设计项目，客户已付款。她打开复盘经验库，选择"品牌设计"项目模板，填入预估工时 40 小时、实际工时 52 小时、预算 8000 元、实际成本 9500 元（含购买素材和修改返工），客户满意度 4 星。系统自动生成分析报告：时薪 182 元（低于她设定的 250 元目标），利润率 -18.75%（亏损）。系统标记这是一个"需要深度复盘"的项目，提示她记录"客户需求反复变更导致返工"的教训。

#### 场景 2：报价前的经验检索
> 程序员老王收到一个小程序开发需求，报价前他打开复盘经验库，按"项目类型=小程序开发"、"行业=电商"筛选，找到 3 个类似项目的复盘记录。他发现这类项目平均实际工时是预估的 1.5 倍，主要坑在"支付接口对接"和"审核流程"。他据此调整报价，将工时预估上调 50%，并在合同中明确支付接口对接的额外收费条款。

#### 场景 3：客户价值评估
> 咨询师张老师年底盘点客户，打开客户价值排名报告。系统显示：客户 A 贡献了最高利润（利润率 65%），但满意度只有 3 星（沟通困难）；客户 B 利润率一般（40%），但满意度 5 星且复购 3 次。她据此调整策略：对客户 A 提高报价以补偿沟通成本，对客户 B 提供长期合作优惠以维护关系。

#### 场景 4：团队经验共享
> 外包团队负责人小王让团队成员每完成一个项目都填写复盘表。新成员接手类似项目时，先在经验库中搜索前人踩过的坑和解决方案，避免重复犯错。团队周会上，大家基于复盘数据进行讨论，持续优化报价模板和项目流程。

## 3. 需求概述

### 3.1 产品定位

**不做项目过程管理**，而是聚焦于"项目结束后的复盘 + 经验沉淀 + 辅助未来决策"。核心差异化：
- **聚焦复盘**：不介入项目执行过程，只在项目结束后介入
- **数据驱动**：基于工时、成本、满意度等量化数据生成分析报告
- **经验复用**：将复盘经验结构化沉淀，支持未来检索和参考
- **自由职业者专属**：针对单人/小团队的工作模式设计，非通用项目管理工具

## 4. 功能需求

### 4.1 项目复盘管理

#### 4.1.1 需求描述

用户能够在项目结束后，通过填写结构化复盘模板，记录项目的工时、成本、客户满意度和经验教训，形成完整的项目复盘档案。

#### 4.1.2 功能要点

| 编号 | 功能点 | 描述 |
|------|--------|------|
| F-1.1 | 项目基本信息登记 | 创建复盘记录时填写：项目名称、客户名称、客户类型（企业/个人/政府）、项目类型（设计/开发/咨询/写作等）、所属行业、项目起止时间、合同金额 |
| F-1.2 | 工时对比填写 | 分别填写预估工时和实际工时（单位：小时），系统自动计算工时偏差率 = (实际工时 - 预估工时) / 预估工时 × 100% |
| F-1.3 | 成本对比填写 | 分别填写预算和实际成本，实际成本包括：时间成本（实际工时 × 期望时薪）、直接成本（素材购买、差旅等）、间接成本（沟通成本、修改返工等） |
| F-1.4 | 客户满意度评分 | 用户主观评分（1-5 星），可附加客户反馈备注；如有客户直接反馈，可上传截图或文字记录 |
| F-1.5 | 问题与解决方案记录 | 记录项目中遇到的主要问题（至少 1 个，最多 10 个），每个问题包括：问题描述、影响程度（高/中/低）、解决方案、是否可复用 |
| F-1.6 | 复盘模板 | 系统预置基础复盘模板，专业版支持自定义模板字段和 industry-specific 模板 |
| F-1.7 | 复盘状态管理 | 复盘状态流转：待复盘 → 已复盘 → 需深度复盘（系统根据利润率、工时偏差等指标自动建议） |
| F-1.8 | 复盘草稿保存 | 支持分多次填写，自动保存草稿，避免数据丢失 |
| F-1.9 | 历史复盘查看 | 支持查看所有历史复盘记录，按时间、客户、项目类型筛选 |

#### 4.1.3 验收标准

- 复盘表单填写流畅，无卡顿
- 工时偏差率、成本偏差率计算准确
- 草稿自动保存间隔 ≤ 30 秒
- 支持至少 5 种项目类型的预置模板

### 4.2 盈利分析报告

#### 4.2.1 需求描述

系统基于用户填写的复盘数据，自动生成项目盈利能力分析报告，帮助用户量化项目的真实收益，识别高价值客户和低效项目。

#### 4.2.2 功能要点

| 编号 | 功能点 | 描述 |
|------|--------|------|
| F-2.1 | 时薪计算 | 自动计算实际时薪 = (合同金额 - 实际成本) / 实际工时，与用户设定的期望时薪对比 |
| F-2.2 | 利润率计算 | 自动计算利润率 = (合同金额 - 实际成本) / 合同金额 × 100% |
| F-2.3 | 客户价值排名 | 综合考量：累计收入、平均利润率、客户满意度、复购次数，生成客户价值排名（Top 10） |
| F-2.4 | 项目类型盈利对比 | 按项目类型统计平均时薪、利润率，帮助用户识别高性价比项目类型 |
| F-2.5 | "值得复盘"标记 | 系统自动标记以下情况的项目：利润率 < 0%（亏损）、工时偏差率 > 30%（严重超支）、客户满意度 ≤ 2 星（不满意），提示用户深度复盘 |
| F-2.6 | 趋势分析（专业版） | 展示用户时薪、利润率随时间的变化趋势，帮助用户评估自身成长 |
| F-2.7 | 报告生成 | 自动生成单个项目的复盘报告，包含：项目概况、工时对比、成本对比、满意度、问题清单、经验教训 |
| F-2.8 | 报告导出（专业版） | 支持将复盘报告导出为 PDF（用于存档）和 Excel（用于进一步分析） |

#### 4.2.3 核心指标计算规则

**时薪计算**：
```
实际时薪 = (合同金额 - 实际成本) / 实际工时
期望时薪 = 用户设定的目标时薪（首次使用时引导用户填写）
时薪达成率 = 实际时薪 / 期望时薪 × 100%
```

**利润率计算**：
```
实际成本 = 时间成本 + 直接成本 + 间接成本
时间成本 = 实际工时 × 期望时薪
利润率 = (合同金额 - 实际成本) / 合同金额 × 100%
```

**客户价值评分**（专业版）：
```
客户价值分 = 累计收入权重 × 0.4 + 平均利润率权重 × 0.3 + 客户满意度权重 × 0.2 + 复购率权重 × 0.1
```

#### 4.2.4 验收标准

- 时薪、利润率计算准确率 100%
- 客户价值排名实时更新（新增复盘记录后自动重算）
- "值得复盘"标记触发准确，无遗漏
- 报告生成时间 ≤ 3 秒

### 4.3 经验库管理

#### 4.3.1 需求描述

用户可以将项目复盘中提炼的经验教训整理为结构化的经验条目，按客户类型、项目类型、行业等维度分类，支持关键词搜索和全文检索，便于未来报价和接单时快速参考。

#### 4.3.2 功能要点

| 编号 | 功能点 | 描述 |
|------|--------|------|
| F-3.1 | 经验条目创建 | 从复盘记录一键生成经验条目草稿，用户可编辑完善；也可手动创建独立经验条目 |
| F-3.2 | 经验条目结构 | 每个经验条目包括：标题、问题描述、解决方案、适用场景、标签、关联项目复盘（可选） |
| F-3.3 | 分类体系 | 支持按客户类型（企业/个人/政府）、项目类型（设计/开发/咨询/写作等）、行业（互联网/教育/金融/制造等）三级分类 |
| F-3.4 | 标签系统（专业版） | 用户可为经验条目添加自定义标签（如"支付接口"、"需求变更"、"跨部门协作"），便于多维度检索 |
| F-3.5 | 关键词搜索 | 免费版支持基于标题、标签的关键词搜索 |
| F-3.6 | 全文检索（专业版） | 支持对经验条目的全部内容（问题描述、解决方案等）进行全文检索 |
| F-3.7 | 经验条目关联 | 经验条目可关联到一个或多个项目复盘记录，便于追溯经验来源 |
| F-3.8 | 经验条目编辑与删除 | 支持对已有经验条目进行编辑和删除，编辑历史记录保留（专业版） |
| F-3.9 | 经验条目导出（专业版） | 支持将经验库导出为 PDF 或 Markdown 格式，便于离线查阅和分享 |

#### 4.3.3 验收标准

- 关键词搜索响应时间 ≤ 1 秒
- 全文检索响应时间 ≤ 2 秒（专业版）
- 经验条目创建流程 ≤ 3 步完成
- 分类体系支持至少 3 级嵌套

### 4.4 简单看板

#### 4.4.1 需求描述

提供一个简洁的看板界面，让用户快速了解项目复盘的整体状态和关键指标。

#### 4.4.2 功能要点

| 编号 | 功能点 | 描述 |
|------|--------|------|
| F-4.1 | 复盘状态概览 | 展示：待复盘项目数、已复盘项目数、需深度复盘项目数 |
| F-4.2 | 关键指标卡片 | 展示：本月复盘项目数、平均时薪、平均利润率、客户满意度均值 |
| F-4.3 | 近期复盘列表 | 展示最近 10 个复盘记录，包括：项目名称、客户名称、复盘时间、利润率、状态 |
| F-4.4 | 快速入口 | 提供"新建复盘""查看经验库""生成报告"的快速入口 |
| F-4.5 | 筛选与排序 | 支持按客户、项目类型、时间范围筛选，按利润率、时薪、时间排序 |

#### 4.4.3 验收标准

- 看板加载时间 ≤ 2 秒
- 关键指标实时更新（新增复盘记录后自动刷新）
- 看板界面简洁直观，无需培训即可上手

## 6. 业务规则

### 6.1 复盘触发规则

1. 项目状态标记为"已完成"后，系统自动提醒用户进行复盘
2. 超过 7 天未复盘的项目，系统发送提醒通知
3. 系统根据以下规则自动建议"需深度复盘"：
   - 利润率 < 0%（亏损项目）
   - 工时偏差率 > 30%（严重超支）
   - 客户满意度 ≤ 2 星（不满意）
   - 出现 3 个以上"高影响"问题

### 6.2 成本计算规则

1. **时间成本** = 实际工时 × 用户设定的期望时薪
2. **直接成本** = 用户填写的素材购买、差旅等直接支出
3. **间接成本** = 用户填写的沟通成本、修改返工等间接支出（按工时估算）
4. **实际成本** = 时间成本 + 直接成本 + 间接成本
5. **利润率** = (合同金额 - 实际成本) / 合同金额 × 100%

### 6.3 版本限制规则

1. 免费版最多记录 10 个项目，超出后无法新建复盘（需升级专业版）
2. 免费版使用基础复盘模板，不支持自定义模板
3. 免费版不支持数据导出、云端备份、团队协作功能
4. 免费版历史数据保留 6 个月，超出部分无法查看（但不会删除）
5. 专业版按年订阅享受 8 折优惠（¥182/年，相当于 ¥15.2/月）

## 8. 约束与假设

### 8.1 约束

1. **MVP 开发周期**：约 7 天，需优先实现核心功能（复盘表单 + 成本计算 + 简单看板 + 经验条目管理）
2. **技术栈限制**：前端采用 Web 技术栈，后端采用轻量级框架，确保快速迭代
3. **功能边界**：不做项目过程管理（如任务分配、进度跟踪），只做项目结束后的复盘和经验沉淀
4. **数据录入**：所有数据均由用户手动填写，不支持从外部系统自动导入

### 8.2 假设

1. 用户有项目复盘的意愿，能够坚持在每个项目结束后填写复盘表
2. 用户填写的数据基本准确，能够真实反映项目的工时、成本和满意度
3. 用户具备一定的财务基础知识，能够理解时薪、利润率等概念
4. MVP 阶段以个人用户为主，团队协作场景在后续版本迭代
