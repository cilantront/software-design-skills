## 1. 产品概述

### 1.1 产品定位

**AI截图知识整理助手（SnapMind）** 是一款面向个人用户的**AI驱动型截图智能整理工具**，聚焦"导入→OCR识别→AI分类→语义搜索"核心链路，帮助用户将堆积在手机/电脑相册中无法检索的截图转化为可按语义检索、可分类利用的结构化知识资产。

**一句话定位：** 你的AI截图管家——截图不遗忘，信息随手找。

### 1.2 产品理念

**"截图≠遗忘"**：大多数截图工具只做存储，用户依然要面对"翻相册找截图"的困境。SnapMind 将OCR识别与AI语义分类作为第一原则——用户只需完成"导入截图"这一个动作，后续的文字提取、类型识别、主题归类、标签生成、索引构建全部由AI自动完成，让每一张截图都能被找到、被利用、被沉淀。

### 1.3 产品形态

| 端 | 形态 | 说明 |
|---|---|---|
| 移动端APP | iOS + Android 客户端 | 核心使用入口，含截图导入、OCR识别、AI分类、搜索、浏览全模块 |
| 运营管理后台 | Web端管理控制台 | 用户管理、会员管理、运营数据统计、系统配置 |

### 1.4 目标用户画像

#### 画像A：小赵（知识工作者）
- **年龄：** 28岁，互联网公司运营
- **现状：** 每天手机截图20-40张，包括行业报告数据、竞品页面、微信群工作通知、商品价格对比
- **痛点：** 一周后想找某张竞品的价格截图，翻遍相册也找不到；收藏的Excel数据截图在哪个群里更是石沉大海
- **期望：** 导入截图后AI自动分类，搜"上周看到的XX竞品价格"就能秒定位

#### 画像B：小李（学生）
- **年龄：** 22岁，研究生
- **现状：** 看论文、查资料时习惯截图关键段落、公式、数据图表，截图数百张堆积在相册
- **痛点：** 写论文时想找之前截图的某个数据，需要一张张翻，效率极低
- **期望：** 截图按"论文A"/"论文B"/"方法论"自动归类，搜关键词或描述就能找到

#### 画像C：老王（程序员）
- **年龄：** 32岁，后端工程师
- **现状：** 经常在手机上看到技术文章、代码片段、报错信息就截图保存
- **痛点：** 代码截图混杂在聊天记录、商品截图中，需要时根本找不到
- **期望：** AI能识别出"代码截图"、"报错信息"等类型，搜索"那段Python异步代码"直接出结果

## 3. 核心功能详细设计

### 3.1 模块一：截图导入

#### 3.1.1 相册批量导入

**功能描述：** 用户从手机相册中批量选择截图，一键导入到SnapMind进行AI处理。

**页面元素：**
- 导入入口：首页顶部悬浮"+"按钮 → 点击弹出导入方式选择
- 相册选择页：
  - 顶部筛选栏：[全部截图] [最近7天] [最近30天] [全部]
  - 截图瀑布流展示：自动过滤出图片中的截图（按尺寸/特征识别）
  - 每张截图右上角复选框，支持全选/反选
  - 底部操作栏：已选N张 / [取消] / [开始导入]

**交互说明：**
1. 进入相册选择页 → 默认展示最近30天的截图
2. 用户点击选择截图 → 复选框打钩，底部计数更新
3. 点击"开始导入" → 跳转导入处理进度页
4. 导入完成 → Toast提示"✓ 已导入N张截图，AI处理完成"，自动返回首页

**数据规则：**
- 单次批量导入上限：100张（免费版20张）
- 单张图片上限：20MB
- 支持格式：JPG/PNG/HEIC/WebP
- 自动检测重复截图（基于图片哈希），提示"该截图已于X日导入"

**边界情况：**
- 相册无截图 → 显示空状态引导"去截图试试"
- 单次超过100张 → 提示"单次最多导入100张，请分批导入"
- 网络中断 → 已上传的继续处理，未上传的暂停，恢复后自动续传
- 重复截图 → 弹窗"检测到N张重复截图，是否跳过？"

#### 3.1.2 拍照导入

**功能描述：** 用户直接拍摄屏幕、纸质文档、书籍等，将拍摄内容作为截图导入。

**页面元素：**
- 相机取景框（全屏）
- 顶部工具栏：[闪光灯] [HDR] [网格线]
- 底部操作区：[拍照按钮] [相册导入] [取消]
- 拍照后预览页：[重拍] [确认导入]

**交互说明：**
1. 点击"拍照导入" → 调起相机
2. 用户对准屏幕/文档拍照 → 点击拍照按钮
3. 预览拍摄结果 → 点击"确认导入" → 进入AI处理流程
4. 支持连续拍摄模式（拍完一张自动进入下一张）

#### 3.1.3 系统分享导入

**功能描述：** 用户在任意APP（微信/QQ/浏览器/微博等）中，通过系统"分享"功能直接将截图发送到SnapMind。

**iOS实现：**
- 通过Share Extension接入系统分享菜单
- 用户点击"分享到SnapMind" → 弹出简化确认界面 → 自动导入

**Android实现：**
- 通过Share Intent接收系统分享
- 用户在任意APP点击"分享" → 选择SnapMind → 自动导入

**交互说明：**
- 分享后APP无需打开，后台静默处理
- 打开APP后可在首页看到刚导入的截图及AI处理结果
- 支持批量分享（多张图片同时分享）

#### 3.1.4 导入管理

**功能描述：** 清晰展示批量导入的进度、状态和结果。

**进度页页面元素：**
- 顶部：总进度条（已处理X张/共Y张）+ 预估剩余时间
- 列表区：每张截图的处理状态
  - 上传中（蓝色进度环）
  - OCR识别中（蓝色脉动）
  - AI分类中（紫色脉动）
  - 已完成（绿色对勾）
  - 失败（红色叹号）→ 点击弹出失败原因 + [重试]按钮
- 底部操作：[后台处理] [取消导入]

**数据规则：**
- 单张截图完整处理时间：≤ 10秒（OCR + AI分类）
- 批量100张整体处理时间：≤ 10分钟
- 支持后台处理：用户可切换到其他页面，处理完成后推送通知
- 失败自动重试：OCR失败重试2次，AI分类失败重试2次，仍失败标记为失败

### 3.2 模块二：OCR文字识别

#### 3.2.1 截图文本提取

**功能描述：** 对导入的截图进行OCR识别，提取图中所有文字内容，作为后续AI分类和搜索的基础。

**处理流程：**
```
截图上传 → 图片预处理（旋转校正/去噪/增强） → OCR文字识别 → 文本结构化 → 存储
```

**多场景识别能力：**

| 场景 | 识别特点 | 输出结构化 |
|---|---|---|
| 聊天截图 | 识别对话双方、时间戳 | 对话文本 + 时间 + 发送者 |
| 网页截图 | 识别标题/正文/链接 | 标题 + 正文 + URL列表 |
| 商品页截图 | 识别商品名/价格/规格 | 商品名 + 价格 + 关键属性 |
| 代码截图 | 识别代码语法、语言类型 | 代码文本 + 编程语言标识 |
| 发票/票据 | 识别金额/日期/商家 | 金额 + 日期 + 商家名 |
| 地址信息 | 识别地址/电话/地图 | 地址文本 + 电话 + 地理信息 |
| 文章/笔记 | 识别标题/段落/列表 | 标题 + 正文结构化 |

**数据规则：**
- 单张截图OCR识别响应时间：≤ 3秒
- 识别准确率：印刷体 ≥ 98%，手写体 ≥ 85%
- 支持语言：中文（简/繁）、英文、中英混合
- 识别结果文本大小上限：50KB/张

**边界情况：**
- 截图模糊/分辨率过低 → 提示"截图清晰度较低，识别结果可能不准确"
- 截图为纯图片无文字 → 标记为"无文字截图"，仅依靠AI图像分类
- 识别失败 → 自动重试2次，仍失败标记为"识别失败"，支持用户手动重试

#### 3.2.2 识别结果校对

**功能描述：** 用户可手动校对和修正OCR识别结果，提升数据准确性。

**页面元素：**
- 截图详情页 → OCR文本区域
- 顶部：识别准确率预估（如"识别置信度 96%"）
- 文本区：OCR识别结果（可编辑）
- 低置信度字/词：红色下划线标记
- 底部：[保存修改] [重置为原识别结果]

**交互说明：**
1. 用户点击OCR文本区域 → 进入编辑模式
2. 修改识别错误 → 点击"保存"
3. 保存后 → 重新建立搜索索引（基于修正后的文本）
4. 校对后的文本标记为"已人工校对"

### 3.3 模块三：AI智能分类

#### 3.3.1 AI分类流水线

**功能描述：** 截图完成OCR识别后，自动进入AI分类流水线，完成类型识别、主题归类、标签生成三个步骤。

**处理流程（异步执行）：**

```
OCR文本 + 图像特征 → 类型识别 → 主题归类 → 标签生成 → 写入索引
```

**步骤详解：**

| 步骤 | 输入 | 输出 | 技术实现 | 耗时 |
|---|---|---|---|---|
| 1. 类型识别 | 图片+OCR文本 | 截图类型 | 多模态LLM分类 | 1-2秒 |
| 2. 主题归类 | OCR文本+类型 | 主题分类 | LLM主题分类 | 0.5-1秒 |
| 3. 标签生成 | OCR文本+类型+主题 | 3-8个标签 | LLM关键词提取 | 0.5-1秒 |

**总处理时间：** 单张截图AI分类 ≤ 2秒

**截图类型体系（默认）：**
- 聊天截图（微信/QQ/短信/邮件）
- 商品页（电商/外卖/团购）
- 代码片段（各编程语言）
- 文章/资讯（新闻/公众号/博客）
- 地址/导航（地图/位置信息）
- 价格/金额（支付/账单/报价）
- 票据（发票/收据/小票）
- 证件/证明（身份证/驾照/证书）
- 社交内容（朋友圈/微博/小红书）
- 其他

**主题分类体系（默认）：**
- 工作（会议/项目/协作/汇报）
- 学习（课程/论文/笔记/考试）
- 购物（商品/价格/订单/优惠）
- 生活（出行/餐饮/健康/娱乐）
- 社交（聊天/朋友圈/群组）
- 财务（账单/支付/理财/发票）
- 技术（代码/文档/报错/方案）
- 其他

用户可自定义类型和主题，AI结果可被手动覆盖。

#### 3.3.2 自动打标签

**功能描述：** AI根据截图内容自动生成3-8个标签，便于后续筛选和检索。

**标签生成规则：**
- 从OCR文本中提取关键词（人名、地名、商品名、数字等实体）
- 结合截图类型生成场景标签（如"Python代码"、"外卖订单"）
- 结合主题生成领域标签（如"工作"、"学习"）
- 每张截图最多8个标签，按相关性排序

**标签展示：**
- 截图列表卡片：显示前2个核心标签
- 截图详情页：显示全部标签 + [编辑]入口
- 分类浏览页：标签云展示，可按标签筛选

#### 3.3.3 手动标签管理

**功能描述：** 用户可对AI生成的标签进行编辑，补充、修改、删除。

**操作方式：**
- 添加标签：点击"+" → 从已有标签选择 / 新建标签
- 删除标签：点击标签右侧×号
- 修改标签：长按标签 → 重命名（所有使用该标签的截图同步更新）
- 合并标签：选择2个以上相似标签 → 合并为一个

**数据规则：**
- 用户手动标签优先级高于AI标签
- 修改标签后 → 重新建立搜索索引
- 全局标签数量上限：1000个（超过提示清理低频标签）

### 3.4 模块四：智能搜索

#### 3.4.1 自然语言搜索

**功能描述：** 用户输入自然语言描述，系统从所有截图的OCR文本、标签、分类中检索匹配的截图，不依赖精确关键词。

**搜索入口：**
- 底部TabBar "搜索" 页面
- 首页顶部搜索图标（快捷入口）
- 分类页顶部搜索框

**搜索流程：**
1. 用户输入查询（如"上周看到的XX商品价格"、"那段Python异步代码"）
2. 查询文本语义向量化（输入停顿300ms后自动触发）
3. 向量数据库检索Top-50相似截图
4. 综合相关性、时间、分类匹配度排序
5. 展示搜索结果列表，高亮关键匹配信息

**搜索结果展示：**
- 每条结果卡片：截图缩略图 / 截图类型角标 / 匹配片段（高亮关键词）/ AI标签 / 时间
- 匹配片段：从OCR文本中截取与查询最相关的100字片段
- 高亮显示：查询中的关键词在匹配片段中以黄色高亮

**搜索筛选：**
- 范围筛选：全部 / 指定类型 / 指定主题 / 指定标签
- 时间筛选：全部 / 最近7天 / 最近30天 / 自定义时间段
- 排序方式：相关度（默认）/ 时间（最新）/ 收藏优先

**响应时间：** ≤ 2秒

#### 3.4.2 关键词搜索

**功能描述：** 支持传统的OCR文本关键词精确匹配搜索。

**使用场景：**
- 用户记得截图中的某个具体关键词（人名、商品名、代码变量名等）
- 关键词搜索在免费版也可使用

**交互说明：**
- 输入关键词 → 实时显示匹配数量
- 支持多关键词（空格分隔，AND逻辑）
- 支持精确匹配（双引号包裹）

#### 3.4.3 搜索历史

**功能描述：** 记录用户最近30条搜索历史，便于快速重复搜索。

**页面元素：**
- 搜索页默认展示：
  - 搜索框（自动聚焦）
  - 搜索历史列表（最近30条，按时间倒序）
  - 热门/推荐搜索（基于用户常用类型和标签）
- 每条历史：查询文本 + 搜索时间 + [删除]
- 顶部：[清空历史] 按钮

**交互说明：**
- 点击历史项 → 直接执行搜索
- 长按历史项 → 删除该条
- 搜索输入框聚焦 → 自动展示搜索历史

#### 3.5.3 收藏管理

#### 3.5.4 截图详情页

### 6.2 安全与隐私

| 项目 | 要求 |
|---|---|
| 数据传输 | 全站HTTPS |
| 数据存储 | 用户截图加密存储（AES-256） |
| 隐私保护 | 用户截图不用于模型训练，不向第三方共享 |
| OCR后处理 | 用户可选择仅保留文本索引，删除原图 |
| 认证 | JWT令牌 + Refresh Token，Token过期时间2小时 |
| 权限控制 | 按订阅版本控制功能权限 |
| 内容安全 | OCR识别内容经过内容安全检测，过滤违规内容 |
| 备份 | 每日自动备份，保留30天 |

### 6.3 兼容性

| 端 | 要求 |
|---|---|
| iOS | iOS 14.0 及以上 |
| Android | Android 8.0 及以上 |
| 网络 | 4G/5G/WiFi，弱网支持断点续传 |
| 后台服务 | 需要后端支撑OCR、AI分类、搜索索引 |

### 6.4 可用性

- 系统可用性 ≥ 99.5%
- OCR/AI处理失败自动重试3次，仍失败标记为"处理失败"由用户手动重试
- 关键操作（导入/搜索/删除）需提供loading状态和成功/失败反馈
- 批量导入支持后台处理 + 完成推送通知
- 空状态设计：首次导入引导、无搜索结果引导

### 7.2 AI模型选型

| 能力 | 推荐方案 | 备选方案 |
|---|---|---|
| OCR识别 | 百度OCR / 阿里云OCR | 开源：PaddleOCR |

| 主题归类+标签生成 | LLM（GPT-4o-mini / 通义千问） | 规则+TF-IDF |
| 语义搜索嵌入 | OpenAI text-embedding-3-small | 开源：BGE-M3 |
| 搜索引擎 | MeiliSearch | Elasticsearch |

**模型抽象层设计：** 所有AI调用通过统一接口，底层模型可替换，便于切换供应商。
