# 1. 文档概述

## 1.1 文档目的

本文档是「宠物店会员与寄养周期管理助手」MVP 版本的产品设计文档，在需求文档（URS v1.1）基础上，进一步细化功能规格、信息架构、页面流程、交互说明和数据字典，作为研发团队的开发依据。

## 1.2 MVP 范围

**MVP 目标**：7-10 天内交付最小可交付集合，覆盖单店核心业务闭环。

**MVP 包含的功能模块**：
1. 会员/宠物主人端（小程序）：微信授权登录、宠物档案管理、消费记录查看、会员等级查看（含降级预警）、洗护提醒订阅与接收、寄养预约与状态查看、寄养日志查看、费用明细
2. 店员端（Web）：工作台首页、会员管理、宠物档案管理、消费录入与查询、洗护排期管理、寄养管理（排期/审核/日志/结算）
3. 店长端（Web）：经营数据看板、门店设置（含节假日三级加价配置）、会员等级管理（含降级规则与预警配置）、员工管理

**MVP 不包含**（V1.1 迭代）：
- 短信提醒通道
- 分级降级周期（按等级设置不同容忍期）
- 多店版功能
- 在线微信支付（可延后至 MVP 后期或 V1.1）
- 会员标签、权益配置等非核心功能

## 1.3 关键决策

| 决策事项 | 最终方案 |
|---------|---------|
| 会员降级 | 固定 180 天无消费降级 + 降级前 30 天预警通知（MVP） |
| 寄养加价 | 三级日期类型：普通日 1.0x / 周末 1.2x / 法定假日 1.5x，店长可自定义 |
| 消息通道 | MVP 仅微信订阅消息，一次性订阅需强引导 |
| 节假日日历 | 支持每年更新（适配国务院年度法定假日安排） |

## 1.4 术语表

| 术语 | 说明 |
|-----|------|
| 洗护周期 | 两只洗护之间的间隔天数，默认狗 45 天、猫 60 天，门店可自定义 |
| 寄养档期 | 一个笼位/房间在某段时间内的占用状态 |
| 三级日期类型 | 普通日（系数 1.0）、周末（系数 1.2）、法定假日（系数 1.5） |
| 一次性订阅 | 微信订阅消息机制，用户每次点击"允许"后开发者获得一次发送机会 |
| 降级预警 | 连续无消费达到 150 天（降级前 30 天）时推送的预警通知 |

# 3. 功能规格详述

## 3.1 小程序端功能规格

### 3.1.1 微信授权登录与注册

**功能描述**：用户通过微信小程序一键授权登录，首次登录需绑定手机号。

**交互流程**：
1. 用户打开小程序 → 系统调用 `wx.login()` 获取 code
2. 后端用 code 换取 openid + session_key
3. 判断是否已注册：
   - 已注册 → 进入首页
   - 未注册 → 跳转手机号绑定页
4. 手机号绑定：调用 `wx.getPhoneNumber()` 获取微信绑定手机号，或手动输入
5. 绑定成功 → 进入首页

**交互说明**：
- 登录页展示品牌 logo、产品名称、一键登录按钮
- 获取手机号失败时，支持手动输入手机号 + 验证码方式
- 登录状态过期（session 失效）时自动静默重新登录

**数据规则**：
- openid 唯一标识用户，不可重复
- 手机号需格式校验（11 位手机号）
- 用户信息（头像/昵称）通过微信授权获取，支持后续修改

### 3.1.2 宠物档案管理

**功能描述**：为每只宠物建立完整的电子档案。

**页面说明 — 宠物列表页**：
- 卡片式展示，每张卡片包含：宠物照片（圆形）、昵称、品种、体型标签
- 右上角「+添加宠物」按钮
- 每张卡片点击进入宠物详情页
- 长按/左滑显示删除操作（软删除）

**页面说明 — 添加/编辑宠物页**：
- 表单字段：
  - 宠物照片（必填，支持拍照/相册，最多 5 张，第一张为头像）
  - 宠物昵称（必填，≤20 字符）
  - 品种（必填，下拉选择，按狗/猫/其他分类）
  - 体型（必填，单选：小型 ≤5kg / 中型 5-15kg / 大型 ≥15kg）
  - 性别（必填，单选：公/母/已绝育公/已绝育母）
  - 生日（选填，日期选择器）
  - 备注（选填，≤200 字符）

**数据规则**：
- 单用户最多添加 10 只宠物（MVP 限制）
- 品种数据来源于系统预置品种库
- 体型影响寄养费用计算
- 删除宠物为软删除，保留历史消费记录

### 3.1.3 疫苗记录

**功能描述**：记录宠物的疫苗接种信息。

**页面说明**：
- 在宠物详情页内以列表展示
- 每条记录：疫苗名称、接种日期、下次接种日期（自动计算或手动设置）
- 支持新增/编辑/删除
- 下次接种日期到期时，通过微信订阅消息推送提醒（与洗护提醒复用通道）

### 3.1.4 消费记录查看

**功能描述**：宠物主人查看自己宠物在本店的所有消费记录。

**页面说明**：
- 在宠物详情页内展示
- 列表按时间倒序排列
- 每条记录：日期、服务项目（如"洗澡-中型犬"）、金额、门店名称
- 点击进入详情：完整服务项目清单、操作店员、支付方式、备注

### 3.1.5 会员等级

**功能描述**：展示会员当前等级、升级进度、降级预警。

**页面说明 — 会员等级页**：
- 顶部：当前等级徽章 + 等级名称
- 进度条：当前累计消费 / 下一等级阈值
- 文字提示：「再消费 XX 元即可升级为 XX 会员」
- 等级权益说明折叠面板

**降级预警交互**：
- 当连续无消费达到 150 天（降级前 30 天），系统通过微信订阅消息推送预警
- 预警通知文案示例：「亲爱的 XX，您的 XX 会员等级将在 30 天后降级。在此期间消费任意金额即可保级哦~」
- 点击通知进入小程序会员等级页

**等级体系（默认）**：
| 等级 | 升级阈值（累计消费） | 折扣 |
|-----|-------------------|-----|
| 普通会员 | 0 元 | 无折扣 |
| 银卡会员 | 2000 元 | 9.5 折 |
| 金卡会员 | 5000 元 | 9 折 |
| 钻石会员 | 10000 元 | 8.5 折 |

> 以上阈值和折扣均由店长在后台可配置。

### 3.1.6 洗护提醒订阅与接收

**功能描述**：用户订阅微信订阅消息提醒，在洗护到期前收到提醒通知。

**订阅引导设计（一次性订阅强引导）**：

**触点 1 — 洗护完成弹窗**：
- 店员完成洗护服务后，宠物主人端弹出订阅引导弹窗
- 弹窗内容：标题「订阅洗护提醒，不再错过爱宠美容时间~」+ 说明文字 + 两个按钮（"暂不订阅" / "立即订阅"）
- 点击"立即订阅" → 调用 `wx.requestSubscribeMessage()` → 用户确认 → 订阅成功
- 点击"暂不订阅" → 关闭弹窗，下次洗护完成时再次引导

**触点 2 — 会员首页订阅状态**：
- 首页「洗护提醒」区域显示订阅状态
- 已订阅：显示「已开启提醒 ✓」+ 下次提醒日期
- 未订阅：显示醒目的「开启提醒」按钮

**触点 3 — 降级预警前再次引导**：
- 降级预警通知中附带订阅引导

**提醒接收**：
- 到期前 3 天推送提醒（可配置提前天数）
- 提醒消息内容：宠物昵称 + 距上次洗护天数 + 建议洗护日期 + 点击预约入口
- 消息中携带小程序跳转路径，用户点击可直接进入预约流程

### 3.1.7 寄养预约

**功能描述**：宠物主人通过小程序查看寄养日历并预约寄养。

**页面说明 — 寄养日历页**：
- 月历视图，每日格子显示剩余可预约笼位数
- 颜色标识：绿色（充足）、黄色（紧张）、灰色（已满）、蓝色（当前选中）
- 不可选过去的日期
- 法定节假日以特殊标记标识

**预约流程**：
1. 选择寄养日期范围（入住日 → 退房日）
2. 选择寄养宠物（可多选）
3. 系统自动计算预估费用：
   - 遍历日期范围内每一天
   - 判断每日日期类型（普通日/周末/法定假日）
   - 按宠物体型对应的日基础价 × 日期系数 累计
4. 展示费用预估明细（按日拆分显示每日价格）
5. 填写备注（选填）
6. 提交预约申请

**预约状态**：
| 状态 | 说明 | 用户可操作 |
|-----|------|----------|
| 待审核 | 已提交，等待店员确认 | 可取消 |
| 已确认 | 店员已确认 | 可取消（需提前 24 小时） |
| 已拒绝 | 店员拒绝/档期冲突 | 可重新预约 |
| 寄养中 | 宠物已入住 | 查看日志 |
| 已完成 | 已退房并结算 | 查看费用明细 |
| 已取消 | 用户取消/超时未到 | — |

### 3.1.8 寄养日志查看

**功能描述**：宠物主人实时查看寄养期间每日照片和状态记录。

**页面说明**：
- 时间线形式展示，每天一条记录
- 每条记录：日期、状态标签（饮食/精神/排便）、照片网格（点击可放大）、店员备注
- 支持下拉刷新获取最新日志

### 3.1.9 寄养费用明细

**功能描述**：展示寄养费用的详细计算过程。

**页面说明**：
- 顶部：总费用
- 明细列表：
  - 每日明细：日期 + 日期类型标签（普通日/周末/法定假日）+ 基础价 + 系数 + 当日费用
  - 额外费用项（如有）：项目名 + 金额
- 底部：优惠金额（会员折扣）+ 实付金额

## 3.2 Web 端（店员）功能规格

### 3.2.1 工作台首页（今日概览）

**功能描述**：店员登录后看到的第一个页面，展示当日关键数据和快捷操作。

**页面说明**：
- 数据卡片区（4 个卡片）：
  - 今日洗护预约数（点击跳转洗护排期）
  - 待寄养入住数（今日需入住的寄养数量）
  - 今日洗护到期提醒数（系统计算今日应提醒的宠物数）
  - 待处理提醒数（未审核的寄养预约 + 未推送的提醒）
- 快捷操作区：
  - 「快速录入消费」按钮 → 弹出快捷录入弹窗
  - 「新建寄养」按钮 → 跳转寄养新建页

### 3.2.2 快速录入消费（弹窗）

**功能描述**：一键完成消费录入的快捷操作。

**交互流程**：
1. 弹窗打开 → 搜索/选择宠物（输入宠物名或主人手机号）
2. 选择宠物后，显示该宠物信息（品种、体型）
3. 选择服务项目（多选，从预设服务列表中选择）
4. 系统自动计算总价（根据体型/数量）
5. 店员可手动调整金额
6. 选择支付方式（现金/微信/POS）
7. 确认 → 保存消费记录 + 更新会员累计消费

### 3.2.3 会员管理

**功能描述**：查看和管理所有会员信息。

**页面说明 — 会员列表页**：
- 表格展示：头像、昵称、手机号、宠物数量、会员等级、累计消费、最近消费日期
- 搜索栏：支持姓名/手机号/宠物名模糊搜索
- 筛选：会员等级下拉筛选
- 排序：按累计消费/最近消费日期排序

**页面说明 — 会员详情页**：
- Tab 结构：
  - 基本信息：头像、昵称、手机号、注册时间、会员等级、标签
  - 宠物列表：该会员下所有宠物卡片（点击进入宠物详情）
  - 消费记录：该会员所有宠物的消费记录汇总（时间倒序）
  - 等级信息：等级升降历史、降级预警记录

### 3.2.4 宠物档案管理

**功能描述**：管理所有宠物档案信息。

**页面说明**：
- 列表页：照片、昵称、主人姓名、品种、体型、会员等级、最近洗护日期
- 支持搜索/筛选（按品种/体型/主人）
- 详情页：完整信息展示 + 疫苗记录 + 消费记录 + 洗护历史
- 新建/编辑表单（同小程序端字段，额外增加健康备注字段）

### 3.2.5 消费管理

**功能描述**：录入和管理所有消费记录。

**页面说明 — 消费录入页**：
- 选择宠物（搜索选择）
- 选择服务类型（洗护/寄养/商品）
- 选择具体服务项目（从预设列表选择，可多选）
- 填写金额（默认根据预设价格自动计算，可修改）
- 选择支付方式
- 填写备注（选填）
- 保存

**页面说明 — 消费记录列表**：

### 3.2.6 洗护排期管理

**页面说明 — 提醒管理**：

### 3.2.7 寄养管理

#### 3.2.7.1 寄养日历看板

**功能描述**：可视化展示寄养档期占用情况。

**页面说明**：
- 月视图日历，每一行代表一个笼位/房间
- 每个寄养订单显示为横跨入住日到退房日的色条
- 颜色区分：蓝色（待审核）→ 绿色（已确认/寄养中）→ 灰色（已完成）→ 红色（已拒绝/已取消）
- 法定节假日区域背景色标识
- 空白日期格显示「可预订」
- 点击色条 → 打开寄养详情侧边栏
- 点击空白格 → 新建寄养预约

**新建寄养预约（弹窗/侧边栏）**：
- 选择宠物（搜索选择）
- 选择笼位/房间类型
- 选择入住日期 + 退房日期
- 系统自动计算预估费用（展示每日明细）
- 填写备注
- 提交（代客录入时为「已确认」状态；宠物主人提交时为「待审核」状态）

#### 3.2.7.2 预约审核

**功能描述**：审核宠物主人提交的寄养预约。

**页面说明**：
- 待审核列表：宠物名、主人姓名、预约日期范围、笼位类型、预约时间
- 操作：确认（分配笼位）/ 拒绝（填写拒绝原因）
- 确认时自动检测笼位冲突
- 确认后自动推送确认通知给宠物主人

#### 3.2.7.3 寄养执行 — 每日日志

**功能描述**：记录寄养期间每日宠物状态并推送给主人。

**页面说明 — 在寄列表**：
- 卡片式展示当前所有在寄宠物
- 每张卡片：宠物照片、昵称、入住日期、主人姓名、今日日志状态（已记录/未记录）

**页面说明 — 日志记录页**：
- 选择日期（默认今天）
- 状态记录：
  - 饮食情况：正常 / 偏少 / 未进食
  - 精神状态：活泼 / 正常 / 偏低
  - 排便情况：正常 / 偏软 / 腹泻 / 未排便
- 照片上传：支持拍照/上传，最多 9 张
- 文字备注（选填）
- 保存后显示「推送给主人」按钮
- 点击推送 → 通过微信订阅消息发送给宠物主人

#### 3.2.7.4 费用结算

**功能描述**：寄养结束后的费用结算。

**费用自动计算规则**：
```
总费用 = Σ(每日费用) + 额外费用 - 会员折扣

每日费用 = 基础日价(按体型) × 日期系数

日期系数：
- 普通日 → 1.0（默认）
- 周末（周六/周日）→ 1.2（默认）
- 法定假日 → 1.5（默认）

基础日价（按体型，店长可配置）：
- 小型犬/猫：80 元/天
- 中型犬：100 元/天
- 大型犬：130 元/天

会员折扣（按等级）：
- 普通会员：无折扣
- 银卡：9.5 折
- 金卡：9 折
- 钻石：8.5 折
```

**页面说明 — 结算页**：
- 宠物信息 + 寄养日期范围
- 费用明细表（按日展示：日期、日期类型、系数、当日费用）
- 额外费用项列表（可添加/删除）
- 会员折扣
- 应收金额
- 收款方式选择（微信/现金/POS）
- 确认结算 → 生成账单 + 更新寄养状态为「已完成」

# 4. 页面流程

## 4.1 小程序端核心流程

### 4.1.1 首次使用流程

```
打开小程序 → 微信授权登录 → 绑定手机号 → 引导添加宠物 → 引导订阅洗护提醒 → 进入首页
```

### 4.1.2 寄养预约流程

```
首页 → 寄养服务 → 查看日历 → 选择日期范围 → 选择宠物 → 查看预估费用 → 填写备注 → 提交预约
→ 等待审核 → 收到确认通知 → 到店入住 → 每日查看日志 → 寄养结束 → 查看费用明细 → 支付
```

### 4.1.3 洗护提醒闭环流程

```
收到洗护提醒通知 → 点击进入小程序 → 查看宠物洗护建议 → 点击预约洗护 → 选择时段 → 确认预约
→ 到店洗护 → 洗护完成 → 弹出订阅引导 → 订阅下次提醒 → 闭环
```

## 4.2 Web 端核心流程

### 4.2.1 店员日常操作流程

```
登录 → 查看今日概览 → 处理待办事项
├── 接待到店客户 → 快速录入消费 / 安排洗护
├── 处理寄养预约审核 → 确认/拒绝
├── 寄养中宠物 → 记录每日日志 → 推送照片给主人
└── 寄养到期 → 退房 → 费用结算 → 收款
```

### 4.2.2 寄养费用结算流程

```
寄养退房 → 系统自动计算费用（遍历每日：日期类型×系数×体型基础价）
→ 显示费用明细 → 添加额外费用（如有）→ 应用会员折扣 → 确认应收金额
→ 选择收款方式 → 确认结算 → 生成账单 → 推送账单通知给宠物主人
```

# 5. 交互说明

## 5.1 全局交互规范

### 5.1.1 加载状态
- 页面初次加载：骨架屏（Skeleton）占位
- 操作提交：按钮 loading 状态 + 禁用重复点击
- 列表加载更多：底部 loading 指示器

### 5.1.2 操作反馈
- 成功操作：顶部 Toast 提示（绿色，2 秒自动消失）
- 失败操作：顶部 Toast 提示（红色，需手动关闭或 5 秒消失）
- 确认操作（删除、取消预约等）：弹出确认对话框，需二次确认
- 危险操作（删除宠物、作废消费记录）：红色确认按钮 + 需输入确认文字

### 5.1.3 空状态
- 列表为空时展示空状态插图 + 引导文案 + 操作按钮
- 如「还没有添加宠物哦~」+「立即添加」按钮

### 5.1.4 表单校验
- 实时校验：输入时校验格式（如手机号、必填项）
- 提交校验：提交时统一校验，未通过项高亮显示 + 错误提示文字
- 校验规则明确标注：必填项标星号（*），字数限制显示「X/20」

## 5.2 关键交互说明

### 5.2.1 寄养日历拖拽交互（Web 端）
- 拖拽色条左右两端 → 调整入住/退房日期
- 拖拽色条中间 → 整体移动（需目标日期有空位）
- 拖拽到已满日期 → 色条回弹 + 提示「该日期笼位已满」
- 拖拽结束后 → 弹出确认框「确认调整寄养日期？」→ 确认后重新计算费用

### 5.2.2 微信订阅消息引导交互（小程序端）
- 弹窗层级：最高层级，覆盖当前页面
- 弹窗不可滑动关闭，必须点击按钮选择
- "暂不订阅"按钮使用弱化样式（灰色文字按钮）
- "立即订阅"按钮使用强调样式（主色调填充按钮）
- 弹窗触发时机：
  1. 洗护完成时（最强触发点）
  2. 首次添加宠物时
  3. 进入会员等级页时（降级预警前）

### 5.2.3 寄养日志照片上传交互
- 支持两种上传方式：拍照（调用相机）/ 从相册选择
- 上传过程：缩略图显示上传进度条
- 上传失败：显示红色感叹号 + 「重试」按钮
- 支持长按拖拽排序
- 最多 9 张，单张不超过 2MB，自动压缩

### 5.2.5 后台侧边栏模块切换交互（Web 端）
- 店员工作台侧边栏的每个一级功能均需可点击切换，并显示独立完整页面或模块：今日概览、会员管理、宠物档案、消费管理、洗护排期、寄养管理、提醒管理、经营数据、门店设置。
- 寄养管理内部以二级页签承载「日历看板 / 预约审核 / 寄养日志 / 费用结算」，避免侧边栏重复堆叠，同时保证每个子流程可直接切换。
- 店长管理侧边栏需覆盖经营数据看板、基础配置、节假日配置、会员等级管理、提醒渠道配置、员工管理；基础配置内部以二级菜单承载门店信息、服务项目与价格、洗护周期默认值、寄养价格策略。
- 点击侧边栏时需同步更新页面标题、激活态和主内容区；顶部快捷按钮仅作为常用入口，不替代侧边栏完整导航。

# 7. 非功能需求（补充）

## 7.1 安全需求
- 手机号等敏感信息加密存储（AES-256）
- API 接口使用 JWT Token 认证
- 微信 openid 等敏感数据不暴露给前端
- 寄养照片设置访问权限（仅相关会员和门店员工可查看）

## 7.2 可用性需求
- 系统可用性 ≥ 99.5%
- 数据每日自动备份
- 关键操作（消费录入、寄养结算）支持操作日志追溯

## 7.3 扩展性需求
- 架构设计预留多店版扩展接口
- 消息通道抽象层设计，便于 V1.1 扩展短信通道
- 等级规则引擎设计，便于 V1.1 扩展分级降级周期
