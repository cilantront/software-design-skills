# 1 概述 
## 1.1 需求背景 

随着短视频和社交媒体平台的兴起，越来越多的内容创作者开始在多个平台同时运营。个人创作者通常需要在小红书、B站、抖音、公众号等3-5个平台同步发布内容，而MCN机构则需要管理多位达人的内容排期。目前，这些创作者普遍使用备忘录、微信群聊、Excel表格等零散工具来管理选题和排期，导致以下痛点：

1. **选题分散易丢失**：灵感记录在备忘录、聊天记录中，无法集中管理和沉淀
2. **排期混乱易遗漏**：多平台发布计划缺乏统一的可视化视图，经常忘记某平台该更新或出现撞期
3. **协作效率低下**：小团队成员之间选题讨论结果无法沉淀，发布计划缺乏透明度
4. **节奏难以把控**：无法直观感知各平台的发布频率和节奏，难以建立稳定的更新计划

本产品定位为轻量级Web SaaS工具，聚焦"选题管理 + 发布日历 + 排期协同"三大核心场景，帮助创作者将散落在各处的选题灵感集中管理，以可视化日历统筹多平台发布节奏。采用免费+付费的商业模式，免费版支持50个选题和3个平台，专业版¥29/月提供无限选题、无限平台和团队协作功能。

预期目标：上线30天内达到500注册用户，周活跃用户达到注册用户的30%，人均创建选题数≥5个，免费版转化率达到5%以上。

### 1.3.2 目标用户与使用场景

**目标用户：**
- 多平台个人创作者：同时在3-5个平台运营，每周发布5-15条内容
- MCN运营人员：管理5-20位达人，需要协调多位创作者的内容排期
- 自媒体小团队：2-5人的小团队，共同运营1-3个账号矩阵

**核心价值：**
1. 选题不丢失：将散落在备忘录、聊天记录中的选题集中到选题库统一管理
2. 排期不混乱：多平台发布计划在同一个日历上一目了然，避免撞期或遗漏
3. 节奏可感知：可视化展示各平台发布频率与节奏，帮助创作者建立稳定的更新计划

**核心使用场景：**
- 灵感捕捉：创作者快速记录选题到选题库，标记为"灵感"状态
- 选题规划：周末集中规划下周内容，从选题库挑选选题拖拽到日历安排发布日期
- 排期查看：打开日历视图，一眼看到本周各平台的发布安排
- 状态流转：内容制作完成后，将选题状态从"草稿"推进到"待发布"，发布后标记为"已发布"

### 3.1.1 邮箱注册与登录

功能描述：提供用户身份认证入口，支持邮箱注册和登录，使用JWT token维持会话，有效期7天。MVP版本不支持微信登录，延至P1版本实现。

优先级与依赖说明：
| 项 | 内容 |
| --- | --- |
| 优先级 | P0 |
| 依赖需求 | 无 |
| 前置条件 | 用户需拥有有效邮箱地址 |

### 3.1.2 邮箱注册与登录—详细流程 

```mermaid
flowchart TD
    A[用户访问首页] --> B{是否已登录?}
    B -->|是| C[进入工作台]
    B -->|否| D[显示登录页面]
    D --> E{已有账号?}
    E -->|是| F[输入邮箱和密码]
    E -->|否| G[输入邮箱、密码、确认密码]
    G --> H[提交注册]
    H --> I[后端验证邮箱格式]
    I --> J{邮箱是否已注册?}
    J -->|是| K[提示邮箱已存在]
    J -->|否| L[创建用户账户]
    L --> M[生成JWT token]
    M --> N[返回token给前端]
    N --> O[前端保存token到localStorage]
    O --> C
    
    F --> P[提交登录]
    P --> Q[后端验证凭据]
    Q --> R{验证成功?}
    R -->|是| M
    R -->|否| S[提示邮箱或密码错误]
    
    C --> T[加载用户工作空间]
    T --> U[显示工作台仪表盘]
```

业务规则说明：
1. 邮箱格式必须符合标准邮箱格式（xxx@xxx.xxx）
2. 密码长度至少8位，包含字母和数字
3. JWT token有效期为7天，过期后需重新登录
4. 同一邮箱只能注册一次，重复注册时提示邮箱已存在
5. 登录失败时不明确提示是邮箱不存在还是密码错误，统一提示"邮箱或密码错误"
6. Token存储在浏览器localStorage中，每次请求携带在Authorization header

7. 后续迭代支持AI辅助评分和用户自定义权重

flowchart LR
    A[登录/注册页] --> B[工作台仪表盘]

    B --> D[选题库]

    D --> J[看板视图]

- 特殊交互：
  1. 拖拽排期：从选题库拖拽选题到日历，显示预览幽灵卡片，释放后弹出确认对话框

  6. 图片上传：支持拖拽上传和点击上传，上传过程显示进度条
  7. 富文本编辑：选题描述支持基本富文本格式，工具栏悬浮选中文本时出现

## 6.2 安全需求 

| 编号 | 项（系统数据 / 处理过程） |
| --- | --- |
| 0001 | 用户密码必须加盐哈希后存储，禁止明文存储密码 |
| 0002 | JWT token有效期为7天，过期后需重新认证 |
| 0003 | 用户数据严格隔离，不可跨工作空间访问他人数据 |
| 0004 | 敏感操作（删除选题、移除团队成员）需二次确认 |
| 0005 | API接口实施速率限制，防止恶意刷请求（单IP每分钟不超过100次） |
| 0006 | 所有HTTPS传输，禁止HTTP明文传输敏感数据 |
| 0007 | 图片上传需校验文件类型和大小，防止恶意文件上传 |
| 0008 | SQL查询使用参数化语句，防止SQL注入攻击 |
