## 2. 产品定位与价值主张

### 2.1 产品定位

一款面向多平台内容创作者的**轻量级选题管理与发布排期工具**，聚焦"选题管理 + 发布日历 + 排期协同"三大核心场景，帮助创作者将散落在各处的选题灵感集中管理，以可视化日历统筹多平台发布节奏。

### 2.2 差异化定位

- **不做**通用项目管理（区别于 Notion / 飞书模板）
- **不做**大型 MCN 管理系统（区别于企业级 SaaS）
- **专注**个人创作者与小团队的"选题 → 排期 → 发布"工作流
- **轻量、专注、低价**切入市场

### 2.3 核心价值

1. **选题不丢失**：将散落在备忘录、聊天记录中的选题集中到一个选题库，统一管理
2. **排期不混乱**：多平台发布计划在同一个日历上一目了然，避免撞期或遗漏
3. **节奏可感知**：可视化展示各平台发布频率与节奏，帮助创作者建立稳定的更新计划

## 3. 目标用户与场景

### 3.1 用户画像

#### 画像 A：多平台个人创作者

- **特征**：同时在 3-5 个平台运营，每周发布 5-15 条内容
- **痛点**：选题记在备忘录里，排期靠脑子记，经常忘记某平台该更新了
- **诉求**：一个地方看到所有平台的排期，选题从灵感到发布有清晰的流转

#### 画像 B：MCN 运营人员

- **特征**：管理 5-20 位达人，需要协调多位创作者的内容排期
- **痛点**：各达人的选题和排期散落在微信群和 Excel 中，协调困难
- **诉求**：能按达人查看排期，团队成员可以协同编辑和审核

#### 画像 C：自媒体小团队

- **特征**：2-5 人的小团队，共同运营 1-3 个账号矩阵
- **痛点**：选题讨论结果无法沉淀，发布计划缺乏可视化
- **诉求**：团队共享选题库，协同管理发布计划

### 3.2 核心使用场景

| 场景编号 | 场景名称 | 描述 |
|----------|----------|------|
| S-01 | 灵感捕捉 | 创作者在通勤时想到一个选题，快速打开工具记录到选题库，标记为"灵感"状态 |
| S-02 | 选题规划 | 周末集中规划下周内容，从选题库中挑选选题，拖拽到日历上安排各平台发布日期 |
| S-03 | 排期查看 | 打开日历视图，一眼看到本周各平台的发布安排，确认是否有遗漏或冲突 |
| S-04 | 状态流转 | 内容制作完成后，将选题状态从"草稿"推进到"待发布"，发布后标记为"已发布" |
| S-05 | 发布时间参考 | 查看平台推荐的最佳发布时间，结合自己的数据选择最合适的发布时段 |
| S-06 | 团队协作 | MCN 运营将排期任务分配给团队成员，成员更新进度后可实时同步 |

### 4.2 发布日历模块

#### 4.2.1 日历视图

**FR-CAL-001：月视图**
- 以月为单位展示，每个日期格子内显示当天各平台的发布计划
- 每条发布计划以平台色块标识，显示选题名称（截断至 10 字）
- 点击日期可查看当天详细发布列表
- 支持上下月切换

**FR-CAL-002：周视图**
- 以周为单位展示，横轴为周一至周日，纵轴为各平台
- 每个单元格显示当天的发布计划卡片
- 卡片显示：选题名称、发布时间、平台图标
- 支持点击卡片查看详情或编辑

**FR-CAL-003：列表视图**
- 以时间倒序展示未来 30 天的发布计划
- 每条记录显示：日期、时间、平台、选题名称、状态
- 支持按平台筛选
- 支持按状态筛选

#### 4.2.2 拖拽排期

**FR-CAL-010：拖拽创建**
- 从选题库中拖拽选题到日历日期上，自动创建发布计划
- 拖拽时显示选题卡片预览
- 放置后弹出确认弹窗，可设置发布时间、平台特定标题

**FR-CAL-011：拖拽调整**
- 已有发布计划可在日历格子间拖拽移动，调整发布日期
- 跨平台移动时（周视图）自动更新所属平台
- 拖拽后自动保存，支持撤销（Ctrl+Z）

**FR-CAL-012：冲突提示**
- 同一平台同一天安排超过 2 条发布时，显示黄色警告标识
- 提供"查看详情"链接，展示当天该平台的全部发布计划

### 4.3 选题管理模块

#### 4.3.1 选题库

**FR-TOPIC-001：看板视图**
- 默认以看板形式展示，列按选题状态分组：
  - 💡 灵感 → 📝 草稿 → 📋 待发布 → ✅ 已发布
- 每张卡片显示：选题名称、关联平台图标、计划发布日期、负责人头像
- 支持列内拖拽调整卡片顺序
- 支持跨列拖拽变更选题状态

**FR-TOPIC-002：列表视图**
- 以表格形式展示所有选题
- 列：选题名称、状态、关联平台、计划发布日期、创建时间、负责人
- 支持多列排序
- 支持关键词搜索

**FR-TOPIC-003：新建选题**
- 必填：选题名称
- 选填：选题描述、灵感来源（URL/文本）、标签、关联平台
- 新建选题默认进入"灵感"状态
- 支持快速新建（仅输入名称即可）和详细新建两种模式

**FR-TOPIC-004：选题详情**
- 选题名称（可编辑）
- 选题描述（富文本）
- 灵感来源记录
- 关联平台及对应发布计划（可添加多个平台的发布计划）
- 标签管理
- 状态流转历史（时间线形式）
- 备注/评论

#### 4.3.2 选题状态流转

**FR-TOPIC-010：状态定义**

| 状态 | 含义 | 流转规则 |
|------|------|----------|
| 💡 灵感 | 初始想法，尚未展开 | 可流转到"草稿"或"已放弃" |
| 📝 草稿 | 正在制作内容 | 可流转到"待发布"或退回"灵感" |
| 📋 待发布 | 内容制作完成，等待发布 | 可流转到"已发布"或退回"草稿" |
| ✅ 已发布 | 内容已在平台发布 | 终态，不可变更 |
| ❌ 已放弃 | 选题被放弃 | 终态，不可变更 |

**FR-TOPIC-011：批量操作**
- 支持多选选题，批量变更状态
- 支持批量打标签
- 支持批量删除（需二次确认）

**FR-TOPIC-012：筛选与搜索**
- 按状态筛选
- 按关联平台筛选
- 按标签筛选
- 按关键词搜索（匹配名称和描述）
- 按创建时间范围筛选

### 4.4 多平台管理模块

#### 4.4.1 平台配置

**FR-PLAT-001：预置平台**
系统预置以下主流平台（含平台图标、主题色）：
- 小红书
- B站（哔哩哔哩）
- 抖音
- 公众号（微信公众号）
- 视频号
- 快手
- 微博
- 今日头条

**FR-PLAT-002：自定义平台**
- 用户可添加自定义平台
- 需填写：平台名称、选择/上传图标、主题色
- 自定义平台与预置平台在功能上完全一致

**FR-PLAT-003：平台启用/禁用**
- 用户可启用/禁用平台
- 禁用后该平台不在日历和选题库中显示，但数据保留
- 免费版限制 3 个平台，专业版不限

#### 4.4.2 平台模板

**FR-PLAT-010：内容模板**
- 每个平台可配置内容模板，包含：
  - 标题格式要求
  - 正文模板（占位符支持）
  - 标签/话题建议
  - 封面尺寸参考
  - 发布注意事项
- 从选题创建发布计划时，自动加载对应平台模板

**FR-PLAT-011：预置模板**
- 系统为每个预置平台提供默认模板
- 用户可修改默认模板，也可创建多套模板并选择使用

#### 4.4.3 发布状态汇总

**FR-PLAT-020：平台概览面板**
- 显示各平台本周/本月的发布数量
- 显示各平台当前待发布数量
- 显示各平台最近一次发布时间
- 以卡片组形式展示，每个平台一张卡片

**FR-PLAT-021：发布频率统计**
- 以折线图展示各平台近 30 天的发布频率
- 标注平均发布间隔
- 高亮发布频率异常（如某平台超过 7 天未更新）

### 4.5 智能推荐模块

#### 4.5.1 最佳发布时间推荐

**FR-REC-001：基础推荐**
- 按平台提供通用推荐时段（基于行业经验数据）
- 示例：
  - 小红书：工作日 12:00-13:00、18:00-20:00；周末 10:00-12:00
  - B站：工作日 17:00-19:00；周末 10:00-12:00
  - 抖音：工作日 12:00-13:00、18:00-22:00
  - 公众号：工作日 7:00-8:00、12:00-13:00、20:00-22:00
- 排期时以建议标签形式展示

**FR-REC-002：个性化推荐（专业版）**
- 基于用户历史发布数据的互动量分析
- 按平台推荐最佳发布日与时间段
- 推荐置信度随数据量增加而提升
- 数据不足时提示"暂无足够数据，使用通用推荐"

### 4.6 用户与团队模块

**FR-USER-001：注册/登录**
- 支持邮箱注册
- 支持微信快捷登录（MVP 阶段可后延）

**FR-USER-002：个人空间**
- 每位用户默认拥有一个个人的工作空间
- 空间内包含完整的选题库、日历、平台配置

**FR-USER-003：团队空间（专业版）**
- 支持创建团队空间，邀请成员加入
- 团队空间内选题库、日历共享
- 支持分配选题负责人

**FR-USER-004：成员权限（专业版）**

| 角色 | 权限 |
|------|------|
| 管理员 | 全部权限，含成员管理、空间设置 |
| 编辑者 | 创建/编辑选题、排期、标记发布状态 |
| 查看者 | 仅查看，不可编辑 |

## 5. 业务规则

### 5.1 版本限制规则

| 功能 | 免费版 | 专业版 |
|------|--------|--------|
| 选题数量 | 50 个 | 不限 |
| 平台数量 | 3 个 | 不限 |
| 团队协作 | 不支持 | 支持 |
| 发布数据分析 | 基础统计 | 完整分析 |
| 选题评分 | 不支持 | 支持 |
| 内容模板 | 3 套 | 不限 |
| 发布时间推荐 | 通用推荐 | 个性化推荐 |

### 5.2 数据规则

- 选题被删除后进入回收站，30 天后永久删除
- 发布计划关联的选题被删除时，发布计划一并移除
- 同一选题可关联多个平台的发布计划，各计划独立管理
- 选题状态变更时，自动记录变更时间并推送通知（团队空间内）

### 5.3 免费额度提醒

- 选题数达到 40/50 时，顶部提示"选题即将用满，升级专业版享受不限选题"
- 选题数达到 50/50 时，新建选题弹窗提示升级
- 平台数达到 3/3 时，添加平台弹窗提示升级

## 6. 非功能需求

### 6.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面首屏加载 | ≤ 2 秒（4G 网络） |
| 日历视图切换 | ≤ 500ms |
| 选题拖拽响应 | 即时反馈，操作延迟 ≤ 100ms |
| 列表视图加载（1000 条） | ≤ 1 秒 |
| 数据保存 | 操作后 ≤ 500ms 完成持久化 |

### 6.2 兼容性需求

| 维度 | 要求 |
|------|------|
| 浏览器 | Chrome 90+、Safari 15+、Edge 90+、Firefox 90+ |
| 屏幕 | 最小支持 1024×768，推荐 1440×900 及以上 |
| 移动端 | MVP 阶段优先 Web 端，后续考虑移动端适配 |

### 6.3 可用性需求

- 系统可用性 ≥ 99.5%
- 数据每日自动备份
- 支持数据导出（选题列表导出为 CSV/Excel）

### 6.4 安全需求

- 用户数据隔离，不可跨空间访问
- 登录态使用 JWT，过期时间 7 天
- 敏感操作（删除空间、移除成员）需二次确认
- API 接口限流，防止滥用

## 9. 约束与假设

### 9.1 约束

- MVP 开发周期 7 天，需严格控制范围
- 免费版需有明确的使用限制，为付费转化留出空间
- MVP 阶段不接入各平台 API（不自动发布），仅提供排期管理
- 移动端适配不在 MVP 范围内

### 9.2 假设

- 目标用户具备基本的互联网工具使用能力
- 用户已有各平台的账号，且习惯在多平台发布内容
- 发布时间推荐数据可从行业公开报告或平台官方建议中获取
- 用户愿意为"提升多平台运营效率"付费（¥29/月的价格敏感区间）
