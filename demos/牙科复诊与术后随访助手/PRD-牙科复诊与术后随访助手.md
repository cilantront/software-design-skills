# 1. 产品概述

## 1.1 产品定位

**牙科复诊与术后随访助手**是一款面向小型口腔诊所（1-5家门店）的垂直行业 SaaS 工具，聚焦于口腔治疗后"复诊提醒"和"术后随访闭环"两大核心场景，帮助诊所实现随访流程自动化、风险预警智能化、随访数据可量化。

**不做的事情**：
- 不做 HIS（医院信息系统）或诊所管理系统
- 不做电子病历（仅保留与随访相关的治疗摘要）
- 不做在线问诊
- 不对接医保结算

**产品价值主张**：
> "让每一次术后随访都不被遗漏，让每一位患者都感受到诊所的专业关怀。"

## 1.2 目标用户

| 用户角色 | 典型画像 | 核心痛点 | 产品价值 |
|---------|---------|---------|---------|
| **诊所管理员** | 经营 1-5 家口腔诊所的老板或运营负责人 | 随访质量参差不齐，缺乏数据支撑运营决策 | 标准化随访流程，可视化运营数据 |
| **牙科医生** | 种植/正畸/全科牙医 | 担心术后并发症被遗漏，人工跟踪耗时 | 高风险患者自动预警，聚焦诊疗本身 |
| **前台/客服** | 诊所前台接待或客户运营人员 | 每天手动拨打电话提醒复诊，工作量大易遗漏 | 自动化任务列表，一键发送提醒 |
| **患者** | 接受种植/正畸/补牙等治疗的术后患者 | 忘记复诊时间，不清楚术后恢复是否正常 | 及时提醒，方便的反馈渠道，感受到关怀 |

## 1.3 商业目标

| 指标 | 目标值 | 说明 |
|------|-------|------|
| MVP 上线时间 | 7-10 天 | 诊所后台核心功能 + 患者 H5 + 后台任务引擎 |
| 首年签约诊所数 | 200+ | 聚焦一二线城市小型口腔诊所 |
| 付费转化率 | ≥30% | 免费试用 14 天后转付费 |
| 月均 ARPU | ¥150-250/门店 | 基础订阅 + 增值服务 |
| 随访完成率提升 | ≥60% | 相比人工提醒模式 |

## 1.5 核心业务流程

### 1.5.1 端到端随访闭环

```
患者到院治疗 → 前台登记治疗信息 → 系统匹配随访方案模板 → 自动生成随访计划
                                                              ↓
                                                     随访任务调度引擎
                                                              ↓
                                                     到达随访时间点
                                                              ↓
                                          ┌───────────────────┼───────────────────┐
                                          ↓                   ↓                   ↓
                                     发送短信提醒         推送微信消息         系统内通知
                                          ↓                   ↓
                                     患者收到提醒（含 H5 链接）
                                          ↓
                                   患者打开 H5 填写恢复问卷/上传照片
                                          ↓
                                    系统分析反馈内容
                                          ↓
                              ┌───────────┴───────────┐
                              ↓                       ↓
                        正常反馈                 高风险信号
                              ↓                       ↓
                      记录随访完成           预警通知医生/前台
                              ↓                       ↓
                      进入下一节点           人工回访并记录结果
                              ↓                       ↓
                      随访计划完成 ←───────────── 调整治疗方案（如需要）
                              ↓
                         归档随访记录
```

# 7. 数据模型

## 7.1 核心数据实体

| 实体 | 描述 | 关键字段 |
|------|------|---------|
| **Clinic（诊所）** | 诊所基本信息 | id, name, address, phone, logo, brand_color |
| **User（用户）** | 系统用户（管理员/医生/前台）| id, clinic_id, name, phone, role, status |
| **Patient（患者）** | 患者档案 | id, clinic_id, name, phone, gender, age |
| **Treatment（治疗记录）** | 患者治疗记录 | id, patient_id, doctor_id, type, treatment_date, followup_plan_id |
| **FollowupTemplate（随访方案模板）** | 随访方案配置 | id, clinic_id, treatment_type, name, status, nodes_config |
| **FollowupPlan（随访计划）** | 实例化的随访计划 | id, treatment_id, template_id, patient_id, status |
| **FollowupNode（随访节点）** | 随访计划的具体节点 | id, plan_id, node_index, scheduled_date, status, sent_at |
| **FollowupTask（随访任务）** | 待执行的随访任务 | id, node_id, task_type, status, sent_at, feedback_id |
| **Feedback（患者反馈）** | 患者提交的问卷回答 | id, task_id, answers, photos, risk_level, submitted_at |
| **RiskAlert（风险预警）** | 高风险预警记录 | id, feedback_id, patient_id, level, status, handler_id |
| **Callback（回访记录）** | 人工回访记录 | id, alert_id, handler_id, method, result, callback_time |
| **MessageLog（消息记录）** | 短信/微信发送记录 | id, task_id, channel, content, status, sent_at |
| **Satisfaction（满意度评价）** | 患者满意度评价 | id, patient_id, plan_id, rating, comment, created_at |
| **Subscription（订阅）** | 诊所订阅信息 | id, clinic_id, plan_type, start_date, end_date, status |

## 7.2 核心数据关系

```
Clinic 1 ──── N User
Clinic 1 ──── N Patient
Clinic 1 ──── N FollowupTemplate
Clinic 1 ──── 1 Subscription

Patient 1 ──── N Treatment
Treatment N ──── 1 FollowupTemplate
Treatment 1 ──── 1 FollowupPlan

FollowupPlan 1 ──── N FollowupNode
FollowupNode 1 ──── N FollowupTask
FollowupTask 1 ──── 1 Feedback (可选)

Feedback 1 ──── N RiskAlert (可选)
RiskAlert 1 ──── N Callback (可选)

FollowupTask N ──── N MessageLog
Treatment 1 ──── N Satisfaction (可选)
```

# 9. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 短信发送到达率不稳定 | 患者收不到提醒 | 对接多家短信服务商，实现自动切换 |
| 微信模板消息审核不通过 | 无法使用微信渠道 | 提前准备合规模板，预留短信兜底 |
| 患者不愿配合填写问卷 | 随访完成率降低 | 简化问卷设计，短信文案强调关怀 |
| 诊所人员流动大 | 系统使用中断 | 简化操作，提供操作引导 |
| 患者隐私数据泄露 | 合规风险 | 加密存储、权限控制、安全审计 |
