## 1. 产品概述

### 1.1 产品定位

创作者评论区互动话术助手是一款面向多平台内容创作者的**智能评论区管理工具**，聚焦于"评论回复效率"这一高频痛点，通过 AI 技术提供跨平台评论聚合、情感分析、多风格话术生成和互动数据洞察四大核心能力。

### 1.2 产品愿景

> 让每一条值得回复的评论，都不被错过。

### 1.3 核心价值主张

| 价值维度 | 具体描述 |
|----------|----------|
| **效率提升** | 将分散在多个平台的评论统一聚合，减少创作者在多个 App 之间切换的时间成本 |
| **回复质量** | AI 生成亲切/幽默/专业三种风格的话术，让回复不再千篇一律 |
| **数据洞察** | 通过互动周报和情感分析，帮助创作者了解粉丝需求，优化内容方向 |
| **粉丝粘性** | 高效、有温度的回复提升粉丝满意度，促进算法推荐 |

### 1.4 产品差异化

- **不做**内容发布工具（区别于新榜、蝉妈妈）
- **不做**数据分析大盘（区别于飞瓜、灰豚）
- **只做**评论区互动话术生成 + 情感分析 + 高价值评论识别

### 1.5 MVP 范围（第一期）

**包含：**
- 多平台评论聚合（小红书、抖音、B站）
- AI 情感分析与优先级排序
- 三种风格话术生成
- 基础数据看板
- 免费版/Pro版订阅体系

**不包含（第二期）：**
- 快手、微博、知乎等平台接入
- 个性化话术定制（基于创作者历史风格学习）
- 批量回复功能
- 评论危机预警
- 移动端小程序
- 运营管理后台

## 2. 用户画像与使用场景

### 2.1 目标用户画像

#### 用户A：小鹿（个人创作者）
- **年龄：** 24岁
- **平台：** 小红书（12万粉）、B站（5万粉）
- **内容类型：** 美妆护肤分享
- **日均评论量：** 约80条
- **痛点：** 每天花2小时回复评论，回复慢导致粉丝流失；不知道如何回复负面评论
- **期望：** 快速生成得体的回复，节省时间

#### 用户B：老张（头部KOL）
- **年龄：** 30岁
- **平台：** 抖音（200万粉）、B站（80万粉）、小红书（50万粉）
- **内容类型：** 数码科技评测
- **日均评论量：** 约500条
- **痛点：** 评论量太大，重要评论容易被淹没；有助理想帮忙但无法统一话术风格
- **期望：** 智能筛选高价值评论，批量生成话术

#### 用户C：小李（MCN运营）
- **年龄：** 26岁
- **管理账号：** 15个创作者账号
- **痛点：** 需要同时管理多个账号的评论回复，效率低下
- **期望：** 统一管理多账号评论，提升团队整体回复效率

### 2.2 核心使用场景

| 场景 | 触发条件 | 用户行为 | 期望结果 |
|------|----------|----------|----------|
| **晨间评论处理** | 每天早9点 | 打开工作台，浏览过夜的评论，按优先级处理 | 30分钟内处理完所有重要评论 |
| **爆款视频评论应对** | 视频 unexpectedly 爆火 | 快速筛选正面评论，批量生成感谢话术 | 抓住热度窗口，快速互动 |
| **负面评论危机** | 出现负面评论 | 识别负面评论，选择专业风格生成安抚话术 | 得体回应，避免舆情扩大 |
| **周报复盘** | 每周日 | 查看互动周报，了解本周粉丝关注点 | 获得下周内容创作方向参考 |

## 5. 核心功能详细说明

### 5.1 平台账号管理

#### 5.1.1 功能概述

支持用户绑定小红书、抖音、B站三个平台的账号，实现跨平台评论数据聚合。

#### 5.1.2 功能规则

| 规则编号 | 规则描述 |
|----------|----------|
| ACC-01 | 同一平台支持绑定多个账号（免费版最多3个账号，Pro版不限） |
| ACC-02 | 绑定流程采用 OAuth 2.0 授权，用户需在平台页面完成授权确认 |
| ACC-03 | 授权有效期与平台策略一致，系统自动续期；续期失败时提醒用户重新授权 |
| ACC-04 | 授权状态实时显示：已授权（绿色）、即将过期（黄色）、已失效（红色） |
| ACC-05 | 解绑账号后，该账号的历史评论数据保留90天，之后自动清理 |

#### 5.1.3 页面元素

**平台账号管理页**

| 元素 | 类型 | 说明 |
|------|------|------|
| 已绑定账号列表 | 卡片列表 | 展示每个已绑定账号的平台图标、昵称、头像、授权状态 |
| 绑定新账号按钮 | 主按钮 | 点击弹出平台选择弹窗 |
| 授权状态标签 | 标签 | 绿色"已授权" / 黄色"即将过期" / 红色"已失效" |
| 操作菜单 | 下拉菜单 | 刷新授权 / 解绑账号 / 查看数据 |
| 最近同步时间 | 文本 | 显示该账号最近一次评论同步的时间 |

#### 5.1.4 交互流程

```
用户点击"绑定新账号" 
  → 弹出平台选择弹窗（小红书/抖音/B站）
  → 用户选择平台
  → 跳转至平台OAuth授权页面（新窗口）
  → 用户在平台页面完成授权
  → 回调至本系统，显示授权成功
  → 自动触发首次评论同步
  → 账号卡片出现在列表中
```

### 5.2 评论聚合与管理

#### 5.2.1 功能概述

将多平台评论统一聚合到一个工作台中，支持筛选、排序、搜索等操作，并提供评论处理状态管理。

#### 5.2.2 功能规则

| 规则编号 | 规则描述 |
|----------|----------|
| COM-01 | 评论自动同步频率：每5分钟一次（Pro版可自定义频率） |
| COM-02 | 评论列表默认按"优先级"排序（AI 综合评分），可切换为按时间排序 |
| COM-03 | 评论状态：待处理 → 已回复 / 已忽略，支持批量操作 |
| COM-04 | 支持按平台、情感类型、处理状态、时间范围进行组合筛选 |
| COM-05 | 评论数据保留180天（免费版）/ 永久保留（Pro版） |
| COM-06 | 同一内容（相同视频/图文）下的评论支持折叠展示 |
| COM-07 | 支持关键词搜索，搜索范围包括评论内容和创作者内容标题 |

#### 5.2.3 评论列表页

| 情感筛选 | 多选标签 | 全部 / 正面 / 中性 / 负面 |

| 搜索框 | 文本输入 | 搜索评论内容或内容标题 |

**评论卡片**

### 5.3 AI 情感分析

#### 5.3.1 功能概述

自动为每条评论进行 AI 情感分析，标注情感倾向（正面/中性/负面），并计算情感强度评分，为优先级排序提供依据。

#### 5.3.2 功能规则

| 规则编号 | 规则描述 |
|----------|----------|
| AI-01 | 情感分类：正面（positive）、中性（neutral）、负面（negative）三类 |
| AI-02 | 情感强度评分：1-5分，1为最弱，5为最强 |
| AI-03 | 新评论到达后500ms内完成情感分析 |
| AI-04 | 分析结果包含：情感类型、强度评分、关键词标签 |
| AI-05 | 用户可对 AI 分析结果进行纠错反馈（点赞/点踩图标） |
| AI-06 | 纠错数据用于后续模型优化 |

#### 5.3.3 情感分析展示

**情感标签样式**

| 情感类型 | 颜色 | 图标 | 示例 |
|----------|------|------|------|
| 正面-强烈 | 深绿 #059669 | 😊 | 强度 4-5 |
| 正面-温和 | 浅绿 #10B981 | 🙂 | 强度 2-3 |
| 中性 | 灰色 #6B7280 | 😐 | 强度 1-3 |
| 负面-温和 | 浅橙 #F59E0B | 😟 | 强度 2-3 |
| 负面-强烈 | 红色 #EF4444 | 😠 | 强度 4-5 |

### 5.4 智能话术生成

#### 5.4.1 功能概述

基于评论内容和情感分析结果，AI 自动生成 3 种不同风格的回复话术供用户选择和编辑。

#### 5.4.2 功能规则

| 规则编号 | 规则描述 |
|----------|----------|
| REP-01 | 三种风格：亲切型（友好温暖）、幽默型（轻松有趣）、专业型（严谨专业） |
| REP-02 | 每条评论生成 3-5 条候选话术 |
| REP-03 | 话术生成时间 < 3秒 |
| REP-04 | 话术内容自动过滤敏感词和不当内容 |
| REP-05 | 用户可对生成的话术进行编辑调整 |
| REP-06 | 支持一键复制话术到剪贴板 |
| REP-07 | 话术长度控制在 20-200 字之间（适合评论区回复场景） |
| REP-08 | 话术生成时考虑评论内容的情感倾向：正面→感谢互动，中性→引导问答，负面→安抚解释 |
| REP-09 | 免费版每月最多生成 300 条话术；Pro版不限 |
| REP-10 | 支持将常用话术保存为个人模板 |

#### 5.4.3 话术生成面板

**触发方式：** 点击评论卡片中的"生成话术"按钮，或点击评论卡片进入详情页

**面板布局：**

| 区域 | 元素 | 说明 |
|------|------|------|
| 顶部 | 原始评论展示 | 展示被回复的评论内容和上下文 |
| 风格选择 | 标签切换 | 亲切 / 幽默 / 专业 / 全部 |
| 话术候选区 | 卡片列表 | 每条话术一个卡片，包含话术内容、复制按钮、编辑按钮 |
| 编辑区 | 文本编辑器 | 支持对选中话术进行二次编辑 |
| 底部操作 | 按钮组 | 重新生成 / 保存为模板 / 复制并标记已回复 |

**话术卡片**

| 元素 | 类型 | 说明 |
|------|------|------|
| 风格标签 | 标签 | 亲切/幽默/专业 |
| 话术内容 | 文本 | 生成的回复话术内容 |
| 复制按钮 | 图标按钮 | 一键复制到剪贴板 |
| 编辑按钮 | 图标按钮 | 进入编辑模式 |
| 收藏按钮 | 图标按钮 | 保存为个人模板 |
| 质量反馈 | 图标按钮 | 👍/ 👎 对 AI 质量进行反馈 |

#### 5.4.4 话术生成 Prompt 设计

**亲切型 Prompt 模板：**
```
你是一位友好温暖的社交媒体创作者。请针对以下粉丝评论，生成一条亲切、温暖的回复。
要求：
- 语气亲切自然，像朋友之间的对话
- 适当使用语气词和表情符号
- 长度控制在20-100字
- 如果评论是正面的，表达感谢和互动意愿
- 如果评论是负面的，温和回应并表达改进意愿

评论内容：{comment_content}
评论情感：{sentiment}
创作领域：{content_category}
```

**幽默型 Prompt 模板：**
```
你是一位风趣幽默的社交媒体创作者。请针对以下粉丝评论，生成一条轻松有趣的回复。
要求：
- 适当使用网络流行语和梗
- 保持积极正面，不要冒犯他人
- 长度控制在20-100字
- 可以自嘲但不贬低自己

评论内容：{comment_content}
评论情感：{sentiment}
创作领域：{content_category}
```

**专业型 Prompt 模板：**

### 5.5 优先级排序

#### 5.5.1 功能概述

基于多维度因素对评论进行智能排序，帮助创作者优先处理最有价值的评论。

#### 5.5.2 排序算法

**优先级评分公式：**

```
Priority Score = W1 × 情感强度 + W2 × 粉丝价值 + W3 × 互动潜力 + W4 × 时效性
```

**权重分配：**

| 因子 | 权重 | 说明 |
|------|------|------|
| 情感强度 | 30% | 情感越强烈（无论正负），优先级越高 |
| 粉丝价值 | 25% | 评论者粉丝数、是否为创作者粉丝 |
| 互动潜力 | 25% | 评论是否容易引发二次互动 |
| 时效性 | 20% | 评论越新，优先级越高 |

**优先级标签规则：**

| 评分范围 | 标签 | 颜色 |
|----------|------|------|
| ≥ 80 | 🔥 高价值 | 红色 |
| 60-79 | ⚡ 待关注 | 橙色 |
| 40-59 | 📌 普通 | 蓝色 |
| < 40 | 普通 | 灰色 |

## 8. 业务规则与逻辑

### 8.1 评论同步规则

| 规则 | 说明 |
|------|------|
| 同步频率 | 每5分钟自动同步一次 |
| 同步范围 | 拉取最近24小时内的新评论 |

| 数据清洗 | 去除系统评论、广告评论、重复评论 |

### 8.2 额度计算规则

| 操作 | 消耗额度 | 说明 |

| 情感分析 | 0 | 随评论同步自动完成 |
| 生成话术 | 1条/次 | 每次生成消耗1条额度（不论生成几个候选） |
| 查看周报 | 0 | 不消耗额度 |

### 8.3 额度超限处理

| 场景 | 处理方式 |

### 8.4 内容安全规则

| 规则 | 说明 |

## 9. 异常与边界情况处理

### 9.1 网络异常

| 场景 | 处理方式 |
|------|----------|
| 评论同步时网络中断 | 本地缓存未同步数据，网络恢复后自动续传 |
| AI服务不可用 | 显示"AI服务暂时不可用，请稍后重试"，保留用户操作记录 |
| 平台API异常 | 标记该账号同步异常，通知用户检查授权 |

### 9.2 数据异常

| 场景 | 处理方式 |
|------|----------|
| 评论内容为空 | 跳过该评论，不入库 |
| 评论内容超长（>2000字） | 截断保存，标记"长评论" |
| 重复评论 | 基于平台+评论ID去重，忽略重复 |
| 平台账号解绑后重新绑定 | 重新拉取最新评论，历史数据与账号重新关联 |

### 9.3 业务异常

| 场景 | 处理方式 |
|------|----------|
| 话术生成失败 | 重试1次，仍失败则提示用户手动回复 |
| 情感分析置信度低 | 标记为"中性"，并在UI上提示"分析不确定" |
| 复制失败（浏览器限制） | 降级为选中文本，提示用户手动复制 |
| 并发操作冲突 | 乐观锁机制，提示"数据已被修改，请刷新" |
