# 1. 产品概述

## 1.1 产品定位

**小餐馆食材采购记录本**是一款面向 1-5 人小餐馆、快餐店、面馆、奶茶店的**轻量级采购记账 + 成本分析工具**。产品以微信小程序为主要载体，配合轻量 Web 后台，帮助不会用 Excel、不依赖复杂 ERP 的小餐饮经营者，用手机快速完成每日食材采购记账，自动生成成本报表，并在价格异常时及时提醒。

**一句话定位**：小餐馆老板的"食材记账本 + 成本分析助手"。

**差异化定位**：
- ✅ 纯记录工具，中立不绑定任何供应商
- ✅ 极简操作，30 秒完成一条采购记录
- ✅ 傻瓜式成本报表，看懂账、算清账
- ❌ 不做餐饮 ERP（无进销存、无库存管理、无菜品配方）
- ❌ 不做供应链平台（不对接美团快驴等 B2B 采购）
- ❌ 不做外卖/收银对接

## 1.2 目标用户

| 用户角色 | 画像描述 | 核心痛点 | 使用场景 |
|---------|---------|---------|---------|
| **店主/老板**（主要用户） | 30-50 岁，初中到大专学历，夫妻店或小餐馆老板，每天亲自采购或监督采购 | 靠记忆或纸笔记账，月底算不清成本；食材价格涨了不知道涨了多少；不知道哪个品类花钱最多 | 每天采购后记账、月底看成本报表、设置价格预警 |
| **采购员/厨师**（协作用户） | 20-45 岁，店内厨师或采购帮工，受店主委托执行采购 | 采购完需要向老板报账，微信发消息零散不方便 | 采购现场快速录入，拍照留痕 |
| **多店老板**（专业版用户） | 拥有 2-5 家同类门店的小连锁老板 | 无法实时掌握各门店成本情况，缺乏统一管控手段 | 跨店查看汇总数据、统一配置预警阈值 |

## 1.3 核心价值主张

1. **省时**：30 秒完成一条采购记录（语音录入 + 拍照留痕），替代纸笔和微信聊天记录
2. **省钱**：自动成本报表 + 价格异常预警，帮店主发现"钱花在哪里、哪里花多了"
3. **零门槛**：不用学 Excel，不用装 APP，微信扫码即用，界面像"记账本"一样简单
4. **中立安全**：不绑定供应商，数据云端加密，商业信息严格隔离

# 3. 功能详细描述

## 3.1 账户与登录模块

### 3.1.1 微信授权登录

**功能说明**：用户通过微信小程序一键授权登录，获取微信昵称和头像。

**交互说明**：
1. 用户首次打开小程序 → 弹出微信授权弹窗
2. 用户点击"允许" → 系统获取微信昵称/头像 → 自动完成登录
3. 首次登录后引导绑定手机号（见 3.1.2）

**业务规则**：
- 微信授权为 mandatory，拒绝授权则无法使用小程序
- 登录态有效期 7 天，过期需重新授权
- 同一微信号只能绑定一个店铺（作为店主）或多个店铺（作为协作成员）

**数据流向**：
```
用户点击授权 → 微信API获取code → 后端换取openid/session_key → 创建/匹配用户记录 → 返回登录态token
```

### 3.1.2 手机号绑定

**功能说明**：首次登录后引导绑定手机号，用于接收微信订阅消息及账号找回。

**交互说明**：
1. 首次登录成功后 → 跳转手机号绑定页
2. 用户点击"获取微信手机号"按钮（微信一键获取）或使用键盘输入手机号
3. 输入短信验证码（键盘输入场景）
4. 绑定成功 → 进入店铺创建/加入流程

**业务规则**：
- 手机号为 mandatory（用于接收价格预警订阅消息）
- 一个手机号可绑定多个店铺（作为不同角色）
- 手机号变更需在"我的"页面操作

### 3.1.3 创建店铺

**功能说明**：首次登录的店主引导创建店铺。

**交互说明**：
1. 绑定手机号后 → 进入"创建您的店铺"引导页
2. 填写：店名（必填）、店铺类型（选择：快餐/面馆/奶茶/早餐/正餐/其他）、地址（选填）、联系电话（默认绑定手机号）
3. 点击"创建完成" → 系统根据店铺类型推荐预置食材库 → 进入首页

**业务规则**：
- 一个用户只能创建一个店铺（作为店主）
- 店铺创建后，系统根据店铺类型自动启用对应的预置食材库
- 店铺创建者自动成为"店主"角色

### 3.1.4 邀请协作成员

**功能说明**：店主生成邀请码或小程序码，邀请采购员/厨师加入店铺。

**交互说明**：
1. 店主进入"我的" → "成员管理" → "邀请成员"
2. 展示店铺邀请码（6位数字+字母）和小程序码
3. 采购员/厨师打开小程序 → "加入店铺" → 输入邀请码或扫描小程序码
4. 店主收到通知 → 确认加入

**业务规则**：
- 邀请码有效期 7 天，过期自动刷新
- 单店最多 5 个协作成员（含店主共 6 人）
- 协作成员角色为"采购员"，只能录入采购记录，不能查看报表和设置

## 3.2 采购记录模块

### 3.2.1 手工录入

**功能说明**：用户手动填写采购信息。

**交互说明**：
1. 首页点击"记一笔"大按钮 → 进入录入页
2. 选择食材品类（从常用库选择或手动输入）
3. 填写数量、单位、单价
4. 选择供应商（可选，从常用供应商选择或"记一下"临时供应商）
5. 系统自动计算金额 = 数量 × 单价
6. 可选：拍照留痕
7. 点击"保存" → 系统检查价格是否触发预警 → 保存成功提示

**业务规则**：
- 金额 = 数量 × 单价，系统自动计算，不允许手动修改金额
- 食材品类为必填，支持从常用库选择或手动输入新品类
- 供应商为选填，可选择"记一下"作为临时供应商名称
- 保存后自动触发价格预警判断
- 支持离线录入（见 3.2.6）

### 3.2.2 语音录入

**功能说明**：用户通过语音输入采购信息，系统自动识别并解析。

**交互说明**：
1. 录入页点击"语音录入"大按钮（按钮占屏幕宽度 60%，适配湿手场景）
2. 长按说话（如"猪肉 5 斤 28 块一斤"），松手后系统开始识别
3. 识别完成后进入**两步模式**：
   - **Step 1 - 识别展示**：系统展示解析结果，关键字段（品类=猪肉、数量=5斤、单价=28元/斤）以大号加粗高亮字体展示
   - **Step 2 - 确认保存**：
     - 若识别置信度 > 95% 且用户未关闭自动保存：显示 5 秒倒计时自动保存，用户可点击"修改"中途修正
     - 若识别置信度 ≤ 95% 或用户已关闭自动保存：展示待确认表单，显示极大的"确认保存"绿色按钮（占屏幕宽度 80%，高度 60px），湿手可按
4. 保存后同手工录入流程（价格预警检查等）

**语音录入设置**（用户可在"设置"中调整）：
- 开关"高置信度自动保存"（默认开启）
- 切换"直接保存"模式（跳过确认步骤，依赖用户后续修正）

**业务规则**：
- 语音识别延迟 ≤ 2 秒（4G 网络环境）
- 识别结果必须经过用户确认或自动保存倒计时结束后才写入数据库
- 语音识别对接第三方服务（腾讯云/阿里云语音识别 API）
- 文本解析使用内部 NLP 服务，将自然语言解析为结构化字段

**数据流向**：
```
用户语音 → 上传音频流 → 语音识别API → 识别文本 → NLP解析 → 结构化字段（品类/数量/单价）→ 展示确认 → 保存
```

### 3.2.3 拍照留痕

**功能说明**：为采购记录拍照作为凭证。

**交互说明**：
1. 录入页或记录详情页点击"拍照"按钮
2. 选择"拍照"或"从相册选择"
3. 拍照后自动压缩（单张 ≤ 2MB）→ 预览 → 确认上传
4. 支持多张照片（最多 5 张）

**业务规则**：
- 照片为选填项
- 照片上传支持离线缓存，联网后自动同步
- 照片存储在对象存储服务（阿里云 OSS / 腾讯云 COS）

### 3.2.4 常用食材/供应商快捷选择

**功能说明**：常用食材和供应商置顶显示，减少输入。

**交互说明**：
1. 录入页品类选择区域，顶部展示"常用食材"（最多 10 个），点击即用
2. 供应商选择区域，顶部展示"常用供应商"（最多 5 个）
3. 常用食材基于用户历史录入频率自动排序

**业务规则**：
- 系统根据店铺类型预置推荐食材库
- 用户可自定义添加食材到常用库
- 常用食材排序可按使用频率自动排列

### 3.2.5 采购历史

**功能说明**：查看今日/历史采购记录。

**交互说明**：
1. 首页展示"今日采购列表"，按时间倒序排列
2. 每条记录显示：食材名称、数量、金额、供应商、异常标记（红色感叹号）
3. 点击记录 → 查看详情页（含照片、备注、异常标记详情）
4. 店主可修改/删除任意记录；协作角色仅可修改/删除自己录入的记录
5. 历史查询支持按日期范围、品类、供应商筛选

**业务规则**：
- 免费版仅可查看近 30 天记录
- 专业版可查看所有历史记录
- 修改/删除操作记录操作日志

### 3.2.6 离线录入与同步

**功能说明**：弱网/无网络环境下支持本地缓存录入，联网后自动同步。

**交互说明**：
1. 离线状态下录入采购 → 数据保存至本地缓存 → 页面提示"已离线保存，联网后自动同步"
2. 联网恢复后 → 本地缓存记录全量上传
3. 若检测到疑似重复记录 → 系统标记为"疑似重复" → 推送消息至消息中心
4. 用户进入消息中心 → 查看疑似重复记录对比 → 选择处理方式：
   - 保留全部（视为两条独立记录）
   - 删除重复项
   - 合并为一条

**业务规则**：
- **核心原则：采购记录 = 钱的流水，绝对不能丢数据**
- 离线期间的所有录入均视为独立记录，联网后全量同步，不自动覆盖或合并
- 疑似重复检测逻辑：相同食材 + 相同时间 + 相同金额 → 高置信度重复
- 冲突处理提示以消息形式推送至消息中心，不弹窗打断
- 同步延迟 ≤ 10 秒（联网后）

## 3.3 成本报表模块

### 3.3.1 今日成本日报

**功能说明**：展示今日食材成本汇总。

**交互说明**：
1. 进入"成本报表" → 默认展示今日日报
2. 页面展示：
   - 今日总成本（大号字体）
   - 与昨日对比（箭头 + 百分比变化）
   - 各品类成本占比（横向柱状图）

## 3.4 价格预警模块

### 3.4.1 品类默认阈值

**功能说明**：系统按食材品类预置差异化默认预警阈值。

**默认阈值配置**：

| 品类 | 默认预警阈值 | 设计理由 |
|------|-------------|---------|
| 蔬菜 | 30% | 受天气/季节影响大，叶菜类日波动可达 20-50% |
| 肉类（猪牛羊） | 15% | 单价高，15%涨幅意味着显著成本增加 |
| 禽蛋 | 15% | 季节性明显，波动中等 |
| 水产 | 20% | 受休渔期/天气影响，波动中等偏大 |
| 粮油 | 10% | 价格非常稳定，10%涨幅已是重大异常 |
| 调料干货 | 10% | 保质期长、价格极其稳定 |
| 饮料包材 | 10% | 工业品属性，价格最稳定 |

**业务规则**：
- 用户无需逐个配置即可开箱即用
- 用户可在"预警设置"中调整品类默认阈值
- 阈值含义：本次采购价 vs 近 30 天均价的涨幅百分比

### 3.4.2 单品阈值覆盖

**功能说明**：针对特定食材单独设置阈值，覆盖值优先于品类默认值。

**交互说明**：
1. 进入"预警设置" → "单品阈值"
2. 选择食材 → 设置自定义阈值（百分比或绝对值）
3. 保存后立即生效

**业务规则**：
- 单品覆盖值优先级高于品类默认值
- 支持按百分比（如 8%）或绝对值（如 ±0.5 元/斤）设置
- MVP 阶段支持单品覆盖；V1.1 引入基于历史数据的智能推荐

### 3.4.3 预警通知

**功能说明**：采购价超出阈值时自动推送提醒。

**交互说明**：
1. 采购员录入新记录 → 系统自动判断是否触发预警
2. 触发预警 → 生成预警消息
3. 通知通道：
   - **应用内通知**：推送至店主小程序消息中心
   - **微信订阅消息**：推送至店主微信（需用户授权一次性订阅）
4. 用户点击通知 → 查看预警详情

**微信订阅消息引导策略**（因一次性订阅机制）：
- 首次开启预警时引导订阅
- 每次预警推送后引导再次订阅
- "我的"页面显著位置展示订阅状态

### 3.4.4 预警详情

**功能说明**：查看预警食材的详细信息。

**交互说明**：
1. 预警详情页展示：
   - 本次采购价 vs 近 30 天均价
   - 涨幅百分比
   - 历史价格走势折线图（近 30 天）
   - 该食材的预警阈值设置
2. 用户可执行操作：
   - 调整阈值
   - 查看该食材在其他供应商的历史报价

| 离线数据同步 | ≤ 10 秒（联网后） |
| 预警推送 | ≤ 1 分钟（触发后） |
| 并发支持 | 单店 ≤ 5 协作角色同时操作，≤ 1000 店主账号并发 |

## 5.2 安全要求

| 需求项 | 说明 |
|--------|------|
| 数据加密 | 店主手机号等个人信息加密存储（AES-256） |
| 数据隔离 | 采购数据按店铺严格隔离，协作角色按权限隔离 |
| 传输安全 | 所有通信基于 HTTPS 协议 |
| 支付安全 | 微信支付接口安全对接 |
| 登录安全 | 登录态 7 天有效，过期重新授权 |

| 端 | 兼容范围 |
|----|---------|
| 小程序端 | 微信客户端 7.0+，适配主流手机尺寸（含小屏 iPhone SE） |
| Web端 | Chrome 90+、Edge 90+、Safari 14+、Firefox 90+，适配 1280px+ 桌面屏幕 |

## 5.4 界面设计要求

| 需求项 | 说明 |
|--------|------|
| 小程序端风格 | 清爽、朴实、贴近"记账本"质感，主色调暖色系（米黄+草绿），避免冰冷商务感 |
