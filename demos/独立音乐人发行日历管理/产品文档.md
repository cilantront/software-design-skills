# 1. 产品概述

## 1.1 产品定位

ReleaseCal 是一款面向独立音乐人、乐队、小型厂牌的**发行计划管理 SaaS 工具**。产品聚焦于"发行计划管理 + 多平台关键节点提醒 + 发行数据复盘"三大核心场景，帮助用户以专业化的方式统筹多平台发行全流程。

**不做的事：** 不做音乐发行/分发平台（不做 DistroKid 替代品）、不做音乐播放、不做社交。

## 1.2 产品愿景

让每一位独立音乐人都能像专业厂牌一样管理发行——不遗漏关键节点、不丢失平台数据、不依赖碎片化工具。

## 1.3 目标用户画像

| 画像 | 描述 | 核心痛点 | 使用场景 |
|------|------|----------|----------|
| **solo 音乐人 小张** | 25岁，独立唱作人，在网易云和 Spotify 发布原创音乐 | 用 Excel 记录各平台发行日期，经常忘记提前提交审核；发行后不知道数据表现如何 | 创建单曲发行计划 → 系统自动提醒关键节点 → 发行后录入数据查看复盘 |
| **乐队经纪人 阿明** | 28岁，乐队经纪人，负责管理5人乐队的发行事务 | 乐队成员各自记录发行信息，物料交接混乱；多平台发行节奏难以统筹 | 创建乐队工作空间 → 邀请成员协作 → 日历看板查看整体发行节奏 |
| **小型厂牌运营 Luna** | 30岁，独立音乐厂牌 A&R，管理旗下8位音乐人 | 同时管理多位音乐人的发行计划，需要全局视角；需要复盘数据优化厂牌发行策略 | 管理多位艺人发行计划 → 对比各歌曲数据 → 导出报告给团队 |

## 1.4 商业模式

| 套餐 | 价格 | 核心权益 |
|------|------|----------|
| **免费版** | ¥0 | 管理 ≤5 首歌曲、基础日历看板、站内提醒、手动数据录入、单曲复盘报告（含水印） |
| **专业版（月付）** | ¥29/月 | 不限歌曲数、多平台数据自动同步、邮件+微信提醒、团队协作（≤10人）、发行模板、无水印报告、多工作空间 |
| **专业版（年付）** | ¥269/年（≈¥22.4/月） | 同月付，享 23% 折扣 |

## 1.5 核心指标（North Star Metrics）

| 指标 | 目标（上线3个月） | 衡量方式 |
|------|-------------------|----------|
| 周活跃用户数（WAU） | ≥ 500 | 每周至少登录1次并查看日历 |
| 付费转化率 | ≥ 5% | 免费版用户中升级专业版的比例 |
| 发行计划创建数 | ≥ 2000 | 累计创建的发行计划总数 |
| 用户次月留存率 | ≥ 40% | 注册后次月仍活跃 |

# 2. 信息架构

│   ├── 专辑报告
│   ├── 对比分析
│   └── 数据导入

# 3. 功能规格说明

## 3.1 仪表盘（Dashboard）

### 3.1.1 页面目标
用户登录后首屏，快速掌握今日待办、近期发行、关键提醒。

### 3.1.2 页面元素

| 区块 | 元素 | 说明 |
|------|------|------|
| **顶部欢迎区** | 用户头像 + 问候语 + 套餐标签 | "下午好，小张 · 免费版" |
| **本月概览卡片** | 本月发行数 / 即将到期节点数 / 本月新增播放量 | 三列卡片式布局 |
| **今日待办** | 今日需确认的里程碑节点列表 | 点击跳转至对应发行计划 |
| **近期发行** | 未来14天内即将发行的计划卡片 | 显示封面缩略图、歌曲名、发行日期、目标平台图标 |
| **快速入口** | "新建发行计划"按钮 + "导入数据"按钮 | 突出主操作 |
| **使用量提示** | 免费版显示"已用 X/5 首"进度条 + 升级引导 | 达到上限时红色警示 |

### 3.1.3 交互规则
- 页面加载时自动刷新数据
- "近期发行"卡片按发行日期升序排列
- 点击卡片跳转至对应发行计划详情页
- 无发行计划时显示空状态引导（"创建你的第一个发行计划"）

### 3.2.2 新建发行计划（分步向导）

#### 步骤设计

| 步骤 | 标题 | 表单字段 | 校验规则 |
|------|------|----------|----------|
| **Step 1** | 基本信息 | 发行类型（单曲/专辑/EP）、歌曲名称、艺人名称、流派（下拉多选）、语言、时长、作词、作曲、编曲 | 歌曲名称必填，≤100字符 |
| **Step 2** | 平台与日期 | 目标平台（多选：网易云/QQ音乐/Spotify/Apple Music/Amazon Music/YouTube Music/Bandcamp）、各平台发行日期 | 至少选择1个平台；发行日期不早于今天+14天 |
| **Step 3** | 编码与物料 | ISRC 编码、UPC/EAN 编码（专辑时必填）、封面图片上传 | ISRC 格式校验（2字符国家码+5字符注册码+2字符年份+3字符序号）；封面 ≥3000×3000px |
| **Step 4** | 确认创建 | 汇总展示所有已填信息 | 可返回修改任一步骤 |

#### 交互规则
- 顶部进度条显示当前步骤（1/4 → 2/4 → 3/4 → 4/4）
- "上一步""下一步"按钮 + 键盘左右箭头切换
- 支持"保存草稿"随时退出
- Step 2 选择平台后，自动展示各平台发行规则提示（如"Spotify 建议提前 4 周提交"）
- 专辑类型时 Step 1 增加"曲目列表管理"子表单（添加曲目、排序、每首独立 ISRC）
- 专业版用户可在 Step 1 选择"使用发行模板"快速填充

## 3.4 里程碑与提醒

### 3.4.1 里程碑自动生成规则

| 节点 | 计算公式 | 说明 |
|------|----------|------|
| 上传母带截止 | 最早平台发行日 - 14天 | 确保有足够时间上传到所有平台 |
| 提交平台审核 | 最早平台发行日 - 7天 | 各平台审核周期约 1-5 天 |
| 预热宣传启动 | 最早平台发行日 - 3天 | 开始社交媒体预热 |
| 正式发行日 | 各平台设定的发行日期 | 可能有多个日期（分平台发行） |
| 首周数据复盘 | 最早平台发行日 + 7天 | 录入首周数据并生成报告 |

### 3.4.2 提醒策略

| 提醒类型 | 触发条件 | 提醒方式 | 备注 |
|----------|----------|----------|------|
| 常规提醒 | 到达用户设置的提醒提前量 | 站内通知 | 所有用户 |
| 邮件提醒 | 到达提醒时间 | 邮件 | 专业版 |
| 微信提醒 | 到达提醒时间 | 微信服务号模板消息 | 专业版，需绑定微信 |
| 升级提醒 | 里程碑到期未确认 | 站内+邮件+微信同时 | 到期当天每小时一次，逾期每天一次 |

### 3.4.3 提醒中心页面

| 区块 | 元素 | 说明 |
|------|------|------|
| **筛选** | 全部 / 今日 / 本周 / 已逾期 / 已确认 | Tab 切换 |
| **提醒列表** | 每条提醒显示：里程碑名称、关联歌曲名、计划时间、状态标签（待处理/已提醒/已确认/逾期）、操作按钮（确认完成/延后） | 逾期项红色高亮 |
| **批量操作** | "全部确认"按钮 | 一键确认所有已完成的提醒 |

#### 交互规则
- 数据摘要卡片显示累计值 + 较上周环比变化（↑/↓ + 百分比）
- 图表支持 hover 查看具体数值（tooltip）
- 平台对比图支持切换指标（播放量/收藏量/收入）
- 阶段数据卡片点击展开详细数据表
- 导出 PDF 免费版带"ReleaseCal 免费版"水印
- 支持选择时间范围查看趋势（发行后7天/30天/90天/全部）

### 3.5.2 对比分析页

#### 功能说明
- 选择 2-5 首歌曲进行横向对比
- 对比维度：播放量、收入、收藏量、增长率
- 可视化方式：多折线对比图 + 数据表格
- 支持与历史发行数据纵向对比（当前 vs 上一首）

### 3.5.3 数据导入

#### 功能说明
- 提供标准 Excel/CSV 导入模板下载
- 模板包含字段：日期、平台、播放量、收藏量、评论数、分享数、下载量、收入
- 上传后系统自动校验数据格式，展示校验结果（成功/失败行数）
- 支持预览导入数据，确认后写入

# 5. 数据模型（核心实体）

## 5.1 实体关系

```
用户(User) ──1:N── 工作空间(Workspace)
工作空间 ──1:N── 发行计划(ReleasePlan)
发行计划 ──1:N── 平台发行记录(PlatformRelease)
发行计划 ──1:N── 里程碑节点(Milestone)
发行计划 ──1:N── 物料(Asset)
发行计划 ──1:N── 平台数据(PlatformData)
工作空间 ──1:N── 成员(WorkspaceMember)
```

## 5.2 核心字段说明

### 发行计划（ReleasePlan）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| workspace_id | UUID | 所属工作空间 |
| type | Enum | 类型：single/album/ep |
| title | String | 歌曲/专辑名称 |
| artist_name | String | 艺人名称 |
| genre | String[] | 流派标签 |
| language | String | 语言 |
| duration | Integer | 时长（秒） |
| isrc | String | ISRC 编码 |
| upc | String | UPC/EAN 编码（专辑） |
| cover_url | String | 封面图片 URL |
| status | Enum | 状态：draft/preparing/pending_review/scheduled/released/archived |
| release_date | Date | 最早发行日期 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 平台发行记录（PlatformRelease）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| release_plan_id | UUID | 关联发行计划 |
| platform | Enum | 平台标识 |
| release_date | Date | 该平台发行日期 |
| platform_url | String | 平台链接 |
| status | Enum | 状态：not_submitted/reviewing/approved/rejected/live |
| isrc | String | 平台对应的 ISRC（可能不同） |

### 里程碑节点（Milestone）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| release_plan_id | UUID | 关联发行计划 |
| name | String | 节点名称 |
| due_date | Date | 节点日期 |
| status | Enum | 待处理/已提醒/已确认/逾期未确认 |
| is_auto | Boolean | 是否系统自动生成 |
| confirmed_at | DateTime | 确认完成时间 |
