## 1. 文档概述

### 1.1 文档目的

本文档为「交付前资料风险确认台」产品的用户需求说明书，旨在明确产品的目标用户、使用场景、核心功能及优先级排序，为后续产品文档（PRD）和 UI 原型设计提供依据。

### 1.2 产品定位

「交付前资料风险确认台」是一款面向独立顾问、自由职业者及小型 B2B 服务商的**轻量级交付前资料协同工具**，聚焦于"交付前资料完整性确认"和"资料有效期管理"两大核心痛点，帮助用户在服务正式交付前高效识别并消除资料缺失/过期风险。

### 1.3 差异化价值

| 对比维度 | 泛用资料收集工具 | 本工具 |
|---------|--------------|--------|
| 关注点 | 资料收集本身 | 交付前的风险识别与确认 |
| 有效期管理 | 无/手动 | 自动识别过期资料并提醒 |
| 协同模式 | 通用表单 | 面向客户-服务商的轻量协同 |
| 风险可视化 | 无 | 风险看板 + 一键导出 |

## 2. 目标用户

### 2.1 用户画像

| 用户类型 | 典型特征 | 核心痛点 |
|---------|---------|---------|
| **独立顾问** | 同时服务 3-10 个客户，单人运营 | 资料催收耗时、容易遗漏导致交付延期 |
| **自由职业者** | 按项目制工作，客户更换频繁 | 不清楚每个项目还缺什么资料，依赖记忆 |
| **小型 B2B 服务商** | 2-10 人团队，标准化服务流程 | 团队成员不清楚客户资料状态，交接混乱 |

### 2.2 用户规模预估

- 免费版：支持最多 3 个活跃客户
- 专业版：¥39/月，不限客户数；可按团队席位加收

## 3. 用户场景

### 场景 1：新客户启动 — 生成资料提交链接与必填清单

**角色**：独立顾问（王先生，管理咨询顾问）  
**背景**：王先生签约了一个新客户，需要客户提供营业执照、财务报表、组织架构图等 8 项资料才能启动项目。  
**当前做法**：通过微信/邮件逐一发送资料清单，反复催促，容易遗漏。  
**期望体验**：
1. 王先生在系统中创建一个新客户项目
2. 从模板库选择"管理咨询-启动阶段"资料清单模板（或自定义）
3. 系统自动生成一个专属的资料提交链接
4. 客户打开链接，看到清晰的待提交资料列表，可逐项上传
5. 王先生在看板上实时看到哪些已提交、哪些未提交

**验收标准**：
- 创建客户到生成链接 ≤ 3 步操作
- 客户无需注册即可查看和提交资料
- 每项资料可设置"必填/选填"和"有效期"

---

### 场景 2：资料催收 — 自动识别缺失资料并发送温和提醒

**角色**：自由职业者（李女士，品牌设计师）  
**背景**：李女士有 5 个客户同时在推进，其中 2 个客户的资料已经 3 天未提交完整。  
**当前做法**：手动检查每个客户的资料状态，逐一发微信催促。  
**期望体验**：
1. 系统自动检测到 2 个客户有缺失资料超过设定阈值（如 48 小时）
2. 自动生成温和的提醒消息（可自定义模板）
3. 李女士确认后，系统通过邮件/短信发送提醒
4. 提醒中包含一键提交链接，客户点击即可直达缺失项

**验收标准**：
- 自动检测频率可配置（每 12/24/48 小时）
- 提醒消息可预览、编辑后再发送
- 支持批量提醒多个缺失项

---

### 场景 3：资料有效期管理 — 自动识别过期资料

**角色**：小型 B2B 服务商（张总的合规咨询公司）  
**背景**：服务的客户需要定期提供有效期文件（如营业执照年检、保险单、资质证书），这些文件有明确的到期日。  
**当前做法**：用 Excel 记录到期日，手动检查，经常出现遗漏。  
**期望体验**：
1. 为每项资料类型设置有效期规则（如"营业执照-1年"、"保险单-到期日前30天提醒"）
2. 系统自动追踪每份已提交资料的到期日
3. 到期前自动发送续期提醒（提前 30/15/7 天）
4. 过期资料在看板上标记为红色风险项

**验收标准**：
- 支持按资料类型设置不同的有效期规则
- 过期/即将过期资料在看板上有明确视觉区分
- 提醒时间可自定义配置

---

### 场景 4：交付前风险汇总 — 导出风险报告

**角色**：独立顾问（王先生）  
**背景**：项目即将进入交付阶段，王先生需要向团队和客户确认所有资料齐备。  
**当前做法**：逐个检查客户资料，手动汇总哪些还缺失/过期，写邮件沟通。  
**期望体验**：
1. 在风险看板上看到所有客户的资料完成状态
2. 一键生成"交付前风险报告"，列出：
   - 各客户资料完成率
   - 缺失/过期资料明细
   - 风险等级评估（高/中/低）
3. 导出为 PDF/Excel，可直接发送给团队或客户

**验收标准**：
- 报告包含所有客户的汇总视图
- 支持 PDF 和 Excel 两种导出格式
- 风险等级计算规则可配置

---

### 场景 5：团队协作 — 多人管理客户资料

**角色**：小型 B2B 服务商团队成员（赵女士，项目经理）  
**背景**：团队有 3 人，分别负责不同客户，需要互相了解客户的资料状态。  
**当前做法**：在共享文件夹里管理，版本混乱，不知道谁在跟进哪个客户。  
**期望体验**：
1. 团队管理员邀请成员加入工作空间
2. 为每个客户分配负责人
3. 团队成员可看到自己负责的客户资料状态
4. 管理员可看到所有客户的汇总视图
5. 资料状态变更时通知相关负责人

**验收标准**：
- 支持邀请成员（通过邮件链接）
- 支持按客户分配负责人
- 角色权限区分（管理员/成员）

## 6. 非功能需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | ≤ 2 秒（首屏） |
| 文件上传速度 | 受带宽限制，UI 需展示进度 |
| 并发用户数 | MVP 阶段支持 100 并发 |
| 提醒发送延迟 | ≤ 5 分钟 |

### 6.2 安全要求

| 要求 | 说明 |
|------|------|
| 传输加密 | 全站 HTTPS |
| 文件存储 | 加密存储，访问需鉴权 |
| 提交链接 | 唯一 URL + 可选密码保护 |
| 数据备份 | 每日自动备份 |

### 6.3 可用性要求

## 7. 约束与假设

### 7.1 约束

- MVP 开发周期：1-2 周
- 技术栈需支持快速迭代
- 免费版限制 3 个活跃客户
- 专业版定价 ¥39/月

### 7.2 假设

- 目标用户具备基本的互联网使用能力
- 客户愿意通过链接提交资料（无需注册）
- 文件上传大小限制：单文件 ≤ 50MB
- 邮件送达率依赖第三方邮件服务商
