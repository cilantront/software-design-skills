需求文档生成 Skill

第一阶段 · 需求分析

从零散的会议记录、邮件、聊天记录中智能提取需求信息，自动去重、分类、评估优先级，输出符合标准格式的需求文档。不涉及企业核心知识，适合开源推广。

  • ✦智能挖掘：从非结构化文本中提取有效需求
  • ✦优先级评估：多维度自动评分，辅助排期决策
  • ✦变更追踪：记录需求演进历史，影响范围可追溯
  • ✦多格式导出：Markdown / Word / PDF 无缝对接

PRD文档及原型生成 Skill

第二阶段 · 产品定义

基于已生成的需求文档，自动输出包含产品概述、功能需求、交互设计、数据需求等完整章节的标准 PRD 文档，并同步生成产品原型说明文档，可直接用于设计团队对接。同样不涉及企业核心知识，适合开源推广。

  • ✦标准 PRD 生成：覆盖行业通用格式全部章节
  • ✦原型说明同步输出：页面结构、交互流程、组件规范
  • ✦需求双向追溯：PRD 每一条功能点关联原始需求
  • ✦模板自定义：内置多套标准模板，也支持导入团队模板

常见问题

两款 Skill 是免费的吗？ +
是的，两款 Skill 均采用开源推广策略，SKILL.md 完整公开，你可以自由安装和使用基础功能。
必须先安装需求文档 Skill，才能用 PRD Skill 吗？ +
不是必须的。如果你已经有结构化的需求文档（无论用什么工具生成的），可以直接传入 PRD 文档及原型生成 Skill。但两款 Skill 组合使用效果最佳：需求文档 Skill 负责"把需求整理好"，PRD Skill 负责"把 PRD 写专业"，形成完整闭环。
生成一份文档大概需要多久？ +
生成时间与需求的复杂度和原型页面数量有关，无法用固定秒数衡量。需求条数越多、原型页面定义越详细，生成时间相应增加。两款 Skill 均通过 CLI 调用，生成过程中的思考步骤会实时返回，你可以随时查看进度，不必盲目等待。
支持哪些 Agent 平台？ +
支持所有兼容 Agent Skills 规范的 Agent 平台，如 Codex、Claude、Cursor、OpenCode、GitHub Copilot、Windsurf、Gemini、Cline、Antigravity、ClawBot、Trae、Coze 等。
方案文档 Skill 什么时候发布？ +
方案文档 Skill 采用影子SKILL技术方案，核心分析逻辑通过 CLI 云端运行实现原文隔离保护，技术实现更复杂，预计于 2026 年 Q3 发布。
