# 1. 产品定位

## 1.1 产品愿景

**将客服噪音转化为产品决策输入**——帮助小团队以最低成本，从海量客服反馈中提取有价值的产品需求，驱动数据化产品迭代。

## 1.2 产品定义

客服反馈转产品需求卡AI是一款面向SaaS客服主管、产品运营和售后团队负责人的AI应用工具。系统通过导入客服对话记录和工单数据，自动完成问题分类、场景提取、情绪分析和需求卡片生成，输出可直接进入产品评审流程的结构化需求卡片及周报。

## 1.3 核心价值主张

| 维度 | 传统方式 | 本产品方案 |
|------|----------|------------|
| 需求收集 | 人工翻阅工单，逐条整理 | AI自动分类，批量生成需求卡片 |
| 需求质量 | 依赖个人经验判断 | 标准化场景、频次、影响面评估 |
| 问题发现 | 滞后、零散 | 实时识别高频问题和情绪升高趋势 |
| 决策支撑 | 主观汇报 | 数据化周报，可直接用于评审 |

## 1.4 产品差异化

- **不做通用客服质检**：区别于传统的客服质量监控系统
- **专注转化**：核心价值在于"客服噪音→产品决策"的转化链路
- **面向小团队**：轻量化、低成本、快速上手
- **MVP周期短**：1-2周可交付最小可用版本

## 1.5 目标用户画像

### 画像A：客服主管（李经理）
- **背景**：管理5-10人客服团队，每天处理200+工单
- **痛点**：工单堆积，无法有效提取产品改进信息
- **期望**：快速识别高频问题，向上级汇报有据可依

### 画像B：产品运营（张运营）
- **背景**：负责产品迭代规划，需收集用户需求
- **痛点**：需求来源分散，整理成本高，容易遗漏
- **期望**：结构化的需求卡片，可直接用于评审会

### 画像C：售后团队负责人（王主管）
- **背景**：关注售后服务质量和客户满意度
- **痛点**：无法量化售后反馈中的产品问题
- **期望**：趋势可视化，提前预警风险

## 2.2 P0 - 数据导入模块

### 2.2.1 CSV文件上传

| 功能项 | 描述 |
|--------|------|
| 文件选择 | 支持.csv格式，单文件最大10MB |
| 拖拽上传 | 支持拖拽文件到上传区域，提升交互体验 |
| 格式模板 | 提供CSV格式说明及示例模板下载 |

**CSV文件格式要求：**

| 列名 | 必填 | 说明 |
|------|------|------|
| ticket_id | 是 | 工单唯一标识 |
| content | 是 | 工单内容/对话文本 |
| customer_id | 否 | 客户标识 |
| created_at | 否 | 工单创建时间 |
| channel | 否 | 来源渠道 |

### 2.2.2 数据校验

| 功能项 | 描述 |
|--------|------|
| 格式检查 | 自动检查列名、数据类型是否符合要求 |
| 数据预览 | 上传后展示前5行数据供用户确认 |
| 错误提示 | 格式不符时给出明确的错误定位和修正建议 |

### 2.2.3 导入处理

| 功能项 | 描述 |
|--------|------|
| 开始导入 | 用户确认后启动导入+AI分类流程 |
| 进度展示 | 实时显示处理进度（已处理/总数） |
| 结果报告 | 显示成功/失败数量及失败原因 |

### 验收标准

- [ ] 支持拖拽和点击上传CSV文件
- [ ] 文件格式错误时有明确提示
- [ ] 数据预览正确展示前5行

## 2.3 P0 - AI分类引擎模块

### 2.3.1 问题分类

系统通过AI模型对工单内容进行自动分类，支持以下类别：

| 分类 | 识别特征 | 示例 |
|------|----------|------|
| 功能问题 | 功能缺陷、改进建议 | "这个功能报错了"、"希望增加批量导出" |
| 使用障碍 | 操作困难、找不到入口 | "不会操作"、"找不到设置入口" |
| 价格异议 | 价格、计费相关 | "太贵了"、"计费方式不合理" |
| 其他 | 咨询、建议、投诉 | 不属于以上分类的反馈 |

**分类准确率要求：** 首版目标≥80%，持续优化至≥90%

### 2.3.2 场景提取

| 功能项 | 描述 |
|--------|------|
| 场景描述 | 从反馈中自动生成用户使用场景描述 |
| 频次统计 | 统计同类反馈出现的次数 |
| 影响面评估 | 基于反馈内容和客户数评估影响范围 |

### 2.3.3 情绪分析

| 功能项 | 描述 |
|--------|------|
| 情绪识别 | 正面/中性/负面三分类 |
| 情绪升高检测 | 检测同一主题下情绪逐渐升高的趋势 |
| 风险预警 | 情绪升高时标记为潜在风险 |

### 验收标准

- [ ] 分类准确率≥80%（基于标注测试集）
- [ ] 1000条工单分类处理时间<5分钟
- [ ] 情绪识别覆盖率≥95%
- [ ] 高频问题自动聚合，重复反馈正确归并

# 3. 技术约束

## 3.1 技术架构

```
┌─────────────────────────────────────┐
│            Web前端（SPA）            │
│    HTML5 + CSS3 + JavaScript        │
│    响应式设计，支持Chrome/Firefox/Edge │
└──────────────┬──────────────────────┘
               │ HTTPS
┌──────────────▼──────────────────────┐
│            后端服务                   │
│    RESTful API                      │
│    文件处理 / 用户认证 / 数据管理     │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│          AI分类引擎                  │
│    LLM API（问题分类/场景提取/情绪分析）│
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│           数据存储                   │
│    关系型数据库 + 文件存储            │
└─────────────────────────────────────┘
```

## 3.2 技术约束清单

| 约束项 | 要求 |
|--------|------|
| 部署方式 | 云端部署，支持HTTPS访问 |
| 浏览器兼容 | Chrome 90+、Firefox 88+、Edge 90+ |
| 分辨率适配 | 1920x1080及以上 |
| 文件大小限制 | 单文件≤10MB |
| 并发支持 | 50用户同时在线 |
| 数据安全 | 传输加密（HTTPS），存储隔离 |
| AI服务 | 依赖外部LLM API，需考虑降级策略 |

## 3.3 性能指标

| 指标 | 目标值 |
|------|--------|
| 首屏加载 | <3秒 |
| 文件上传（<10MB） | <30秒 |
| AI分类（1000条） | <5分钟 |
| 页面交互响应 | <500ms |
| 搜索响应 | <1秒 |

## 3.4 安全要求

- 用户密码加密存储（bcrypt）
- API请求需携带有效Token
- 文件上传做类型和大小校验
- 防止SQL注入和XSS攻击
- 敏感操作需二次确认

# 7. 附录

## 7.1 需求卡片状态流转

```
[AI生成] → 待评审 → 已评审 → 已排期 → 已完成
                ↘ 已拒绝
```

## 7.2 优先级计算规则

优先级 = 频次权重 × 0.4 + 影响面权重 × 0.35 + 情绪权重 × 0.25

| 因子 | 权重说明 |
|------|----------|
| 频次 | 出现次数归一化到0-100 |
| 影响面 | 影响用户数归一化到0-100 |
| 情绪 | 负面=100，中性=50，正面=0 |
