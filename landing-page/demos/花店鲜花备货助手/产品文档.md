# 1. 产品概述与定位

## 1.1 产品愿景

**让每一家小花店都能从"凭经验备货"走向"数据驱动决策"，让每一支鲜花都不被浪费。**

## 1.2 产品定位

| 维度 | 描述 |
| --- | --- |
| **目标市场** | 鲜花零售行业，聚焦1-5人规模的独立花店/社区花店 |
| **产品形态** | 微信小程序（一线操作端）+ Web管理后台（数据分析端） |
| **核心价值** | 降低花材损耗率、精准节日备货、科学花束定价 |
| **差异化壁垒** | 不是通用库存系统，而是聚焦"鲜花损耗+节日备货+花束成本"三大鲜花行业专属场景 |
| **竞品区隔** | 避开企业级ERP/通用库存系统，以轻量、行业专属、低门槛切入 |

## 1.3 目标用户画像

### 用户角色A：花店店主/经营者（核心用户）

| 属性 | 描述 |
| --- | --- |
| **典型画像** | 28-45岁，经营1-5人社区花店，有3年以上花店经营经验 |
| **核心痛点** | ① 鲜花损耗率高达20%-40%，不知道损耗出在哪里 ② 节日备货全凭经验，经常滞销或缺货 ③ 花束定价靠感觉，不清楚真实成本 |
| **核心需求** | 实时掌握损耗情况、科学备货建议、精准成本核算 |
| **使用场景** | 日常采购决策、节日备货规划、经营数据分析、定价调整 |
| **使用频率** | 每日使用，节日前一周高频使用 |

### 用户角色B：花艺师

| 属性 | 描述 |
| --- | --- |
| **典型画像** | 有花艺专业技能，负责花束制作和设计 |
| **核心痛点** | 花束成本计算繁琐，每次花材价格变动都要重新算 |
| **核心需求** | 快速核算花束成本、获取定价建议、管理花束配方 |
| **使用场景** | 新品花束开发、日常花束制作、价格调整 |

### 用户角色C：店员/助手

| 属性 | 描述 |
| --- | --- |
| **典型画像** | 花店兼职或全职员工，负责日常花材管理 |
| **核心痛点** | 出入库登记繁琐，损耗记录不规范 |
| **核心需求** | 快速完成出入库和损耗登记，减少操作步骤 |
| **使用场景** | 每日花材入库、销售出库、损耗登记、保鲜期检查 |

## 1.4 核心业务流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  花材建档    │───▶│  采购入库    │───▶│  日常销售    │
│  (首次/新增) │    │  (批次+保鲜) │    │  (出库登记)  │
└─────────────┘    └─────────────┘    └─────────────┘
                           │                   │
                           ▼                   ▼
                    ┌─────────────┐    ┌─────────────┐
                    │  保鲜期监控  │    │  损耗登记    │
                    │  (到期预警)  │    │  (原因+拍照) │
                    └─────────────┘    └─────────────┘
                           │                   │
                           ▼                   ▼
                    ┌─────────────┐    ┌─────────────┐
                    │  节日备货    │    │  损耗分析    │
                    │  (预测+建议) │    │  (报表+趋势) │
                    └─────────────┘    └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  花束成本    │
                    │  (核算+定价) │
                    └─────────────┘
```

## 1.5 商业指标

| 指标 | 目标值 |
| --- | --- |
| 花材损耗率降低 | 使用后3个月内降低5-10个百分点 |
| 节日备货准确率 | 从约50%提升至75%以上 |
| 花束定价偏差 | 成本核算准确率达95%以上 |
| 用户留存率 | 月活留存率 > 60% |
| 付费转化率 | 免费版→花店版转化率 > 15% |

# 3. 功能模块详细PRD

## 3.1 花材管理模块

### 3.1.1 花材分类管理

**用户故事：** 作为花店店主，我希望能够按照鲜花的种类（玫瑰/百合/康乃馨/配花/配草等）对花材进行分类管理，以便快速查找和管理花材。

**功能描述：**
- 系统预设常见鲜花分类模板：玫瑰、百合、康乃馨、郁金香、向日葵、配花、配草、包装资材等
- 支持用户自定义分类，包括分类名称和排序
- 分类支持二级结构（如：玫瑰→红玫瑰/粉玫瑰/白玫瑰/黄玫瑰）

**验收标准：**
1. 首次进入系统时，系统自动加载预设分类模板
2. 用户可新增分类，输入分类名称后保存成功
3. 用户可编辑分类名称和排序
4. 分类下有花材档案时，不可删除该分类（提示"请先移除分类下的花材"）
5. 分类列表按自定义排序展示

**业务规则：**
- BR-001：预设分类不可删除，但可修改名称
- BR-002：分类名称不可重复（同一层级下）
- BR-003：分类最大层级为2级
- BR-004：每个分类可设置一个图标（系统提供鲜花行业常见图标库）

### 3.1.2 花材信息录入

**用户故事：** 作为花店店主，我希望录入每种花材的基本信息（名称、分类、单位、保鲜期、参考进价），以便在出入库和成本核算时使用。

**功能描述：**
- 录入字段：花材名称（必填）、所属分类（必填）、计量单位（支/扎/捆/朵/盆，必填）、默认保鲜期天数（必填）、参考进价（选填）、备注（选填）
- 支持从历史花材复制创建新花材
- 录入时自动检查是否已有同名花材

**验收标准：**
1. 必填字段未填写时，提交按钮置灰并提示
2. 花材名称超过50字符时截断提示
3. 保鲜期天数必须为正整数，范围1-90天
4. 参考进价支持2位小数
5. 保存成功后跳转到花材档案列表
6. 免费版创建第31种花材时，弹出升级提示

**业务规则：**
- BR-005：同一店铺下花材名称不可重复
- BR-006：免费版最多创建30种花材档案
- BR-007：花材档案删除为软删除，已关联出入库/损耗记录的花材不可删除（可标记为"停用"）
- BR-008：计量单位一旦有出入库记录后不可修改

### 3.1.3 花材档案列表

**用户故事：** 作为花店店主，我希望查看所有花材的列表，并按分类或名称搜索筛选，以便快速找到需要操作的花材。

**功能描述：**
- 列表展示：花材名称、分类、当前库存量、保鲜状态（正常/临期/过期）、最近进价
- 支持按分类筛选（下拉选择）
- 支持按名称搜索（模糊匹配）
- 支持按状态筛选：全部/有库存/无库存/临期

**验收标准：**
1. 列表默认按最近入库时间倒序排列
2. 搜索结果实时过滤（输入即搜）
3. 列表支持下拉刷新
4. 每个花材条目点击进入花材详情页
5. 免费版在列表底部显示"已使用 N/30 种花材"

**业务规则：**
- BR-009：保鲜状态判断——距保鲜到期日≤2天标记为"临期"（橙色），已过期标记为"过期"（红色），其他为"正常"（绿色）
- BR-010：库存量为0的花材默认排在列表末尾

## 3.2 出入库管理模块

### 3.2.1 采购入库登记

**用户故事：** 作为花店店主，我希望在采购花材到货后，快速登记入库信息（批次号、数量、单价、到货日期），系统能自动计算保鲜到期日，以便我掌握每批花材的保鲜状态。

**功能描述：**
- 选择已有花材或新建花材
- 录入：批次号（自动生成，支持手动修改）、数量（必填）、单价（元，必填）、到货日期（默认今天，可选过去7天内日期）
- 系统根据花材的默认保鲜期自动计算保鲜到期日
- 入库成功后自动更新花材库存量

**验收标准：**
1. 批次号自动生成格式为：RK+日期+序号（如：RK20260629001）
2. 到货日期默认今天，可选择过去7天内任意日期
3. 保鲜到期日 = 到货日期 + 花材默认保鲜期天数，自动计算并展示
4. 入库数量必须为正整数
5. 单价支持2位小数
6. 保存成功后显示入库结果摘要（花材名、批次、数量、保鲜到期日）
7. 入库成功后花材当前库存量实时更新

**业务规则：**
- BR-011：同一花材可有多个批次同时存在，按先进先出（FIFO）原则出库
- BR-012：入库记录一旦保存，数量和到货日期不可修改（可作废后重新录入）
- BR-013：到货日期不可晚于今天
- BR-014：入库操作记录操作人和操作时间

### 3.2.2 销售出库登记

**用户故事：** 作为店员，我希望在花卉售出或使用时快速登记出库信息，选择出库数量并指定用途（销售/损耗/赠送/自用），以便准确记录花材流向。

**功能描述：**
- 选择花材（展示当前有库存的花材列表）
- 系统自动选择最早到期的批次（FIFO）
- 录入：出库数量（必填）、出库用途（销售/损耗/赠送/自用，必填，默认"销售"）
- 若用途为"损耗"，需额外选择损耗原因
- 出库成功后自动更新库存量

**验收标准：**
1. 出库数量不能超过当前可用库存
2. 出库后库存量实时更新
3. 若选择的批次库存不足，提示"该批次库存不足，是否从下一批次继续出库？"
4. 出库记录可追溯批次信息
5. 用途为"损耗"时，联动创建损耗记录

**业务规则：**
- BR-015：出库遵循先进先出（FIFO）原则，优先出最早到期的批次
- BR-016：出库记录一旦保存不可修改（可作废后重新录入）
- BR-017：用途为"损耗"时，必须选择损耗原因
- BR-018：出库操作自动关联操作人和时间

### 3.2.3 入库记录查询

**用户故事：** 作为花店店主，我希望查看历史入库记录，并能按花材、日期、批次进行筛选，以便追踪采购情况和核对账目。

**功能描述：**

**功能描述：**
- 系统自动分析去年同期各花材的销量和损耗率
- 结合年度增长率（基于近3个月销量趋势）进行修正

- 扣除当前可用库存，生成各花材的备货建议数量

### 3.5.2 自动成本计算

**用户故事：** 作为花艺师，我希望系统能根据花束配方中的花材用量和最近采购均价，自动计算每束花的成本，以便快速了解花束的实际成本。

**功能描述：**
- 系统自动获取配方中每种花材的最近30天采购均价
- 按用量计算各花材成本
- 汇总计算花束总成本
- 成本明细展示：花材名称、用量、单价、小计

**验收标准：**
1. 采购均价取最近30天该花材所有入库记录的加权平均单价
2. 若花材无采购记录，标记"无价格数据"，不计入成本
3. 成本计算结果实时展示
4. 花材进价变动后，配方成本自动更新

**业务规则：**
- BR-044：加权平均单价 = SUM(数量×单价) / SUM(数量)，取最近30天数据
- BR-045：若无近30天采购记录，取最近一次入库单价
- BR-046：若无任何采购记录，该花材成本记为0并标记提示
- BR-047：成本计算为花店版功能

**功能描述：**
- 定价公式：建议售价 = 花束成本 × (1 + 损耗率) × (1 + 目标利润率)
- 默认损耗率取该花束主要花材的近30天平均损耗率
- 默认目标利润率为50%（可在设置中调整）
- 参考同类花束历史售价（如有）
- 展示：成本、损耗分摊金额、目标利润金额、建议售价
- 店主可手动调整最终售价

**验收标准：**
1. 定价建议实时计算
2. 手动调价后展示"建议价 vs 实际价"对比
3. 实际售价低于成本价时，红色警示"低于成本"
4. 历史售价参考展示近5次同类花束的售价范围

**业务规则：**
- BR-048：建议售价 = 花束成本 × (1 + 花材加权损耗率) × (1 + 目标利润率)
- BR-049：目标利润率默认50%，可在设置中调整（范围10%-200%）
- BR-050：售价低于成本时系统发出警告但不阻止保存
- BR-051：定价建议为花店版功能

## 9.2 小程序端原型说明

- **查看入口：** [打开小程序端原型](./UI原型-小程序端.html)
- **设计目标：** 面向店主、花艺师、店员的一线高频操作，强调3步内完成入库、出库、损耗登记等核心任务。
- **原型重点：**
  1. 首页展示今日库存、本周损耗率、销售额、节日倒计时和临期预警。
  2. 花材管理体现鲜花行业属性：分类、单位、保鲜期、库存状态、免费版30种花材限制。
  3. 采购入库自动计算保鲜到期日，减少店员人工推算。
  4. 损耗登记突出损耗原因选择和损耗率即时反馈。
  5. 节日备货预测体现S/A/B节日等级、历史销量对比、当前库存差距。
  6. 花束成本核算展示主花/配花/配叶/包装资材的成本拆分和定价建议。

## 9.3 PC管理后台原型说明

- **查看入口：** [打开PC管理后台原型](./UI原型-PC管理后台.html)
- **设计目标：** 面向店主/经营者的数据分析和后台管理，符合PRD Skill对WEB/PC端"全站单页应用"的原型规范。
- **原型形态：** 单文件HTML；电脑显示器外框；内部只有一个完整后台应用壳；通过左侧导航在同一应用内切换经营看板、损耗分析、备货预测、花材与库存、花束配方、基础设置等功能区，而不是多个页面纵向分散展示。
- **原型重点：**
  1. 经营看板集中展示入库量、损耗量、损耗率、销售额、损耗趋势和高损耗排行。
  2. 损耗分析支持按花材、原因、批次等维度观察损耗结构，并提供改善建议。
  3. 节日备货预测展示全年节日日历、情人节备货建议单、历年销售对比和置信度。
  4. 花材与库存展示批次、保鲜到期日、库存状态和临期提醒。
  5. 花束配方展示成本、售价、利润率，以及进价变动后的批量重算提示。
  6. 基础设置覆盖保鲜期规则、预警阈值、节日日历、店铺与订阅信息。
