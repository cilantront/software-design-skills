# 1 概述

## 1.1 需求背景

随着"副业经济"和"数字游民"趋势的持续增长，2026年独立开发者、自由职业者及小型工作室群体规模日益壮大。这一群体面临一个普遍痛点：**不清楚自己每个项目的真实时薪和利润**。

现有市场上的解决方案存在明显断层：
- **时间追踪工具**（如Toggl、Clockify）：仅关注"时间花了多少"，不做成本核算
- **财务软件**（如QuickBooks）：功能过于重量级，不适合个人/小团队使用
- **项目管理工具**（如Jira、Asana）：聚焦任务管理，缺乏收益维度分析

**业务价值**：本产品填补"轻量级项目成本核算"的市场空白，帮助用户从"时间记录"升级到"时间价值分析"，做出更明智的接单决策。

**预期目标**：
- MVP阶段（V1.0）：7-10天完成核心功能开发，覆盖计时、成本核算、基础报表
- 上线3个月内获取1000+活跃用户
- Pro版转化率达到5%以上

## 1.2 名词解释

| **名词** | **说明** |
| --- | --- |
| 毛利润 | 项目总收入减去所有可记录成本（外包费、工具费等），不含时间成本 |
| 实际时薪 | 毛利润 ÷ 总投入工时，反映用户每小时真实收益 |
| 利润率 | 毛利润 ÷ 总收入 × 100%，反映项目盈利水平 |
| 活跃项目 | 状态为"进行中"或"已暂停"的项目，免费版限制3个 |
| 成本项 | 为项目发生的各类支出，如外包费、软件订阅费、差旅费等 |
| 工时记录 | 一条时间记录，包含起止时间、时长、关联项目、任务类型 |
| 云同步 | Pro功能，将本地数据同步至云端，支持多设备访问 |

## 1.3 产品介绍

个人项目成本核算器是一款面向独立开发者、自由职业者及小型工作室的轻量效率工具，以"项目"为核心维度，帮助用户追踪工作时长、核算项目成本与收益，清晰掌握每个项目的真实回报率。

产品核心理念：**"时间值多少钱"**——不仅仅是记录时间，而是将时间投入与收入挂钩，让每一分钟的付出都有清晰的价值回报。

### 1.3.1 范围说明

| 项 | 内容 |
| --- | --- |
| 包含功能 | 账户与订阅管理、项目管理（创建/编辑/删除/筛选）、计时器（内置计时/手动补录）、成本核算（毛利润/实际时薪/利润率/成本构成）、报表与可视化（项目报表/汇总报表/多项目对比/数据导出）、客户管理（Pro）、数据与同步（本地存储/云同步/多币种） |
| 不包含功能 | 团队协作功能（多人共享项目）、发票管理、税务计算、移动端APP（V1.1规划） |

```mermaid
flowchart TD
    A[进入登录页] --> B{选择登录方式}
    B -->|邮箱登录| C[输入邮箱和密码]
    C --> D{账号是否存在?}
    D -->|是| E{密码是否正确?}
    E -->|是| F[登录成功，进入仪表盘]
    E -->|否| G[提示密码错误]
    D -->|否| H[提示账号不存在，引导注册]
    B -->|邮箱注册| I[填写邮箱、设置密码]
    I --> J[发送验证邮件]
    J --> K[用户点击验证链接]
    K --> L[注册成功，自动登录]
    L --> F
    B -->|微信登录| M[跳转微信授权页]
    M --> N[用户授权]
    N --> O[获取用户信息，创建/绑定账号]
    O --> F
    B -->|GitHub登录| P[跳转GitHub授权页]
    P --> Q[用户授权]
    Q --> R[获取用户信息，创建/绑定账号]
    R --> F
```

**业务规则说明**

1. 邮箱注册时需验证邮箱格式，密码需满足8位以上、包含字母和数字
2. 第三方登录首次使用时自动创建账号，后续登录自动绑定
3. 登录状态有效期为7天，到期需重新登录
4. 连续5次密码错误锁定账号15分钟

**验收标准**

- [ ] 正常流程：用户使用正确的邮箱和密码可成功登录
- [ ] 正常流程：新用户可通过邮箱注册并完成验证
- [ ] 正常流程：微信/GitHub授权后可自动登录
- [ ] 异常流程：密码错误时显示明确提示
- [ ] 异常流程：邮箱未验证时限制部分功能
