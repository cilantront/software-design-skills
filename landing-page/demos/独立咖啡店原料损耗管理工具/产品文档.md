# 1. 产品概述

## 1.1 产品定位

咖损管家是一款面向中国独立咖啡店及小型连锁咖啡店的**垂直行业原料损耗管理工具**。聚焦"咖啡店原料损耗管理"这一细分场景，以轻量、专注、低价为核心卖点，帮助店主实现原料数字化管理，降低损耗成本。

**不做通用库存管理，只做咖啡店原料损耗管理。**

## 1.2 产品形态

| 端 | 形态 | 定位 | 说明 |
|----|------|------|------|
| 移动端 | 微信小程序 | **核心操作端** | 扫码出入库、开封登记、损耗登记、临期提醒、销售录入等日常高频操作 |
| Web管理端 | 浏览器后台 | **管理看板端** | 经营总览、损耗分析报表、采购管理、系统设置等管理决策场景 |

## 1.3 目标用户

| 角色 | 用户画像 | 核心诉求 | 使用端 |
|------|---------|---------|--------|
| 店主 | 咖啡店老板，关注经营成本和利润 | 降低损耗成本、掌握经营数据、管理店铺 | 移动端 + Web端 |
| 店长 | 受店主委托管理日常运营，可管理多店 | 高效管理库存、及时处理临期、生成采购单 | 移动端 + Web端 |
| 店员 | 执行日常出入库和损耗登记操作 | 快速完成扫码出入库、损耗登记，操作简单 | 移动端（小程序） |

## 1.4 版本与定价

| 版本 | 价格 | 功能范围 |
|------|------|---------|
| 免费版 | ¥0 | 最多20个SKU，基础出入库、临期提醒、损耗登记 |
| Pro版 | ¥29/月 | 不限SKU + 损耗分析报表 + 智能采购预测 + 多店管理 + 月度损耗报告 |

## 1.5 MVP范围

MVP聚焦5个核心功能模块，7天内可上线：

1. **原料档案管理** — 扫码/手动新建原料、分类管理
2. **扫码出入库** — 扫码识别原料，快速完成入库/出库
3. **临期提醒** — 保质期追踪，临期自动推送微信通知
4. **损耗登记** — 每日损耗记录，损耗原因分类
5. **采购建议** — 基于销售数据预测采购量，生成采购建议单

# 3. 页面设计说明

## 3.1 移动端页面设计

### 3.1.1 首页（经营看板）

**页面目标：** 一打开小程序就能看到今天最需要关注的事情和快捷操作入口。

**页面布局：**

| 区域 | 内容 | 交互说明 |
|------|------|---------|
| 顶部状态栏 | 店铺名称 + 当前日期 + 用户头像 | 点击店铺名称可切换门店（Pro版） |
| 待办提醒条 | 横向滚动卡片：「X个临期」「X个库存不足」「X个待处理」 | 点击跳转对应列表页 |
| 快捷操作区 | 4个大按钮宫格：「扫码入库」「扫码出库」「损耗登记」「开封登记」 | 点击直接进入对应操作页，扫码按钮调起摄像头 |
| 临期原料卡片 | 展示最紧急的3个临期原料（名称+剩余天数+状态标签） | 点击展开完整临期清单 |
| 今日损耗摘要 | 当月损耗金额、损耗笔数、损耗率 | 点击跳转损耗详情页 |
| 底部TabBar | 首页 / 库存 / 保质期 / 损耗 / 我的 | 5个Tab，当前页高亮 |

**业务规则：**
- 待办提醒条数字实时更新（WebSocket推送或轮询，间隔30秒）
- 临期原料卡片按剩余天数升序排列，≤1天显示红色，2-3天显示橙色，4-7天显示黄色
- 无任何待办事项时，显示"今日无待处理事项 ✅"的友好提示

### 3.1.2 库存列表页

**页面目标：** 快速查看所有原料库存状态，支持筛选和搜索。

**页面布局：**

| 区域 | 内容 | 交互说明 |
|------|------|---------|
| 顶部搜索栏 | 搜索框（原料名称/条码） + 分类下拉筛选 | 搜索支持模糊匹配，实时过滤 |
| 状态筛选条 | 全部 / 正常 / 临期 / 不足 / 过期（横向Tab） | 点击切换筛选，显示对应状态数量Badge |
| 库存列表 | 每个原料卡片显示：名称、分类标签、当前库存量（大字号）、状态标签（颜色编码）、保质期进度条 | 点击进入原料详情页 |
| 浮动操作按钮 | 右下角"+"按钮 | 弹出操作菜单：扫码入库 / 手动入库 |

**库存卡片设计：**
```
┌──────────────────────────────────────┐
│  旺意 阿拉比卡咖啡豆      [咖啡豆]    │
│  ████████████░░░░  78%              │
│  库存：3.5 kg  /  安全库存：2 kg     │
│                    [状态: 正常] 🟢   │
└──────────────────────────────────────┘
```

**状态颜色编码：**
- 🟢 正常（绿色）：库存 ≥ 安全库存，无临期
- 🟡 库存不足（黄色）：库存 < 安全库存
- 🟠 临期（橙色）：有已开封原料距到期 ≤ 3天
- 🔴 过期（红色）：有已开封原料已过期

### 3.1.3 扫码入库页

**页面目标：** 最快速的入库操作，3步完成。

**交互流程：**

```
Step 1: 点击"扫码入库" → 调起摄像头扫码界面
Step 2: 扫码成功 → 自动填充原料信息（名称/品牌/规格），显示确认页
Step 3: 输入入库数量、生产日期（可选）→ 点击"确认入库"→ 完成
```

**页面布局（扫码确认后）：**

| 区域 | 内容 | 交互说明 |
|------|------|---------|
| 原料信息卡 | 原料名称（可编辑）、品牌（只读）、规格（只读）、条码（只读） | 如条码未识别，可手动选择原料 |
| 入库信息表单 | 入库数量（数字键盘）、生产日期（日期选择器）、批次号（可选） | 数量为必填，生产日期默认为今天 |
| 历史入库参考 | 显示该原料最近3次入库记录（日期/数量） | 辅助判断入库量 |
| 确认按钮 | "确认入库" 底部大按钮 | 点击提交，显示成功动画 |

**异常处理：**
- 扫码未识别：提示"未识别到原料"，引导手动选择或手动新建
- 条码对应新原料：弹出新建原料表单，需先完成档案创建才能入库
- 网络异常：暂存本地，联网后自动同步，显示"离线暂存"标识

### 3.1.4 保质期管理页

**页面目标：** 一目了然看到所有已开封原料的保质期状态。

**页面布局：**

| 区域 | 内容 | 交互说明 |
|------|------|---------|
| 顶部统计 | 「X个临期 · X个已过期 · X个正常」 | 实时更新 |
| 紧急处理区 | 红色卡片列表：已过期和≤1天到期的原料 | 每个卡片右侧有"损耗登记"快捷按钮 |
| 临期关注区 | 橙色卡片列表：2-3天到期的原料 | 显示建议处理方式（优先使用/促销） |
| 正常区 | 绿色卡片列表：>3天到期的原料 | 折叠显示，可展开 |
| 浮动按钮 | "开封登记" 按钮 | 进入开封操作页 |

**保质期卡片设计：**
```
┌──────────────────────────────────────┐
│  🟠 明治鲜牛奶                       │
│  开封日期：2026-06-25                │
│  到期日期：2026-06-29（今天）         │
│  ████████████████░░░░  剩余 0 天 🔴  │
│         [立即处理 ▸]                 │
└──────────────────────────────────────┘
```

### 3.1.5 损耗登记页

**页面目标：** 快速完成损耗登记，3步完成。

**交互流程：**

```
Step 1: 选择/扫描损耗原料 → 自动带入原料信息
Step 2: 选择损耗原因（过期/变质/操作失误/其他）+ 输入损耗数量
Step 3: 确认提交 → 库存自动扣减
```

**页面布局：**

| 区域 | 内容 | 交互说明 |
|------|------|---------|
| 原料选择 | 扫码按钮 + 原料搜索选择器 | 从临期清单一键带入 |
| 损耗原因 | 单选标签组：过期 / 变质 / 操作失误 / 其他 | 必选，选择后高亮 |
| 损耗数量 | 数字输入框 + 单位自动显示 | 不超过当前库存量 |
| 备注 | 文本输入框（可选） | 如"开封后变质，有异味" |
| 确认按钮 | "确认报损" 底部大按钮 | 提交后显示成功，库存自动更新 |

### 3.1.6 销售录入页

**页面目标：** 快速录入每日饮品销售数据。

**页面布局：**

| 区域 | 内容 | 交互说明 |
|------|------|---------|
| 日期选择 | 默认今天，可切换日期 | 支持查看历史录入 |
| 饮品列表 | 每个饮品一行：名称 + 当前杯数 + [-1] [+1] 按钮 | 点击 +/- 快速调整杯数 |
| 数字输入 | 点击杯数可弹出数字键盘直接输入 | 适合大批量录入 |
| 配方预览 | 点击饮品展开原料用量详情 | 如"拿铁 = 18g豆 + 200ml奶" |
| 自动换算 | 底部显示今日各原料自动换算消耗量 | 实时计算 |
| 保存按钮 | "保存今日销售" | 保存后可继续修改 |

# 4. 核心交互流程

## 4.1 扫码入库完整流程

```
用户点击"扫码入库"
  → 调起摄像头（权限检查）
  → 扫描商品条码
  → 调用条码数据库查询
     ├─ 匹配成功 → 自动填充原料信息 → 进入入库确认页
     │   ├─ 该原料已存在 → 直接显示入库确认页
     │   └─ 新原料 → 先填写完整档案信息 → 保存后再入库
     └─ 未匹配 → 提示手动选择或新建原料
  → 入库确认页：输入数量、生产日期
  → 点击确认
  → 库存增加，记录入库流水
  → 显示成功提示 + 更新后库存
```

## 4.2 临期提醒处理流程

```
系统每日凌晨2:00定时任务
  → 扫描所有已开封原料
  → 计算距到期日天数
  → 筛选出 ≤ 提醒阈值的原料
  → 生成临期提醒消息
  → 通过微信订阅消息推送给店主/店长
  → 用户点击消息 → 打开小程序 → 跳转临期原料清单
  → 用户选择处理方式：
     ├─ 优先使用 → 标记"已安排使用"
     ├─ 促销处理 → 标记"已安排促销"
     └─ 报损处理 → 跳转损耗登记页（自动带入原料信息）
```

## 4.3 采购建议生成流程

```
系统每周日20:00自动触发（Pro版）
  → 收集近4周各原料出库数据
  → 收集近4周饮品销售数据
  → 基于销售趋势计算下周各原料预测需求量
  → 计算建议采购量 = 预测需求 - 当前库存 + 安全库存
  → 如建议采购量 ≤ 0，不生成该品项
  → 关联供应商信息，预估单价（取最近采购价）
  → 生成采购建议单
  → 推送通知给店主/店长
  → 店主审核 → 可修改品项/数量/供应商 → 确认生成采购单
  → 导出PDF或微信分享给供应商
```

## 5.2 库存管理模块

### 5.2.1 入库规则

- 每次入库创建一条库存批次记录，包含：入库数量、入库日期、生产日期、保质期到期日
- 同一原料可有多批次库存，系统按"先进先出"原则建议出库顺序
- 入库后自动更新该原料的总库存量

### 5.2.2 出库规则

- 出库时默认从最早到期批次扣减（先进先出）
- 用户可手动指定出库批次
- 出库需记录用途：制作饮品 / 样品 / 其他
- 出库后自动更新库存量，如低于安全库存则触发预警

### 5.2.3 库存状态判定

```
IF 有任一已开封批次已过期 → 🔴 过期
ELSE IF 有任一已开封批次距到期 ≤ 3天 → 🟠 临期
ELSE IF 总库存量 < 安全库存量 → 🟡 库存不足
ELSE → 🟢 正常
```

## 5.3 保质期管理模块

### 5.3.1 开封登记

- 选择原料（扫码或列表选择）
- 系统自动记录开封日期 = 当天
- 系统自动计算到期日 = 开封日期 + 开封后保质期天数
- 同一原料可多次开封（不同批次），每次独立计算到期日
- 开封后，该批次进入保质期追踪状态

### 5.3.2 临期提醒规则

| 提醒接收人 | 店主 + 店长 | 店员不接收提醒 |
| 提醒方式 | 微信订阅消息 | 需用户提前授权订阅 |

### 5.3.3 保质期颜色标识

| 状态 | 颜色 | 条件 |
|------|------|------|

### 5.4.1 损耗原因分类

| 原因 | 说明 | 典型场景 |
|------|------|---------|
| 过期 | 超过保质期（含开封保质期） | 牛奶开封后未及时用完 |

### 5.4.2 损耗统计指标

| 指标 | 计算公式 | 说明 |

| 损耗笔数 | 统计周期内损耗记录条数 | |

## 5.5 销售记录模块

### 5.5.1 饮品配方管理

预设常见饮品配方（店主可编辑）：

| 饮品 | 咖啡豆(g) | 牛奶(ml) | 糖浆(ml) | 其他 |
|------|----------|---------|---------|------|
| 美式 | 18 | — | — | 水 |
| 拿铁 | 18 | 200 | — | — |
| 卡布奇诺 | 18 | 120 | — | — |
| 摩卡 | 18 | 180 | 15 | 巧克力粉5g |
| 香草拿铁 | 18 | 200 | 15 | — |
| 抹茶拿铁 | — | 200 | 15 | 抹茶粉3g |

### 5.5.2 自动换算逻辑

```
原料消耗量 = Σ (各饮品销售杯数 × 该原料在该饮品中的用量)
```

例：今日卖出 拿铁20杯 + 美式10杯
- 咖啡豆消耗 = 20×18 + 10×18 = 540g
- 牛奶消耗 = 20×200 = 4000ml = 4L

换算数据用于采购预测的需求量估算。

## 5.6 采购预测模块 ★ Pro版

### 5.6.1 预测算法

```
Step 1: 计算近4周各原料周均消耗量
Step 2: 计算近2周 vs 前2周的趋势系数
  趋势系数 = 近2周消耗量 / 前2周消耗量
Step 3: 预测下周需求量 = 周均消耗量 × 趋势系数
Step 4: 建议采购量 = 预测需求 - 当前库存 + 安全库存
Step 5: 如建议采购量 ≤ 0 → 不生成该品项采购建议
```

### 5.6.2 采购建议单内容

| 字段 | 说明 |
|------|------|
| 原料名称 | |
| 分类 | |
| 近4周消耗量 | 历史参考 |
| 预测下周需求 | 算法计算 |
| 当前库存 | 实时库存 |
| 安全库存 | 原料档案设定 |
| 建议采购量 | 预测需求 - 当前库存 + 安全库存 |
| 预估单价 | 取最近一次采购价 |
| 预估总额 | 建议采购量 × 预估单价 |
| 供应商 | 取该原料常用供应商 |

### 5.6.3 采购单操作

- **编辑**：修改品项/数量/单价/供应商
- **确认**：标记为"已确认"状态
- **导出PDF**：生成格式化的采购单文件
- **微信分享**：生成采购单图片，通过微信发送给供应商

| 并发支持 | 5人同时操作 | 单店场景 |

## 10.1 名词解释
