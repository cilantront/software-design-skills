# 1. 产品概述

## 1.1 产品定位

**课程讲义转练习卡**是一款面向教育内容创作者的AI智能工具，核心价值是**将静态课件快速转化为可交互的练习内容**。产品聚焦"讲义→练习卡/题册"这一垂直场景，通过AI自动提取知识点并生成多题型练习题，帮助讲师将原本数小时的手动出题工作缩短至几分钟。

### 1.1.1 产品愿景

成为小讲师和教研个人用户首选的"课件内容复用"工具，让每一份讲义都能高效转化为优质练习资源。

### 1.1.2 产品核心价值

| 维度 | 价值描述 |
|------|----------|
| **效率** | 从手动出题数小时缩短至AI生成几分钟，效率提升10倍+ |
| **质量** | AI智能提取知识点 + 多题型生成 + 人工审核，保证题目质量 |
| **便捷** | 多渠道导出（小程序/PDF/飞书/Notion），覆盖线上线下场景 |
| **低成本** | 按量付费¥9/份起，无需购买完整LMS系统 |

## 1.2 目标用户画像

### 画像A — 知识付费讲师（张老师）

- **年龄**：28-45岁
- **场景**：独立运营线上课程，有完整PPT课件但缺配套练习
- **痛点**：手动出题耗时、学员缺少课后练习材料
- **期望**：快速生成高质量练习题，一键分享给学员

### 画像B — 教培教研人员（李老师团队）

- **年龄**：25-40岁
- **场景**：机构教研组，需批量为多科目课件配套题库
- **痛点**：题库建设周期长、标准不统一
- **期望**：批量处理、品牌统一、团队协作管理

### 画像C — 大学助教/企业培训师（小李/王经理）

- **年龄**：22-35岁
- **场景**：将教授/企业培训讲义转为复习题或测评
- **痛点**：复习材料制作繁琐、测评需无纸化
- **期望**：快速生成PDF题册或在线测评链接

用户登录后的首页，提供快速入口、关键数据概览和最近创建的练习卡，引导用户快速进入核心操作。

```

│              │                   │                  │
│              │  支持 PPT/PDF/Word │                  │
│              │  单文件≤50MB      │                  │
│              │                   │                  │
│              └───────────────────┘                  │
│                                                     │
│  ┌─ 上传队列 ──────────────────────────────────┐    │

│                                                     │
│              [ 上一步 ]     [ 下一步 → ]             │
│                                                     │
```

### 3.2.3 页面元素详表

| 元素 | 类型 | 状态 | 说明 |
|------|------|------|------|
| 步骤条 | 步骤导航 | 常驻 | 5步流程，当前步骤高亮 |
| 上传区域 | 拖拽/点击区域 | 默认态 | 虚线边框，文件图标居中 |
| 上传区域 | 拖拽/点击区域 | 拖入态 | 边框变蓝，背景浅蓝 |
| 上传区域 | 拖拽/点击区域 | 上传中 | 显示进度条和文件名 |
| 上传区域 | 拖拽/点击区域 | 成功态 | 绿色勾号 + "上传成功，正在解析..." |
| 上传区域 | 拖拽/点击区域 | 失败态 | 红色叹号 + 错误原因 + 重试按钮 |
| 格式提示 | 辅助文本 | 常驻 | "支持 .pptx、.pdf、.docx 格式，单文件不超过50MB" |
| 上传队列 | 列表 | 上传后显示 | 文件名、大小、进度条/状态 |
| 上一步按钮 | 次按钮 | 非首步可用 | 返回上一步 |
| 下一步按钮 | 主按钮 | 上传成功后可用 | 进入知识点确认页面 |

### 3.2.4 交互规则

1. **拖拽上传**：文件拖入区域时，区域边框变为蓝色(#4096FF)，背景色变为#E6F4FF
2. **格式校验**：拖入/选择文件后立即校验格式和大小
   - 格式不支持：弹窗提示"不支持该文件格式，请上传 .pptx/.pdf/.docx 文件"
   - 文件过大：弹窗提示"文件超过50MB限制，请压缩后重试"
3. **上传进度**：实时显示上传百分比进度条
4. **自动解析**：上传完成后自动开始AI解析，显示"正在解析讲义内容..."
5. **解析完成**：自动进入Step2，或显示"解析完成，点击查看知识点"
6. **多文件上传**：订阅版支持多文件，免费版限制单文件
7. **步骤导航**：点击步骤条可跳回已完成的步骤（数据保留）

### 3.3.4 交互规则

1. **默认展示**：解析完成后自动展示知识点大纲，默认选中第一个知识点
2. **展开/折叠**：点击章节名称展开/折叠该章节下的知识点
3. **查看详情**：点击知识点条目，右侧面板显示该知识点的详细内容
4. **编辑知识点**：点击编辑图标后，右侧详情区变为可编辑表单
5. **删除知识点**：点击删除图标，弹出确认框"确定删除知识点「xxx」？"
6. **添加知识点**：点击"添加知识点"，弹出表单（章节选择/新建、标题、摘要、内容）
7. **拖拽排序**：支持拖拽知识点条目调整顺序
8. **确认操作**：点击"确认知识点"按钮进入下一步，按钮上显示知识点总数

### 3.4.3 页面元素详表

| 元素 | 类型 | 说明 |
|------|------|------|
| 题型选择 | 复选框组 | 选择题/判断题/简答题，至少选一种 |
| 题目数量 | 数字输入框 | 每种题型独立设置数量，显示范围提示 |
| 难度等级 | 单选按钮组 | 简单/中等/困难，每个选项附描述说明 |
| 出题范围 | 复选框+树形 | "全部知识点"或按章节勾选 |
| 答案解析开关 | 开关 | 默认开启，关闭后不生成解析 |
| 去重模式 | 开关 | 默认关闭，开启后AI避免生成相似题 |
| 预估信息栏 | 提示栏 | 实时显示预计题目总数和消耗配额 |
| 开始生成按钮 | 主按钮 | 触发生成，按钮变为加载态 |

### 3.4.4 交互规则

1. **题型联动**：勾选/取消题型后，对应的数量输入框显示/隐藏
2. **数量校验**：输入超出范围时红色提示，自动修正为最近有效值
3. **实时预估**：修改配置后实时更新"预计生成X道题目"

| 内容区 | 题目正文 + 选项列表（选择题）/ 对错选择（判断题）/ 文本框（简答题） |
| 答案区 | 正确答案（默认显示） |
| 解析区 | 答案解析（默认折叠，点击展开） |
| 关联区 | 关联知识点名称 |

# 5. 交互规范

## 5.1 全局交互规则

### 5.1.1 加载状态

| 场景 | 加载方式 |
|------|----------|
| 页面加载 | 骨架屏（Skeleton） |
| 文件上传 | 进度条 |
| AI解析 | 加载动画 + 进度文案（"正在解析..."、"正在提取知识点..."） |
| 题目生成 | 进度条 + 当前状态文案（"正在生成选择题..."、"正在生成判断题..."） |
| 导出 | 弹窗内进度条 |
| 按钮操作 | 按钮变为 loading 态，禁止重复点击 |

### 5.1.2 反馈提示

| 类型 | 样式 | 时长 | 使用场景 |
|------|------|------|----------|
| 成功提示 | 绿色顶部通知 | 3秒自动消失 | 保存成功、导出成功 |
| 错误提示 | 红色顶部通知 | 手动关闭 | 操作失败、网络错误 |
| 警告提示 | 黄色顶部通知 | 5秒自动消失 | 配额不足、格式不支持 |
| 确认弹窗 | 模态对话框 | 等待用户操作 | 删除操作、不可逆操作 |

### 5.1.3 空状态

| 页面 | 空状态内容 |
|------|----------|
| 工作台 | "还没有练习卡，立即创建第一个吧！" + CTA按钮 |
| 练习卡管理 | "暂无练习卡" + 插图 + "新建练习卡"按钮 |
| 搜索无结果 | "未找到匹配的练习卡" + 清空搜索按钮 |

## 5.2 创建流程交互规范

### 5.2.1 步骤导航

- 步骤条始终固定在页面顶部
- 已完成步骤可点击跳回，数据保留
- 未完成步骤不可点击
- 点击"上一步"保留当前步骤数据

### 5.2.2 数据持久化

- 每步完成的数据自动保存到后端
- 用户中途退出，再次进入可恢复到上次进度
- 浏览器刷新不丢失数据

### 5.2.3 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 上传中断网 | 提示网络错误，支持断点续传或重新上传 |
| AI解析失败 | 提示"解析失败"，提供重试按钮和联系客服入口 |
| 题目生成失败 | 提示"生成失败"，不消耗配额，支持重试 |
| 保存失败 | 提示保存失败原因，数据保留在页面不丢失 |

## 5.3 响应式设计

### 5.3.1 断点定义

| 断点 | 宽度范围 | 适配策略 |
|------|----------|----------|
| 桌面端 | ≥1200px | 完整布局，左侧导航+主内容区 |
| 平板端 | 768-1199px | 左侧导航收起为图标，主内容区全宽 |
| 移动端 | <768px | 底部Tab导航，单列布局 |

### 5.3.2 关键适配

- 上传区域：桌面端大区域居中，移动端全宽
- 知识点大纲：桌面端左右分栏，移动端上下堆叠
- 题目卡片：桌面端双列，移动端单列
- 练习卡列表：桌面端3-4列网格，移动端单列
