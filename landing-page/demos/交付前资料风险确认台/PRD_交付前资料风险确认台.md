## 1. 产品概述

### 1.1 产品定位

「交付前资料风险确认台」是一款面向**独立顾问、自由职业者及小型 B2B 服务商**的轻量级 SaaS 工具，聚焦于服务交付前的**资料完整性确认**与**资料有效期管理**两大核心场景，帮助用户在交付前高效识别并消除资料缺失/过期风险。

### 1.2 产品愿景

> 让每一次服务交付，都从"资料齐备"开始。

### 1.3 核心价值主张

| 价值维度 | 描述 |
|---------|------|
| **降低遗漏风险** | 系统化管理客户资料清单，自动识别缺失/过期项 |
| **提升协同效率** | 一键生成客户提交链接，减少反复沟通成本 |
| **增强专业形象** | 规范化资料收集流程，输出专业风险报告 |
| **降低使用门槛** | 客户无需注册，打开链接即可提交资料 |

### 1.4 差异化定位

| 对比维度 | 泛用资料收集工具 | 本工具 |
|---------|--------------|--------|
| 核心关注点 | 资料收集本身 | 交付前的风险识别与确认 |
| 有效期管理 | 无/手动管理 | 自动识别过期资料并提醒 |
| 协同模式 | 通用表单 | 面向客户-服务商的轻量协同 |
| 风险可视化 | 无 | 风险看板 + 一键导出报告 |

### 1.5 目标用户

| 用户类型 | 典型特征 | 核心痛点 | 使用频率 |
|---------|---------|---------|---------|
| 独立顾问 | 同时服务 3-10 个客户，单人运营 | 资料催收耗时、遗漏导致交付延期 | 每日 |
| 自由职业者 | 按项目制工作，客户更换频繁 | 不清楚每个项目还缺什么资料 | 按项目周期 |
| 小型 B2B 服务商 | 2-10 人团队，标准化服务流程 | 团队间资料状态不透明，交接混乱 | 每日 |

## 2. 开放问题确认

> 以下 4 个问题已由需求文档结对写作专家提出，并经领域专家A确认。PRD 按确认结论编写。

| 编号 | 问题 | 确认结论 | 在 PRD 中的体现 |
|------|------|---------|---------------|
| **Q1** | 是否需要支持客户通过提交链接在线填写表单？ | **MVP 仅支持文件上传，不支持在线表单**。P2 阶段根据反馈决定。 | M4.5 业务规则 BR-M4-05 标注 |
| **Q2** | 资料有效期"到期日"是手动填写还是自动计算？ | **自动计算为主 + 手动修改为辅（双模式）**。M6.1 增加"到期日计算方式"配置项：① 自动计算 ② 手动指定 ③ 自动+可覆盖。 | M6.1 功能详述中体现三种计算方式 |
| **Q3** | 免费版是否限制提醒次数？ | **免费版仅限制客户数（3 个），不限制邮件提醒次数**。短信提醒作为专业版增值服务。 | 商业模式表格补充短信增值服务说明 |
| **Q4** | 风险报告是否需要包含历史趋势？ | **MVP/P1 仅当前快照，P2 增加历史趋势**。 | M8.2 功能详述中明确标注 |

### 3.3 M1 — 客户管理

#### 功能清单

| 功能ID | 功能名称 | 优先级 | 详细描述 |
|--------|---------|-------|---------|
| M1.1 | 创建客户 | P0 | 输入客户名称（必填）、联系人姓名（必填）、联系方式（手机/邮箱，必填至少一项），创建客户档案。创建成功后自动生成资料清单（从模板或自定义）。 |
| M1.2 | 客户列表 | P0 | 分页展示所有活跃客户，支持按客户名称搜索、按创建时间/状态排序。显示每个客户的资料完成进度（如"5/8 已提交"）和风险等级标签。 |
| M1.3 | 客户详情 | P0 | 单个客户的全景视图：基本信息、资料清单完成状态（逐项）、历史提醒记录、资料提交时间线。 |
| M1.4 | 客户归档 | P1 | 已完成交付的客户可归档。归档后不占活跃客户名额（免费版仅统计活跃客户）。归档客户可随时恢复。 |
| M1.5 | 客户删除 | P2 | 软删除，已归档客户可进一步删除。删除前需二次确认。 |

#### 业务规则

- **BR-M1-01**：免费版最多 3 个活跃客户，达到上限时提示升级专业版
- **BR-M1-02**：客户名称不可重复（同一工作空间内）
- **BR-M1-03**：归档客户不占活跃名额，但资料数据保留
- **BR-M1-04**：删除客户时，关联的资料文件进入 30 天回收站

#### 界面原型

- **客户列表页**：见 UI 原型 `客户列表页`
- **客户详情页**：见 UI 原型 `客户详情页`
- **创建客户弹窗**：见 UI 原型 `创建客户弹窗`

### 3.5 M3 — 资料提交链接

#### 功能清单

| 功能ID | 功能名称 | 优先级 | 详细描述 |
|--------|---------|-------|---------|
| M3.1 | 生成提交链接 | P0 | 为每个客户自动生成唯一的资料提交 URL。客户无需注册即可访问和提交资料。链接包含客户标识 token。 |
| M3.2 | 链接有效期 | P1 | 可设置链接的有效期：7 天 / 30 天 / 90 天 / 永久。过期后链接不可访问，需重新生成。 |
| M3.3 | 链接安全 | P2 | 可选设置访问密码。设置密码后，客户需输入密码才能访问提交页面。 |
| M3.4 | 链接分享 | P0 | 支持一键复制链接。支持通过邮件直接发送链接给客户（邮件内容可自定义）。 |

#### 业务规则

- **BR-M3-01**：链接 URL 格式：`https://{domain}/submit/{unique-token}`
- **BR-M3-02**：每个客户同一时间只有一个有效链接（重新生成会使旧链接失效）
- **BR-M3-03**：链接过期后，已上传的资料不受影响，仅影响后续提交
- **BR-M3-04**：提交页面无需登录，但记录提交者 IP 和设备信息用于安全审计

#### 功能清单

| 功能ID | 功能名称 | 优先级 | 详细描述 |
|--------|---------|-------|---------|
| M4.1 | 文件上传 | P0 | 客户通过提交链接页面上传文件。支持格式：PDF、JPG、PNG、DOC、XLS。单文件 ≤ 50MB。 |
| M4.2 | 上传进度 | P1 | 显示上传进度条（百分比），大文件时提供进度反馈。支持断点续传（网络中断后从断点恢复）。 |
| M4.3 | 资料预览 | P1 | 上传后支持在线预览：PDF 文档、图片（JPG/PNG）直接预览；Office 文档暂不支持预览，提示下载。 |
| M4.4 | 资料替换 | P1 | 客户可替换已上传的资料文件。系统保留历史版本（最多保留 5 个版本），可回溯查看。 |
| M4.5 | 资料确认 | P0 | 服务商可在客户详情中逐项审核资料：标记为"已确认"（绿色）、"需补充"（黄色，附带反馈文字）、"已过期"（红色）。 |

- **BR-M4-04**：资料替换后，自动重置该项的确认状态为"待确认"
- **BR-M4-05**：【已确认 Q1】MVP 阶段提交链接页面仅支持文件上传，不支持在线表单填写

### 3.7 M5 — 自动提醒引擎

#### 功能清单

| 功能ID | 功能名称 | 优先级 | 详细描述 |
|--------|---------|-------|---------|
| M5.1 | 缺失提醒规则 | P0 | 设置资料缺失后的提醒触发间隔：24h / 48h / 72h。超过间隔未提交的资料项自动触发提醒。 |
| M5.2 | 过期提醒规则 | P1 | 设置资料到期前的提醒时间：提前 30 天 / 15 天 / 7 天 / 3 天。可叠加多个提醒节点。 |
| M5.3 | 提醒模板 | P0 | 自定义提醒消息模板。支持变量插入：`{客户名}`、`{缺失项列表}`、`{提交链接}`、`{截止日期}`。预设 3 个默认模板。 |
| M5.4 | 提醒渠道 | P0（邮件）/ P2（短信） | MVP 支持邮件提醒。P2 扩展短信渠道。 |
| M5.5 | 提醒记录 | P1 | 记录每次提醒的发送时间、接收人、发送状态（成功/失败）、客户是否点击提交链接。 |
| M5.6 | 批量提醒 | P2 | 支持手动选择多个客户，批量触发一次提醒。 |

#### 业务规则

- **BR-M5-01**：【已确认 Q3】免费版不限制邮件提醒次数，仅限制活跃客户数为 3 个；短信提醒为专业版增值服务
- **BR-M5-02**：同一资料项在同一提醒周期内不重复发送（避免骚扰）
- **BR-M5-03**：提醒发送延迟 ≤ 5 分钟（从触发到实际发出）
- **BR-M5-04**：提醒邮件默认包含提交链接，客户点击可直接到达缺失项
- **BR-M5-05**：提醒频率上限：同一客户每天最多收到 1 封提醒邮件

### 3.8 M6 — 资料有效期管理

#### 功能清单

| 功能ID | 功能名称 | 优先级 | 详细描述 |
|--------|---------|-------|---------|
| M6.1 | 有效期设置 | P1 | 为资料类型设置有效期规则：固定天数（如"365 天"）或固定日期（如"2026-12-31"）。可在模板中预设，也可在客户清单中单项修改。**到期日计算方式**支持三种模式：① 自动计算（上传时间 + 有效期天数）；② 手动指定（由客户/服务商直接填写到期日）；③ 自动+可覆盖（默认自动计算，允许手动修改覆盖，系统记录修改原因）。可在资料项配置中选择计算方式。 |
| M6.2 | 到期追踪 | P1 | 系统自动计算每份资料的到期日，并标记状态：✅ 正常（>30 天到期）、⚠️ 即将过期（≤30 天）、❌ 已过期。 |
| M6.3 | 续期上传 | P2 | 客户可在原资料位置直接上传续期文件，系统自动更新到期日。 |
| M6.4 | 有效期规则库 | P2 | 预设常见资料有效期规则（如营业执照→1年、保险单→1年、资质证书→3年），用户可自定义规则。 |

#### 业务规则

- **BR-M6-01**：【已确认 Q2】到期日采用"三模式"：① 自动计算（上传时间 + 有效期天数）；② 手动指定（直接填写到期日）；③ 自动+可覆盖（默认自动计算，允许手动修改，需记录修改原因）
- **BR-M6-02**：到期日计算基准：文件上传时间 + 有效期天数
- **BR-M6-03**：客户修改到期日时，需填写修改原因（记录审计日志）
- **BR-M6-04**：资料状态变更（正常→即将过期→已过期）自动触发看板更新

### 3.9 M7 — 风险看板

#### 功能清单

| 功能ID | 功能名称 | 优先级 | 详细描述 |
|--------|---------|-------|---------|
| M7.1 | 总览视图 | P0 | 仪表盘形式展示所有客户的资料完成度概览。显示：总客户数、高风险数（红）、中风险数（黄）、低风险/齐备数（绿）。 |
| M7.2 | 风险标记 | P0 | 颜色标记规则：🔴 红色 = 有过期资料或关键必填项缺失；🟡 黄色 = 有资料即将过期（≤30 天）或非必填项缺失；🟢 绿色 = 资料齐备且在有效期内。 |
| M7.3 | 客户维度视图 | P1 | 按客户展示各资料项状态的详细列表。点击客户卡片可进入客户详情页。 |
| M7.4 | 资料维度视图 | P2 | 按资料类型展示各客户的提交情况（交叉视图），适合批量检查某类资料的收集进度。 |
| M7.5 | 筛选与排序 | P2 | 支持按风险等级、客户名称、最后更新时间筛选和排序。 |

#### 业务规则

- **BR-M7-01**：风险等级计算规则（可在 M10 中配置）：
  - **高风险**：存在已过期资料 或 存在必填项缺失 > 3 天
  - **中风险**：存在即将过期资料（≤30 天）或 存在非必填项缺失
  - **低风险**：所有必填项已提交且无过期
- **BR-M7-02**：看板数据实时更新（WebSocket 或轮询，间隔 ≤ 30 秒）
- **BR-M7-03**：【已确认 Q4】MVP/P1 阶段看板仅展示当前快照，不包含历史趋势图表；历史趋势为 P2 规划

### 3.10 M8 — 风险报告导出

#### 功能清单

| 功能ID | 功能名称 | 优先级 | 详细描述 |
|--------|---------|-------|---------|
| M8.1 | 生成报告（PDF） | P1 | 一键生成 PDF 格式的风险报告。报告内容包含封面、目录、客户汇总、详细风险项、建议措施。 |
| M8.2 | 报告内容 | P1 | 报告包含：报告标题、生成日期、客户名称、资料完成率（%）、缺失/过期明细列表、整体风险等级评估。**本期（MVP/P1）报告为当前状态快照，不含历史趋势数据；历史趋势（如"过去 30 天资料完成率变化"折线图）为 P2 规划功能。** |

| M8.4 | 报告模板 | P2 | 支持自定义报告模板：Logo、标题、落款、页脚。预设 2 个默认模板。 |

#### 业务规则

- **BR-M8-01**：【已确认 Q4】MVP/P1 阶段报告为当前状态快照，不含历史趋势数据；历史趋势功能为 P2 规划
- **BR-M8-02**：报告文件大小：PDF ≤ 10MB（超大附件以链接方式附加）
- **BR-M8-03**：P1 阶段同时支持 Excel 格式导出
- **BR-M8-04**：报告生成时间 ≤ 30 秒（100 个客户以内）

### 3.11 M9 — 团队协作

#### 功能清单

| 功能ID | 功能名称 | 优先级 | 详细描述 |

## 4. 技术约束

### 4.1 技术架构

```
┌─────────────────────────────────────────────┐
│                 前端（SPA）                   │
│   Vue 3 + Element Plus + Pinia              │
│   响应式设计（PC 优先，移动端适配）            │
└────────────────┬────────────────────────────┘
                 │ HTTPS / RESTful API
┌────────────────▼────────────────────────────┐
│               后端服务                        │
│   Node.js (Express/Koa) 或 Python (FastAPI)  │
│   JWT 认证 + RBAC 权限                       │
├──────────────────────────────────────────────┤
│               任务调度                        │
│   定时任务（提醒检测、有效期检查）              │
│   消息队列（提醒邮件发送）                     │
└────────────────┬────────────────────────────┘
                 │
┌────────────────▼────────────────────────────┐
│              数据层                          │
│   PostgreSQL（业务数据）                     │
│   对象存储 OSS/S3（文件存储，加密）           │
│   Redis（缓存、会话、任务队列）               │
└──────────────────────────────────────────────┘
```

### 4.2 技术约束清单

| 约束项 | 要求 | 说明 |
|-------|------|------|
| 开发周期 | 1-2 周（MVP） | 技术栈需支持快速迭代 |
| 部署方式 | SaaS 云部署 | 推荐阿里云/腾讯云，支持弹性伸缩 |
| 传输安全 | 全站 HTTPS（TLS 1.2+） | 强制 HSTS |
| 文件存储 | 加密存储（AES-256） | 对象存储 + 服务端加密 |
| 文件上传 | 单文件 ≤ 50MB | 前端校验 + 后端二次校验 |
| 并发支持 | MVP 100 并发用户 | 基于目标用户规模 |
| 数据库 | PostgreSQL 14+ | 支持 JSON 字段、全文搜索 |
| 缓存 | Redis 6+ | 会话管理、热点数据缓存 |
| 邮件服务 | 第三方 SMTP（如 SendGrid/阿里云邮件推送） | 送达率保障 |
| 浏览器兼容 | Chrome、Firefox、Safari、Edge 最新 2 个版本 | 不支持 IE |
| 响应式 | PC 优先，移动端自适应 | MVP 以 PC 为主 |

### 4.3 关键技术决策

| 决策项 | 选择 | 理由 |
|-------|------|------|
| 前端框架 | Vue 3 + Element Plus | 生态成熟，组件库丰富，开发效率高 |
| 状态管理 | Pinia | Vue 3 官方推荐，轻量且类型安全 |
| 后端框架 | Node.js (Express) 或 Python (FastAPI) | 快速开发，社区活跃 |
| 认证方案 | JWT + Refresh Token | 无状态，适合 SPA 应用 |
| 文件存储 | 阿里云 OSS / AWS S3 | 成熟的云存储方案，支持加密和 CDN |
| 定时任务 | node-cron / APScheduler | 轻量级定时任务调度 |
| 报告生成 | PDFKit (Node) / ReportLab (Python) | 服务端生成 PDF |
| 提醒队列 | Bull (Redis) / Celery | 异步任务处理，保证提醒延迟 ≤ 5 分钟 |
