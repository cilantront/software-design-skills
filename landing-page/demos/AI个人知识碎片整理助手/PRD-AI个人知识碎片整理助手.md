## 1. 产品概述

### 1.1 产品定位

**知识碎片整理助手（FragmentMind）** 是一款面向知识工作者的**AI驱动型个人知识碎片管理工具**，聚焦"收集→分类→检索→复用"四大核心环节，帮助用户告别"收藏即遗忘"的信息焦虑。

**一句话定位：** 你的AI知识管家——收藏不遗忘，碎片变体系。

### 1.2 产品理念

**"收藏≠遗忘"**：传统笔记工具要求用户手动整理，门槛高、坚持难。FragmentMind 将AI自动化作为第一原则——用户只需完成"收藏"这一个动作，后续的分类、标签、摘要、关联全部由AI自动完成，让每一条碎片知识都能被找到、被利用、被连接。

### 1.3 产品形态

| 端 | 形态 | 说明 |
|---|---|---|
| Web端主工作台 | 响应式网页应用 | 核心使用入口，含收件箱、知识库、搜索、图谱全模块 |
| 浏览器插件 | Chrome/Edge/Firefox 扩展 | 网页剪藏入口，3秒完成内容收集 |
| 移动端APP | iOS / Android 客户端 | 移动场景收集、微信/邮件转发、拍照OCR（Pro版多端同步） |

### 1.4 目标用户画像

#### 画像A：陈研究员（知识工作者）
- **年龄：** 32岁，咨询公司分析师
- **现状：** 每天阅读20+篇行业报告/文章，收藏到微信/浏览器后几乎不回头看
- **痛点：** 写报告时需要某个数据点或观点，但翻遍收藏夹也找不到；Notion太复杂，没精力维护
- **期望：** 收藏后AI自动整理，写报告时用自然语言一问就能找到相关内容和来源

#### 画像B：小林（内容创作者）
- **年龄：** 27岁，自媒体博主
- **现状：** 用手机和电脑随时看到好句子/灵感/素材就截图或复制，散落在备忘录、微信收藏、相册各处
- **痛点：** 写稿时想找之前收藏的某个案例，要翻好几个APP；素材没有主题归类，每次创作都从头找
- **期望：** 所有素材汇入一个地方，AI按主题自动分类，写稿时搜主题就能出来一堆相关素材

#### 画像C：张老师（终身学习者）
- **年龄：** 45岁，企业管理层
- **现状：** 每天收藏大量管理/商业/技术类文章，希望构建系统化的个人知识体系
- **痛点：** 收藏的文章越来越多，但知识点之间缺乏关联，看了就忘，无法形成体系化认知
- **期望：** AI能帮助发现知识之间的关联，通过知识图谱看到自己的知识版图

## 3. 核心功能详细设计

### 3.1 模块一：多源内容快速收集

#### 3.1.1 Web端-快速收集面板

**功能描述：** 用户在Web端主工作台可通过多种方式快速将内容汇入收件箱，是整个产品链路的第一步。

**页面元素：**
- 顶部：收件箱标题 + 笔记数量统计 + 列表/卡片视图切换控件（仅切换当前列表展示方式）
- 左侧边栏：承载Web端页面/视图切换入口，包括收件箱、知识库、搜索与问答、知识图谱、账户设置、浏览器插件；当前所在页面以高亮态标识，不在页面顶部额外设置独立页面切换按钮
- 快速收集区（收件箱顶部）
  - 文本粘贴区：大面积空白输入框，提示"粘贴文本、拖拽文件到此处..."
  - 文件拖拽区：支持PDF/Word/TXT/Markdown/图片，支持批量拖拽
  - 工具按钮行：[插入链接] [上传图片] [导入文件]
- 笔记列表区
  - 每条笔记卡片：标题 / AI摘要（已处理）或"AI处理中..."（待处理）/ 标签 / 来源图标 / 时间
  - 状态角标：待处理（橙色）/ 已处理（绿色）/ 处理失败（红色）

**交互说明：**
1. **文本粘贴**：用户粘贴文本后，自动检测粘贴内容，弹出迷你表单确认标题（AI预生成）和分类，点击"收录"或按Enter快速提交
2. **文件拖拽**：拖入文件后显示进度条，批量拖拽显示"N个文件正在上传"，完成后逐个进入AI处理
3. **链接抓取**：粘贴URL后自动识别，点击"抓取"按钮拉取网页正文（需服务端解析）
4. **收集成功反馈**：顶部Toast"✓ 已收录，AI正在处理..."，笔记出现在列表顶部，状态为"处理中"

**数据规则：**
- 单条文本内容上限：50,000字符（超出提示分拆）
- 单文件上限：20MB
- 单次批量上传上限：10个文件
- 来源URL自动提取域名作为来源标识

**边界情况：**
- 粘贴内容为空 → 提示"请输入有效内容"
- 文件格式不支持 → 提示"暂不支持该格式，支持PDF/Word/TXT/Markdown/图片"
- 网络中断 → 本地暂存，恢复后自动上传
- 重复URL → 提示"该链接已于X日收录，是否仍要收录？"

#### 3.1.2 浏览器插件-网页剪藏

**功能描述：** 用户在浏览网页时，通过浏览器插件一键将网页内容剪藏到FragmentMind收件箱，是最高频的收集场景。

**操作流程：**
1. 用户点击浏览器工具栏的FragmentMind图标
2. 插件弹窗显示：
   - 若已选中文本 → 显示"剪藏选中内容"和"剪藏整页"两个选项
   - 若未选中 → 显示"剪藏整页"按钮
3. 点击后进入预览页（默认开启预览模式）：
   - 标题：自动提取网页标题，可编辑
   - 内容：清洗后的正文预览，可编辑
   - 来源URL：自动填充
   - AI建议分类：根据内容自动推荐（如"技术"、"商业"）
   - 备注：用户可添加个人备注
4. 点击"确认剪藏" → 内容提交到服务端，弹窗显示"✓ 已收录"
5. 3秒后弹窗自动关闭，或用户手动关闭

**静默剪藏模式：**
- 用户可在设置中开启"静默剪藏"
- 点击剪藏后跳过预览，直接以默认标题和AI分类提交
- 适合快速收集场景，1秒完成

**数据规则：**
- 网页内容自动清洗：去除导航栏、广告、页脚等无关元素，仅保留正文
- 剪藏结果保留原文格式（段落/标题/列表/代码块）
- 图片以URL形式保留，不下载到本地（节省存储）

**边界情况：**
- 未登录 → 弹窗提示登录，引导打开Web端登录页
- 网页内容为空/纯图片 → 提示"当前页面无可剪藏的文本内容"
- 插件版本过旧 → 提示升级插件
- 网络失败 → 本地暂存，下次打开浏览器重试

#### 3.1.3 移动端-系统分享收集

**功能描述：** 用户在微信、邮件、浏览器等任意APP中，通过系统级"分享"功能将内容直接发送到FragmentMind。

**iOS实现：**
- 通过Share Extension接入系统分享菜单
- 用户点击"分享到FragmentMind" → 弹出简化编辑界面 → 确认保存

**Android实现：**
- 通过Share Intent接收系统分享
- 用户在任意APP点击"分享" → 选择FragmentMind → 自动创建笔记

**支持的内容类型：**
- 文本/URL（微信文章、网页链接）
- 图片（单张/多张）
- 文件（PDF、Word等系统支持的文件类型）

**交互说明：**
- 分享后APP无需打开，后台静默处理
- 打开APP后可在收件箱看到刚分享的内容
- 首次使用需登录账号

### 3.2 模块二：AI自动分类与标签

#### 3.2.1 AI处理流水线

**功能描述：** 每条新内容进入收件箱后，自动进入AI处理流水线，完成分类、标签、摘要、向量化四个步骤，全程无需用户干预。

**处理流程（异步执行）：**

```
原始内容 → 文本清洗 → 主题识别 → 关键词提取 → 摘要生成 → 向量嵌入 → 写入数据库
```

**步骤详解：**

| 步骤 | 输入 | 输出 | 技术实现 | 耗时 |
|---|---|---|---|---|
| 1. 文本清洗 | HTML/原始文本 | 纯净文本 | 规则+正则去除标签/广告 | <1秒 |
| 2. 主题识别 | 纯净文本 | 分类（一级+二级） | LLM分类Prompt | 2-5秒 |
| 3. 关键词提取 | 纯净文本 | 3-8个标签 | LLM提取+TF-IDF辅助 | 1-3秒 |
| 4. 摘要生成 | 纯净文本 | ≤100字摘要 | LLM摘要Prompt | 1-3秒 |
| 5. 向量嵌入 | 纯净文本 | 1536维向量 | 嵌入模型API | 1-2秒 |

**总处理时间：** MVP阶段 ≤ 30秒，正式版 ≤ 10秒

**分类体系（默认）：**
- 技术（编程/AI/产品/设计/数据）
- 商业（行业分析/创业/管理/营销/财务）
- 生活（健康/旅行/美食/家居）
- 学习（课程笔记/读书笔记/语言学习）
- 创意（灵感/素材/文案/设计参考）
- 其他

用户可自定义分类，AI分类结果可被用户手动覆盖。

#### 3.2.2 AI处理状态展示

**功能描述：** 让用户清晰感知每条笔记的AI处理进度。

**状态定义：**

| 状态 | 视觉样式 | 显示文本 | 可执行操作 |
|---|---|---|---|
| 待处理 | 橙色圆点 | "等待AI处理" | 查看原始内容 |
| 处理中 | 蓝色脉动圆点 | "AI分析中..." | 查看原始内容 |
| 已处理 | 绿色圆点 | "AI处理完成" | 查看全部AI结果 |
| 处理失败 | 红色圆点 | "处理失败" | 重试、手动编辑、删除 |

**交互说明：**
- 已处理完成的笔记，在列表卡片上显示AI摘要（1行截断）+ 标签（最多3个）
- 点击笔记进入详情，右侧栏显示完整AI结果：分类、全部标签、摘要、处理时间
- 处理失败的笔记，点击显示失败原因（如"内容过短无法分析"），提供"重试"按钮

#### 3.2.3 用户干预与覆盖

**功能描述：** AI结果仅为建议，用户可随时修改覆盖。

**可修改项：**
- **分类**：点击分类标签，弹出分类选择器，选择新分类（AI分类历史保留但不展示）
- **标签**：点击标签可删除，点击"+"添加新标签（支持从已有标签中选择或新建）
- **摘要**：点击摘要旁的编辑图标，进入文本编辑模式
- **标题**：AI未生成标题时（短文本），用户可手动填写

**规则：**
- 用户修改后的结果优先级高于AI结果
- 修改后触发向量重新计算（确保搜索结果准确）
- 修改历史不保留（仅保留最新状态）

### 3.3 模块三：知识库管理

#### 3.3.1 知识库分类浏览

**功能描述：** 用户按AI自动生成的分类或自定义分类浏览笔记，是知识沉淀后的主要使用入口。

**页面布局：**
- 左侧：分类树形导航（支持多级分类）
  - 全部笔记（显示总数）
  - 技术（128）
    - 编程（45）
    - AI（38）
    - ...
  - 商业（86）
  - ...
  - 未分类（12）
- 右侧：笔记列表（默认按更新时间倒序）
  - 每张卡片：标题 / 摘要 / 标签 / 来源 / 更新时间
  - 右上角：收藏星标

**交互说明：**
1. 点击分类 → 右侧列表过滤为该分类下的笔记
2. 点击分类旁的数字 → 显示该分类下笔记数量
3. 右键分类 → 弹出菜单：重命名/移动/删除/新建子分类
4. 拖拽笔记卡片到左侧分类 → 快速移动分类
5. 双击分类名 → 进入重命名模式

**数据规则：**
- 一条笔记仅属于一个分类（单选），但可有多个标签
- "未分类"分类自动收纳AI未能分类的笔记
- 删除分类时，其下笔记自动移至"未分类"

#### 3.3.2 标签系统

**功能描述：** 标签是对笔记的多维度标注，与分类互补（分类是互斥的，标签可叠加）。

**标签来源：**
- AI自动提取（3-8个/条）
- 用户手动添加

**标签云视图：**
- 展示所有标签，字体大小按使用频次缩放
- 点击标签 → 筛选出含该标签的所有笔记
- 支持多标签组合筛选（AND逻辑）
- 右键标签 → 重命名/合并/删除

**标签管理功能：**
- 合并标签：选择2个以上标签 → 合并为一个（历史笔记的标签同步更新）
- 同义词识别：AI定期建议合并相似标签（如"AI"和"人工智能"）

#### 3.3.3 批量操作

**功能描述：** 用户可批量处理多条笔记，提升管理效率。

**操作方式：**
- 鼠标悬停笔记卡片 → 左侧显示复选框
- 勾选多条后，顶部浮现批量操作工具栏

**可执行操作：**
| 操作 | 说明 |
|---|---|
| 移动到分类 | 选择目标分类，批量移动 |
| 添加标签 | 选择或新建标签，批量添加 |
| 移除标签 | 从选中的笔记中移除某标签 |
| 归档 | 批量归档暂时不用的笔记 |
| 删除 | 批量移至回收站 |
| 导出 | 批量导出为Markdown/PDF（Pro） |

**边界情况：**
- 批量操作超过50条 → 弹出确认对话框"确定对X条笔记执行此操作？"
- 操作过程中有新笔记进入 → 不影响当前批量选择

### 3.4 模块四：语义搜索与智能问答

#### 3.4.1 语义搜索

**功能描述：** 用户输入自然语言查询，系统从知识库中检索语义相关的笔记，不依赖关键词精确匹配。

**搜索入口：**
- 顶部导航栏全局搜索框（快捷键 Ctrl/Cmd + K）
- 知识库页面顶部搜索框
- 浏览器插件搜索（Pro）

**搜索流程：**
1. 用户输入查询文本（如"关于远程办公效率的文章"）
2. 查询文本实时向量化（输入停顿300ms后自动触发）
3. 向量数据库检索Top-20相似笔记
4. 按相似度+时间+相关性综合排序
5. 展示搜索结果列表

**搜索结果展示：**
- 每条结果：标题 / 摘要 / 匹配片段（高亮显示相关部分）/ 相似度分数 / 分类标签
- 匹配片段：从笔记中截取与查询最相关的100字片段，关键词高亮
- 来源标识：显示笔记来源（浏览器剪藏/文本粘贴/微信转发等）

**搜索筛选：**
- 范围筛选：全库 / 指定分类 / 指定标签 / 指定时间段
- 排序方式：相关度（默认）/ 时间（最新）/ 收藏优先

**边界情况：**
- 无结果 → 显示"未找到相关内容，换个表述试试？"+ 相关标签推荐
- 查询过短（<2字）→ 不触发搜索，提示"请输入更具体的描述"
- 查询过长（>500字）→ 截取前500字，提示"查询已截断"

#### 3.4.2 AI智能问答（Pro）

**功能描述：** 用户以自然语言提问，AI基于知识库中的相关内容生成综合回答，并标注引用来源。

**交互形式：**
- 对话式面板（位于搜索结果右侧，或独立页面）
- 上方：问题输入框（支持多行输入）
- 中间：AI回答区域（流式输出）
- 下方：引用来源列表（关联的笔记卡片，点击可跳转）

**问答流程：**
1. 用户输入问题（如"总结一下我收藏的关于远程办公的主要观点"）
2. 系统将问题向量化，检索Top-10相关笔记
3. 将笔记内容作为上下文，注入LLM Prompt
4. LLM生成综合回答（流式输出）
5. 回答中标注引用标记[1][2][3]...
6. 回答下方展示引用来源列表（每条来源显示标题+相关片段）
7. 点击引用标记 → 滚动到对应来源
8. 点击来源卡片 → 在新标签页打开笔记详情

**追问机制：**
- 用户可基于上一轮回答继续追问（如"第2点能展开讲讲吗？"）
- 系统保留最近5轮对话上下文
- 上下文+新问题共同检索，生成更深入回答

**回答质量保障：**
- 回答必须基于知识库内容，不引入外部信息
- 若知识库无相关内容 → 回答"抱歉，你的知识库中暂无关于X的内容，建议先收藏一些相关资料"
- 回答末尾固定显示"基于X条笔记生成"

**边界情况：**
- 知识库为空 → 提示"知识库暂无内容，请先收集一些笔记"
- 查询与知识库完全不相关 → 如实告知，不强行回答
- 回答生成失败 → 显示"AI生成失败，请重试"，不扣减问答次数

#### 3.4.3 搜索与问答的Free/Pro差异

| 功能 | 免费版 | Pro版 |
|---|---|---|
| 语义搜索 | 关键词搜索 | 语义搜索 + 关键词搜索 |
| 搜索范围 | 全库 | 全库 + 分类/标签/时间筛选 |
| AI问答 | ❌ | ✅ |
| 多轮追问 | ❌ | ✅ |
| 引用来源展示 | ❌ | ✅ |

### 3.5 模块五：知识图谱（Pro）

#### 3.5.1 全局知识图谱

**功能描述：** 以节点-边的图形化方式展示用户知识库中所有笔记之间的语义关联，让用户直观看到自己的"知识版图"。

**视觉呈现：**
- 节点 = 一条笔记（大小反映关联度，颜色反映分类）
- 边 = 两条笔记间的语义关联（粗细反映关联强度）
- 布局：力导向图算法，自动聚类（同一分类/主题的节点自然聚集）

**交互操作：**
| 操作 | 效果 |
|---|---|
| 鼠标悬停节点 | 显示笔记标题+摘要浮窗 |
| 点击节点 | 右侧弹出笔记详情面板 |
| 拖拽节点 | 调整节点位置 |
| 滚轮缩放 | 缩放图谱 |
| 拖拽空白区 | 平移图谱 |
| 双击节点 | 以该节点为中心展开局部图谱 |
| 右键节点 | 菜单：查看详情/移除节点/固定位置 |

**筛选工具：**
- 按分类筛选（勾选显示/隐藏某分类）
- 按标签筛选
- 按时间范围筛选
- 按关联强度阈值筛选（只显示强关联）

**性能指标：**
- 1000节点首次渲染 ≤ 3秒
- 交互操作帧率 ≥ 30fps
- 超过2000节点 → 自动聚合为"分类簇"，避免过于密集

#### 3.5.2 局部关联图谱

**功能描述：** 从某一条笔记出发，展示与其关联度最高的N条笔记及其关系链路。

**入口：**
- 笔记详情页底部"关联笔记"区域 → 点击"查看关联图谱"
- 全局图谱中双击某节点

**展示内容：**
- 中心节点 = 当前笔记
- 第一圈 = 关联度Top-5的笔记
- 第二圈 = 这5条笔记各自的关联（不含中心节点）
- 边上标注关联类型（"主题相似"/"引用"/"时间相近"）

## 10. 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|---|---|---|---|
| LLM API成本过高 | 中 | 高 | 使用GPT-4o-mini降低成本；摘要/分类合并Prompt；热门内容缓存 |
| 用户收集后不回流使用 | 高 | 高 | 定期推送"本周新增X条知识"邮件；关联推荐引导回流 |
| 免费版到Pro转化低 | 中 | 高 | 100条限制+语义搜索/AI问答作为Pro强卖点；7天免费体验 |
| 浏览器插件审核不通过 | 低 | 中 | 严格遵循各商店规范；不请求过度权限；预留2周审核缓冲 |
| 向量检索准确率不达预期 | 中 | 中 | 多路召回（向量+关键词）；持续优化嵌入模型；用户反馈机制 |
| 数据隐私泄露 | 低 | 极高 | 端到端加密；安全审计；合规认证；用户内容不用于训练 |
