# 1 概述

## 1.1 需求背景

随着共享经济的深入发展和社区治理的数字化转型，住宅小区、共享办公空间、合租房等社区场景中，居民/使用者对工具类物品的临时借用需求日益增长。电钻、梯子、吸尘器、烧烤架等工具单价较高但使用频率低，家家户户都配备既不经济也不环保。

然而，当前社区工具借用完全依赖口头约定或微信群沟通，存在以下痛点：

1. **借出无记录**：物品借出后无法追踪，经常出现"借了不还"或"不知道在谁手里"的情况，导致物品丢失
2. **损坏责任不清**：物品归还后发现损坏，借用人否认，双方各执一词，引发邻里纠纷
3. **超期无提醒**：借出后缺乏有效的催还机制，物品长期被占用，影响其他人的使用需求
4. **物品信息不透明**：社区有哪些可共享的物品、物品当前状态如何，缺乏统一的信息展示渠道

本系统旨在通过轻量化的数字工具，以"扫码即借、到期提醒、损坏留痕"为核心，解决社区共用物品管理难题，让邻里互助更高效、更透明。

## 1.2 名词解释

| **名词** | **说明** |
| --- | --- |
| 物品档案 | 系统中为每件共享工具/物品建立的数字化记录，包含照片、名称、购入价格、使用说明、存放位置等信息 |
| 借用记录 | 用户借用物品时生成的记录，包含借用人、借出时间、预计归还时间、实际归还时间、状态等信息 |
| 专属二维码 | 系统为每件物品自动生成的唯一二维码，用户扫码可快速进入该物品的借用页面 |
| 超期未还 | 借用记录超过预计归还时间仍未确认归还的状态 |
| 损坏登记 | 物品归还时发现损坏，管理员或物品提供者记录损坏情况并发起赔偿协商的流程 |
| 社区管理员 | 负责管理共享物品台账、处理借用纠纷、登记损坏情况的角色，如物业人员、业委会成员、房东等 |
| 物品提供者 | 拥有可共享物品并发布到平台供他人借用的用户 |
| 普通用户/借用人 | 社区居民或共享空间使用者，浏览并借用平台上的共享物品 |

## 1.3 产品介绍

**邻里工具共享与借用管理**是一款面向住宅小区、共享办公空间、合租房等社区场景的轻量级物品借用管理工具。通过为共享工具/物品建立数字化档案，配合扫码借用、超期提醒、损坏追溯等功能，让社区工具共享变得简单可信。

### 1.3.1 范围说明

| 项 | 内容 |
| --- | --- |
| 包含功能 | 物品档案管理（新增/编辑/删除/上下架）、借用管理（一键借用/归还确认/借用记录查询）、超期提醒（到期前提醒/超期催还）、损坏登记与赔偿协商（第二期）、用户管理、系统设置、数据统计 |
| 不包含功能 | 企业级固定资产管理、财务结算系统、通用库存管理、跨社区物品流转、线下物流配送 |

**目标用户：**
- 社区居民/共享空间使用者（借用人）
- 拥有可共享物品的居民或空间管理方（物品提供者）
- 物业工作人员、业委会成员、共享空间管理员、合租房房东（社区管理员）

**使用场景：**
- 小区居民临时需要使用电钻、梯子等工具，通过小程序查看并借用邻居共享的物品
- 共享办公空间管理员管理公共区域的打印机、投影仪等设备借用
- 合租房室友管理共用吸尘器、厨房电器等物品的借用

**核心价值：**
- **对借用人**：快速了解社区有哪些可借用物品，一键扫码即可借走，到期收到温馨提醒
- **对物品提供者**：物品借出有记录、归还有时限、损坏有追溯，再也不用担心物品"有去无回"
- **对社区管理员**：统一管理社区共享物品台账，实时查看物品状态和借用情况，减少纠纷调解工作

# 3 产品功能

## 3.1 用户端小程序功能

### 3.1.1 微信授权登录

**功能描述**

用户首次进入小程序时，通过微信授权快速获取用户信息（头像、昵称、openid），完成注册登录，无需手动填写账号密码。

**优先级与依赖说明：**

| 项 | 内容 |
| --- | --- |
| 优先级 | P0 |
| 依赖需求 | 无 |
| 前置条件 | 用户已安装微信且微信版本 >= 7.0 |

### 3.1.2 微信授权登录—详细流程

```mermaid
sequenceDiagram
    actor User as 用户
    participant Mini as 小程序
    participant Sys as 后台系统
    participant WX as 微信服务器

    User->>Mini: 打开小程序
    Mini->>Mini: 检查本地是否有有效Token
    alt 无Token或Token过期
        Mini-->>User: 展示登录引导页
        User->>Mini: 点击"微信登录"
        Mini->>WX: wx.login() 获取code
        WX-->>Mini: 返回code
        Mini->>Sys: 调用登录接口(code)
        Sys->>WX: 调用code2Session接口
        WX-->>Sys: 返回openid和session_key
        Sys->>Sys: 查询/创建用户记录
        Sys->>Sys: 生成自定义Token
        Sys-->>Mini: 返回Token和用户信息
        Mini->>Mini: 存储Token到本地
        Mini-->>User: 跳转到首页
    else Token有效
        Mini-->>User: 直接进入首页
    end
```

**业务规则说明：**

1. 用户首次登录时，系统自动创建用户记录，默认角色为"普通用户"
2. Token有效期为7天，过期后需重新授权登录
3. 若用户拒绝授权，可继续浏览物品列表，但无法进行借用操作
4. 用户信息（头像、昵称）变更时，下次登录自动同步更新

## 6.2 安全需求

| 编号 | 项（系统数据 / 处理过程） |
| --- | --- |
| 0001 | 用户手机号等敏感信息必须加密存储，使用AES-256加密算法 |
| 0002 | 借用记录中借用人信息对其他用户脱敏展示（手机号中间4位用*替代） |
| 0003 | 所有API接口必须进行身份认证，使用JWT Token |
| 0004 | 管理端接口必须进行权限校验，只有管理员角色可访问 |
| 0005 | 防止SQL注入、XSS攻击，对所有用户输入进行过滤和转义 |
| 0006 | 图片上传限制文件类型（jpg/png/gif）和大小（单张<5MB） |
| 0007 | 防止重复提交，借用、归还等关键操作增加幂等性校验 |
