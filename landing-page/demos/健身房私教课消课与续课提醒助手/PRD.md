# 1 概述

## 1.1 需求背景

随着全民健身意识的提升，私教工作室在近几年快速增长。然而，绝大多数小型私教工作室（1-10名教练）仍依赖 Excel 表格或纸质记录管理会员课包信息，存在以下核心痛点：

1. **消课操作繁琐**：教练每次上课后需手动在表格中记录消课，耗时且易出错，导致课时统计不准确
2. **续课提醒不及时**：会员剩余课时不足时缺乏自动提醒机制，往往等会员主动询问才发现问题，续课转化率低于行业平均水平
3. **训练进度无法量化展示**：会员的身体数据和力量训练变化缺乏系统化记录，无法直观展示训练效果，影响会员满意度和粘性
4. **运营数据缺失**：工作室管理者无法实时了解消课情况、续课率、会员活跃度等核心运营指标

本系统定位于"轻量、专注、低价"的垂直SaaS工具，聚焦"私教课包消课 + 续课提醒 + 训练进度展示"核心业务闭环，为小型私教工作室提供简单易用、成本可控的数字化管理方案。

**预期目标**：
- 单次消课操作时间 ≤ 5秒
- 续课转化率较未使用系统前提升30%以上
- 会员活跃度提升20%以上
- MVP核心功能7-10天完成开发

## 1.2 名词解释

| **名词** | **说明** |
| --- | --- |
| 课包 | 会员购买的私教课程集合，包含课程类型、总课时、单价、有效期等信息 |
| 消课 | 教练在会员完成一次私教课后，在系统中扣减对应课时的操作 |
| 剩余课时 | 会员课包中尚未使用的课程次数，= 总课时 - 已消课时 |
| 续课提醒 | 当会员剩余课时低于设定阈值时，系统自动推送给会员的通知消息 |
| 课前提醒 | 系统在课程开始前指定时间推送给会员的上课提醒通知 |
| 阈值 | 触发续课提醒的剩余课时临界值，默认为5次 |
| 训练进度 | 会员在训练过程中各项身体数据和力量数据的变化记录 |
| 课包模板 | 预设的常用课包配置（如"10次增肌课包"），用于快速创建课包 |
| 补课 | 会员请假后另行安排的弥补课程，不影响课时扣减逻辑 |
| 工作室版 | 付费版本（¥99/月），不限教练和会员数量，含数据报表和品牌化功能 |
| 免费版 | 基础版本，限1名教练、20名会员，零成本使用 |

## 1.3 产品介绍

健身房私教课消课与续课提醒助手是一款面向小型健身工作室和独立私教教练的轻量级SaaS工具。产品通过教练工作台（WEB端）、会员端（微信小程序）和管理后台（WEB端）三个端协同，实现私教课包管理、一键消课、自动化续课提醒、训练进度追踪等核心功能。

**目标用户**：
- 小型健身工作室 / 私教工作室（1-10名教练）
- 独立私教教练
- 健身房运营 / 前台管理人员

**使用场景**：
- 教练上课后快速消课，系统自动通知会员剩余课时
- 会员课时不足时，自动收到续课提醒，联系教练续课
- 会员随时查看剩余课时、训练进度趋势
- 工作室管理者查看运营报表、管理教练团队

**核心价值**：
- 极简消课操作（5秒内完成），取代纸质/Excel记录
- 自动化续课提醒，提升续课转化率
- 训练进度可视化，增强会员粘性与满意度
- 免费版零成本入门，工作室版低价扩展

### 1.3.1 范围说明

| 项 | 内容 |
| --- | --- |

# 3 产品功能

## 3.1 教练工作台功能

### 3.1.1 课表看板

**功能描述**：
课表看板是教练每日工作的核心入口，以时间轴形式展示当天/本周的预约课程安排。教练可在课表中快速完成创建预约、修改预约、取消预约、一键消课等操作。

**优先级与依赖说明**：

| 项 | 内容 |
| --- | --- |
| 优先级 | P0 |
| 依赖需求 | 会员管理、课包管理 |
| 前置条件 | 教练已登录，至少有一个会员和可用课包 |

#### 3.1.1.1 详细流程

```mermaid
flowchart TD
    A[教练进入课表看板] --> B{选择视图}
    B -->|日视图| C[展示当天预约列表]
    B -->|周视图| D[展示本周7天课程网格]
    C --> E{操作选择}
    E -->|创建预约| F[选择会员→课程类型→时间段→确认]
    E -->|消课| G[点击已完成课程→弹出确认窗口→确认消课]
    E -->|修改预约| H[拖拽调整时间/修改信息→同步通知会员]
    E -->|取消预约| I[选择取消原因→确认取消→通知会员]
    F --> J[系统校验会员课包是否充足]
    J -->|充足| K[预约创建成功]
    J -->|不足| L[提示课时不足，引导续课]
    G --> M[系统扣减课时→推送消课通知→判断续课阈值]
```

#### 3.1.1.2 业务规则

1. 日视图按时间段（08:00-22:00，每30分钟一格）纵向排列，已约时段显示会员姓名+课程类型标签
2. 周视图以7列为横轴、时间段为纵轴，色块区分不同课程类型
3. 创建预约时系统自动校验会员是否有对应课程类型的可用课包（状态为"进行中"且剩余课时>0）
4. 消课操作仅对状态为"已确认"且课程时间已到的预约开放
5. 消课确认后5秒内必须完成课时扣减并推送通知给会员
6. 取消预约需选择原因（会员请假/教练调整/其他），用于后续统计
7. 拖拽修改预约时间时，系统实时校验目标时间段是否冲突

**验收标准**：
- [ ] 正常流程：教练可在日视图中看到当天所有预约，点击消课按钮后弹出确认窗口，确认后课时自动扣减
- [ ] 异常流程：会员课包不足时创建预约应提示课时不足并引导续课
- [ ] 性能要求：课表页面加载 ≤ 1秒，消课操作响应 ≤ 500ms

### 3.1.2 一键消课

**功能描述**：
一键消课是系统的核心功能，教练在课表中对已完成的课程执行消课操作。系统自动扣减会员剩余课时、记录消课明细、推送通知给会员，并在剩余课时低于阈值时自动触发续课提醒。

**优先级与依赖说明**：

| 项 | 内容 |
| --- | --- |
| 优先级 | P0 |
| 依赖需求 | 课包管理、提醒通知 |
| 前置条件 | 存在状态为"已确认"的预约记录，会员有可用课包 |

#### 3.1.2.1 详细流程

```mermaid
flowchart TD
    A[教练点击消课按钮] --> B[弹出消课确认窗口]
    B --> C[显示：会员姓名、课程类型、<br/>本次扣减课时、消课前剩余课时]
    C --> D{教练确认?}
    D -->|确认| E[系统执行扣减]
    D -->|取消| F[关闭弹窗]
    E --> G[剩余课时 = 原剩余 - 1]
    G --> H[记录消课明细到流水]
    H --> I[推送消课通知给会员<br/>含剩余课时数]
    I --> J{剩余课时 ≤ 阈值?}
    J -->|是| K[触发续课提醒<br/>微信/短信]
    J -->|否| L[消课完成]
    K --> L
```

#### 3.1.2.2 业务规则

1. 消课确认弹窗必须显示：会员姓名、课程类型、本次扣减课时数（默认1）、消课前剩余课时
2. 支持单次消课多课时（如补课2节连上），教练可在确认窗口修改扣减数量
3. 消课后会员课包中剩余课时实时更新，会员端可立即看到变化
4. 续课提醒触发逻辑：剩余课时 ≤ 教练设定的阈值（默认5次），且满足提醒频率设置
5. 消课操作不可撤销，如需修正需联系管理员在后台处理（记录修正日志）
6. 补课标记的课程同样执行课时扣减，但在流水中标记为"补课"

**验收标准**：
- [ ] 正常流程：教练点击消课→确认→课时扣减成功→会员收到通知，全流程 ≤ 5秒
- [ ] 异常流程：课包剩余课时为0时不允许消课，提示"课时已用完，请先续课"
- [ ] 性能要求：消课接口响应 ≤ 500ms，通知推送延迟 ≤ 5秒

```mermaid
    E -->|消课报表| F[按教练/课程类型/时间统计→可导出]
    E -->|续课报表| G[续课转化率/平均续课周期/课包排名]
    E -->|活跃报表| H[上课频率/请假率/流失风险会员列表]
```

#### 3.3.1.2 业务规则

1. 今日概览数据每小时刷新一次，关键指标（消课次数）实时更新
2. 月度统计支持按周/月切换，展示同比/环比变化
3. 流失风险会员：连续14天未上课且剩余课时>0的会员
4. 所有报表支持导出为Excel文件
5. 数据看板仅管理员角色可见，教练无法访问管理后台

**验收标准**：
- [ ] 正常流程：管理员可查看实时数据看板和各维度报表，可导出Excel
- [ ] 异常流程：无数据时显示空状态引导
- [ ] 性能要求：数据看板加载 ≤ 2秒，报表导出 ≤ 10秒

## 6.2 安全需求

| 编号 | 项（系统数据 / 处理过程） |
| --- | --- |
| 0001 | 所有客户端与服务端之间采用HTTPS加密通讯 |
| 0002 | 用户密码（管理员/教练）使用bcrypt加盐哈希存储，不可逆 |
| 0003 | 会员手机号等敏感信息脱敏展示（中间4位用*替代） |
| 0004 | 接口鉴权采用JWT Token，Token有效期2小时，支持刷新 |
| 0005 | 工作室数据严格隔离，任何接口不可跨工作室访问数据 |
| 0006 | 退款操作需管理员角色权限，普通教练无权限 |
| 0007 | 短信验证码60秒内不可重复发送，有效期5分钟 |
